## 2. 最重要的发现

### 2.1 V3 比 V1 / V2 更有区分度

V1：

- 4 份全是 `medium`

V2：

- 只有旧样本 `S2` 掉到 `weak`

V3 在新样本上：

- `N1 = weak`
- `N2 = medium`
- `N3 = medium`
- `N4 = weak`

这说明：

- 系统已经不再是“什么都判 medium”
- 区分度开始出现

### 2.2 V3 抓到了一部分人工判断

和人工备注最对齐的是：

- `N4`
  模型说：
  `主引擎清楚，但长期靠外部权力空降、女主被动、关系不长新东西`
  人工说：
  `没懂为什么突然就有三个大佬围着女主转，人设没立住，期待值一般`

这两者高度一致。

- `N3`
  模型说：
  `真假千金主引擎明确，但中段重复羞辱、重复救场、续钩太强`
  人工说：
  `没太烂，但记不住，没创新，期待值一般`

这里也基本对齐。

### 2.3 V3 和人工不一致的地方也有价值

最明显的是 `N1`：

- 模型判 `weak`
- 人工打 `3 星`

但人工备注并不是在说它强，而是在说：

> 前几集爽点还行，后面主要是追妻火葬场，可能更适合免费剧集。

这说明：

1. 人工 `3 星` 不一定代表“叙事强”
2. 星级可能更混合了：
   - 免费剧可看性
   - 爽点即时性
   - 是否适合某种发行模式

而不是纯 `NarrativeStrength`

也就是说，`N1` 的分歧不一定是模型错，也可能是：

> 人工标签本身测的不是同一个东西

## 3. 对系统的真实结论

### 3.1 现在可以说什么

现在可以说：

- V3 相比 V1/V2，开始具备基本区分度
- `pseudo_strength_penalty` 是有用的
- `主引擎 + anti_repetition + protagonist_drive` 这些维度确实在拉开样本

### 3.2 现在还不能说什么

现在还不能说：

- 评判器已经稳定可靠
- 现有星级就是完整真值
- 可以直接定权

原因：

1. 样本仍然太少
2. 标签语义混合
3. 只有 4 个新样本，不够稳定

## 4. 这轮最有价值的纠偏

之前我们一直默认：

> 星级 = 强弱真值

现在第一次有证据说明：

> 星级很可能不是单一的叙事真值，而是混合标签。

比如 `N1`：

- 叙事耐久度不强
- 但免费剧集层面可能 still works

这意味着后面不能只做一个：

`overall_rating`

应该至少拆成：

```yaml
HumanLabel:
  narrative_impression:
  immediate_hook_value:
  retention_expectation:
  paid_series_fit:
  free_series_fit:
```

## 5. 下一步最值动作

### 动作 1

不要继续扩样本，先把这批样本的人类标签重新结构化。

原因：

现在最大的噪音不只是模型，而是标签定义本身太粗。

### 动作 2

保留 V3 主评分结构，但把 `free-series-fit / paid-series-fit` 单独加出来，别再混进 narrative。

### 动作 3

下一轮继续 blind，但不要再只用星级对照。  
要同时读备注，并把备注拆成结构化标签。
