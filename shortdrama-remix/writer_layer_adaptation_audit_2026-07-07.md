# writer 层适配审计 · 2026-07-07

## 审计目标

只审一件事：

`writer` 这一层当前为什么会同时出现两个症状：

1. 高压场景台词不够短剧、不够商业、不够狠；
2. 正文里又冒出小说/网文式作者句，视听语言承载不稳。

本次不审：

- `plan`
- `outline`
- `review`
- 用户需求确认

因为当前证据已经足够说明，主要问题不在这些层。

---

## 已确认前提

### 1. 当前主链没有删掉“视听语言”要求

`/episode` 仍明确要求：

- 景别提示；
- 视听语言必须承担戏剧功能；
- 压迫后必须有反应链；
- 不能只写剧情摘要。

证据：

- `skills/short-drama-write/SKILL.md`
- `references/performance-dialogue-brief.md`

所以“视听语言没了”不是规则消失，而是 `writer` 没稳定执行到成稿表层。

### 2. 当前 writer 输入契约收窄方向是对的

真实项目 `灰烬新生` 的 EP2 / EP3 / EP4 rerun 已证明：

- 首稿更像人说话；
- 角色分声线更清楚；
- 没再明显写成翻译腔说明文。

但 `EP5` 中文商业稿对照也已证明：

- 人话感变好；
- 商业高压狠度仍落后于商业稿；
- 差在 `辱感 / 火气 / 当场踩人的狠度`。

结论：

writer 输入契约收窄解决了“别乱吃输入”和“别写成说明文”的一半问题，
但没有自动解决“高压场景商业狠度”。

## 审计结果

### 1. 商业稿基准 `[修改版]灰烬新生.docx`

**证据强度：** 单一样本  
**适用层级：** benchmark / writer 审计参考  
**短剧适配度：** 强，但不普适  
**处理动作：** 只作 benchmark，不直转规则

**确认到的有效信息：**

- 商业稿强的不是“设定更黑”，而是“压得更连续、更狠、更不留缓冲”；
- 高压场景里，角色会直接把受害者的边界话改写成：
  - 扫兴
  - 不识好歹
  - 不给面子
  - 又在装
- 伤害不是只来自主反派，而是：
  - 反派先装善意
  - 权威角色封口
  - 旁人补刀
  - 主角当场失位

**不能直接拿来用的部分：**

- 具体句子；
- 具体人物关系；
- 具体桥段外形；
- 单一稿件的节奏脾气。

**结论：**

它适合拿来定义“差在哪”，不适合直接定义“怎么普适写”。

---

### 2. `02_商业稿写法反推_不可复刻版.md`

**证据强度：** 中等偏强  
**适用层级：** writer / outline  
**短剧适配度：** 强  
**处理动作：** 改写接入

**确认到的有效信息：**

- 商业稿强度来自连续压迫，不是单点强设定；
- 场面语法可抽象成：

```text
外来者入场打破位置
-> 反派先占道德高地
-> 主角解释被误读
-> 权力方补刀
-> 主角承受公开损失
-> 主角留下反击种子
-> 反派追加更脏动作
-> 卡在下一轮危险前
```

- “误判必须变伤害，不只是信息差”；
- “证据/系统只能最后落锤，不能替代人戏”。

**风险：**

- 这份文档仍然主要来自单一商业稿反推，不够项目级普适；
- 适合提炼“行为链”，不适合直接变成 writer 原样读物。

**结论：**

这是做本地 writer brief 的高价值来源之一。

### 3. `dialogue-mastery.md`

**证据强度：** 中等  
**适用层级：** writer / polish  
**短剧适配度：** 部分适配  
**处理动作：** 拆分后改写接入；不再整包默认直喂 writer

**确认到的有效信息：**

- 冲突不够劲时，可按强度升级；
- 掌控者更短，被压者更长；
- 强情绪场景里，句间要有情绪承接；
- 伪善攻击、命令式否定、心死短句都对短剧有用；
- 公开场合的羞辱和站队会放大伤害。

**不适合直接给 writer 默认吃的部分：**

- 模式名过多；
- 网文/读者/弹幕口径混在一起；
- 有些段落默认的是小说/章节阅读体验，不是短剧剧本正文；
- 有些“操作”会诱导 writer 写成总结式说明，而不是现场戏。

**结论：**

不是没用，是接错了形态。

要保留它的：

- 冲突升级
- 权力差句长
- 装弱补刀
- 心死收缩

不要继续把整份原文当 `/episode` 的默认口粮。

## 总体判断

### 已确认的根因

当前 `writer` 层的问题不是单点，而是两个缺口叠在一起：

#### 缺口 A：高压话术施工没接成默认动作

writer 会写：

- 角色要什么
- 谁在压谁
- 谁在受伤

但还不会稳定写出：

- 怎么把边界话改写成“扫兴 / 不给面子 / 又在装”
- 怎么让权威角色封口
- 怎么让旁人补第二刀第三刀
- 怎么在同一回合里完成“误判 -> 补刀 -> 失位”

#### 缺口 B：情绪外化施工没接成默认动作

writer 知道要有反应链、物件、动作、空间。

但一旦场面写到关键处，仍会退回作者句，因为：

- 没有“作者句 -> 可拍动作句”的默认改写习惯；
- 没有“外化失败时怎么补一拍”的施工卡。

## 接入结论

### 不建议继续的做法

不要再让 `/episode` 默认混读这些异构原始资产：

- `dialogue-mastery.md`
- `character-voice-reference-pack.md`
- `dialogue-polish/artifact.md`
- `fail-002-voice-collapse.md`

不是因为它们没用，
而是因为它们：

- 层级不同；
- 任务不同；
- 文体不同；
- 会把 writer 同时拉向“更像人”“更像网文”“更像 polish 改稿”几个方向。

### 建议的做法

做 2 份新的本地化 writer brief：

#### 1. `pressure-attack-brief.md`

只服务 `/episode` 高压场景首稿。

只回答：

- 边界话如何被改写成扫兴 / 不给面子 / 找事；
- 权威角色怎么封口；
- 旁人怎么补刀；
- 反派怎么用假善意加第二层伤害；
- 什么叫“压她服从”和“踩她失位”的区别。

#### 2. `visual-externalization-brief.md`

只服务 `/episode` 情绪落地。

只回答：

- 哪些句子属于作者总结句；
- 遇到这类句子时，怎么改成：
  - 物件变化
  - 站位变化
  - 手部动作
  - 呼吸 / 声音 / 视线
  - 身体反应
- 怎么补“镜头拍得出来”的最后一拍。

## 本轮审计结论

可以先落一个明确决策：

1. 商业稿只有一个样本，不足以直接定义项目级普适方法论；
2. 但它足够定义当前缺口；
3. 当前 `/episode` 默认直读的原始外部资产，确实存在“层级混读、文体混读、用途混读”问题；
4. 不该粗暴删除，而该先做项目适配，再以本地 writer brief 形式重新接入；
5. 下一步不是继续跑样本，而是：
   - 写 `pressure-attack-brief.md`
   - 写 `visual-externalization-brief.md`
   - 收掉 `/episode` 对异构原始资产的默认混读

这才是当前最小、最准、最不绕的下一步。
