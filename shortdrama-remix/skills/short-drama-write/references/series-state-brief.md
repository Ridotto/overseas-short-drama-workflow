# 全剧状态与长线续写 brief

本文件给 `short-drama-write` 使用。它不是长篇网文工作流，也不是额外 gate；它只负责让一部完整短剧在先交付前 10 集时，仍然保住全剧追看空间、真相梯度、关系状态和后续续写能力。

## 根本原则

只要目标是做一部完整短剧，从 `/plan` 开始就必须有轻量全剧追踪骨架。用户可以先要前 10 集正文，但前 10 集不能被当成孤立样片来设计。

状态追踪的目标不是“检查到完美”，而是防止四种问题：

1. 强真相提前说完，后面没有追看空间。
2. 角色关系写死或倒退，后续火葬场、反杀、复仇没法成立。
3. 付费债务、钩子、伏笔被忘掉。
4. writer 续写时上下文太厚，注意力散掉。

## 分阶段使用

### /plan：生成全剧追踪骨架

`creative-plan.md` 必须在自然语言蓝图里融合以下内容。用户看到的是完整小 draft，不是工程表；但 writer 后续必须能从中抽取执行锚点。

- **全剧核心卖点**：这部剧到底卖什么情绪，不得中途偏移。
- **主情绪与副情绪**：例如不可逆伤害、追妻火葬场、复仇反杀、亲情债、身份碾压。
- **长期真相账本**：哪些真相早揭，哪些真相晚揭，哪些真相不能在首批提前说破。
- **关系状态账本**：核心关系从哪里开始，到哪里结束；哪些关系不能倒退。
- **角色状态账本**：主角、主要反派、火葬场对象的欲望、恐惧、面具、破防点。
- **伏笔 / 债务账本**：哪些债务必须回收，哪些钩子不能空头。
- **爽点兑现账本**：哪些爽点已计划兑现，后续不能用同一种方式空转。

如果这些内容只能写成散表格，说明蓝图还不够成熟；先把它们写进“信息流 / 全剧粗纲 / 当前批集纲 / 当前批状态与禁区”的小 draft。

### /characters：把状态落到角色

`characters.md` 必须能回答：

- 这个角色当前最想要什么？
- 这个角色最怕失去什么？
- 这个角色的常态面具是什么？
- 被压、被夺走、被误判、被看穿时，会怎么反应？
- 面对不同关系对象时，反应是否不同？
- 这个角色本批必须发生什么状态变化？
- 哪些反应不能倒退？

主角和主要反派不能写成固定模板。小配角可以类型化；核心角色必须能根据事件、对象、关系位置产生不同反应，但反应要从同一底层欲望和恐惧里长出来。

### /outline：压成当前批状态

`episode-directory.md` 不只是分集剧情表。每集条目必须把全剧骨架压成当前批执行状态：

- 本集释放什么。
- 本集继续扣住什么。
- 本集让谁的位置改变。
- 本集偿还什么债。
- 本集留下什么债。
- 本集不能提前消费什么。
- 本集用什么可见动作承载源本赚钱功能。

前 10 集必须形成一个“首批状态闭环”：观众知道主冲突、看见不可逆压力、看到主角反击方向、带着更大债务进入下一批。

### /episode：只吃本集速记

写第 N 集前，不要重读一堆外部参考。只从 `creative-plan.md`、`characters.md`、`episode-directory.md`、已有正文和状态增量中抽取本集速记：

```text
上一集状态：
本集必须推进的关系 / 真相 / 债务：
本集不能提前说破的答案：
本集必须可见的代价：
本集角色状态 beat：
本集结束后必须留下的新状态：
```

如果本集速记抽不出来，不要临场写正文，回 `/outline` 补当前集执行包。

### 每集完成后：更新状态增量

每集写完后，在本集文件末尾或项目状态记录中更新增量。增量只记录已确认事实，不新增剧情。

```markdown
## 状态增量

- 已释放信息：
- 继续扣住信息：
- 新增/改变的关系状态：
- 新增/改变的角色状态：
- 已兑现爽点：
- 未偿还债务：
- 后续禁区：
```

增量要短。它是给下一集继承状态，不是写复盘文章。

### /batch-state：批次汇总，不是唯一长线入口

`batch-state.md` 的定位是批次完成后的续写汇总。首批写作时，长线骨架已经在 `creative-plan.md` 和 `episode-directory.md` 里存在；首批完成后再汇总成 `batch-state.md`，供第 11 集以后或下一批写作读取。

下一批开始前，必须先读 `batch-state.md`，再生成下一批 `episode-directory.md`。但 `batch-state.md` 不能替代下一批蓝图，它只负责携带已确认事实、关系状态、真相释放和禁区。

## 本集速记模板

写正文前生成并执行：

```markdown
## 第{N}集本集速记

- 上一集结尾状态：
- 本集商业任务：
- 本集释放：
- 本集继续扣住：
- 本集不可提前消费：
- 本集角色状态 beat：
- 本集关系位置变化：
- 本集一眼可懂代价：
- 本集结尾必须留下的新债：
```

## 状态风险自检

以下问题出现时，先回对应上游，不在正文里硬补：

- 本集没有任何真相释放、关系变化、代价兑现或新债务。
- 本集释放了后续关键答案，导致后面没得追。
- 本集让已经心死、决裂、掌权、破防的角色无理由倒退。
- 本集只在调查、确认、解释、递交，没有当场可见损失。
- 本集钩子和上一集债务无关。
- 本集用了同一种爽点，但没有升级影响范围、揭示深度、身份落差或关系代价。

## 禁止

- 禁止把状态追踪变成用户必须确认的厚表格。
- 禁止每集都回 `/plan` 大修。
- 禁止为了连续性把剧情写成解释性总结。
- 禁止把网文章节、卷、地图、每 3 章/每 50 章等规则搬进短剧。
- 禁止让状态记录新增剧情事实。
