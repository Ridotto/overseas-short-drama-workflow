# Dramatic Carrier Kernel

本文件服务 `short-drama-write`，但不是导演分镜课，不是动作模板库，也不是“多写点镜头”。

它的作用只有一个：让观众不靠作者解释，也能看见权力变化和情绪代价。

最终正文不得输出本文件里的术语、判断或标签。

## Runtime Role

- `/episode`：判断这一拍最适合由什么承载。
- `/review`：判断成稿是否可拍、可演、可剪，是否还在靠作者句和模板动作撑戏。

## `/episode`：Dramatic Carrier Kernel

写每场戏时，先判断这拍的戏剧功能：

- 失位
- 羞辱
- 压迫
- 反击
- 误读
- 封口
- 真相逼近
- 关系变化

再判断：最适合由什么承载？

- 台词
- 沉默
- 站位
- 座位
- 门
- 手
- 物件
- 声音
- 旁观者反应
- 被打断的解释
- 没被接住的动作
- 被迫公开的证据

最后问：

1. 这一拍观众必须看见谁失去什么？
2. 只靠台词够不够？
3. 如果不够，最小需要补哪一个载体？
4. 哪个载体最能让这场的代价一眼可懂？
5. 这一拍能不能直接被演员演、被镜头剪出来？

重点：

- 不是每句都要配动作。
- 不是每场都要堆镜头。
- 不是“可拍”就等于“有戏”。
- 真需要外化时，再补最值钱的一拍。

同一种戏剧功能，也可以用不同承载方式：

```text
同样是“失位”：

写法 A：她话还没说完，椅子已经被别人坐了。
写法 B：门没关，但没人让她进去。
写法 C：证据在她手里，旁边的人却先接过了话。
```

```text
同样是“被封口”：

写法 A：他一句“坐下”，全桌人都不再看她。
写法 B：掌声没停，她的话自己断了。
写法 C：她伸手去拿，盘子先被推回她面前。
```

例子只是说明“载体可以不同”，不是默认动作包。

## 允许的创作动作

- 让代价落在一个最具体的可拍载体上；
- 用站位、物件、声音、沉默承接台词后的压力；
- 用“没被接住的动作”暴露关系变化；
- 用旁观者反应放大羞辱，但只在这场真的需要时使用；
- 让最后一拍承担未偿债务，而不是再解释一遍感受。

## `/review` 最小契约

只保留这些验收口径：

- 是否可拍；
- 是否可演；
- 是否可剪；
- 视听表层是否服务冲突；
- 是否存在作者句 / 心理解释句；
- 是否存在模板化动作。

## Anti-Pattern

不要把它用成：

- 动作模板库；
- 镜头术语堆叠；
- 每句心理都机械外化；
- 所有场景都配动作；
- 默认动作包：
  - 攥拳
  - 垂眼
  - 深吸气
  - 后退半步
  - 指尖发白
  - 空气凝固
  - 眼神一沉

这些不是永远不能用，只是不能成为默认解法。

## Minimal Acceptance

成稿至少要满足：

- 观众不看作者解释，也能看出谁失位、谁受压、谁反击；
- 情绪和代价落在可拍、可演、可剪的载体上；
- 不是靠统一动作包假装有视听语言；
- 压力大的场景能看到最关键的一拍；
- 台词、动作、沉默、物件之间在服务同一冲突，而不是各写各的。
