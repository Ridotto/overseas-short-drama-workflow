# 出海本地化适配 brief

本文件只解决一件事：

```text
用户目标市场 / 平台 / 观众明确后，
如何在不误伤现稿有效性的前提下，
把已经成立的源本赚钱功能放进目标语境。
```

它不是题材百科，不是爽点系统，不是付费卡点系统，也不是自动选题器。

- `genre-guide.md` 负责题材映射、爆款元素和文化符号。
- `satisfaction-matrix.md` 负责爽点类型和爽感机制。
- `paywall-design.md` 负责付费卡点。
- `rhythm-curve.md` 负责节奏。
- 本文件只负责：**已确认的源本赚钱功能，在目标平台/观众语境里用什么本地承载方式表达。**

## 默认姿势

默认用户给的稿已经洗过一轮，也可能已经做过本地化。不要把它当作原始国内稿处理，也不要默认它适配失败。

本地化适配的默认顺序是：

```text
先保护现稿有效点
-> 再识别有证据的适配风险
-> 再给少量高置信建议
-> 用户确认后才进入蓝图或正文执行
```

没有证据，不建议改。证据不足，只标“待用户判断/低置信”，不得包装成改写结论。

## 权限边界

### source-import

只做适配质量复核：

- 现稿疑似已经面向哪个市场/平台；
- 哪些本地化处理已经有效；
- 哪些位置可能只是表层本地化；
- 哪些文化动机、关系逻辑、羞辱场域可能失效；
- 哪些有效赚钱功能必须保护，不能为了本地化洗掉。
- 哪些身体风险、真实伤害或亲密张力是有效商业刺激，不能因为“出海合规”被提前洗弱。

source-import 不拍板新题材，不决定重构方向。

### /plan

唯一方向决策层。它根据用户目标和 source-import 证据，决定：

- 保留哪些现稿有效点；
- 是否需要轻修、局部承载修正或重做蓝图；
- 每个建议的证据、推理链路、置信度和误伤风险；
- 哪些点必须交给用户确认。

### /outline / /episode / /dialogue-polish

只执行用户已经确认的本地化策略。

不得在写正文时临场扩大本地化范围，不得把一个保留项改成新方向。

### /review

只验收，不重新选题：

- 是否执行了用户确认的本地化策略；
- 是否只做了英文名、地点、black card、charity gala 等表层替换；
- 是否为了本地化洗掉了源本有效爽点、付费债务或关系压力；
- 是否为了显得安全，洗掉了已经成立的身体风险、亲密张力、羞辱压迫或反派新伤害；
- 是否发生未被用户确认的方向性重构。

## 建议必须有证据

任何本地化建议都必须使用以下格式：

```markdown
### {建议标题}

- 建议级别：保留 / 轻修 / 局部承载修正 / 建议重做蓝图
- 置信度：高 / 中 / 低
- 文本证据：第几集 / 哪个场面 / 哪句或哪组动作
- 判断：
- 推理链路：
- 预期保护或增强的源本赚钱功能：
- 误伤风险：
- 是否需要用户确认：是 / 否
```

推理允许存在，但必须明说链路：

```text
文本证据
-> 该文本依赖的文化/平台假设
-> 目标观众可能如何理解或误读
-> 为什么建议保留/轻修/局部修正/重做
-> 改动会保护或增强哪个赚钱功能
```

## 建议级别

| 级别 | 使用条件 | 权限 |
| --- | --- | --- |
| 保留 | 现稿已经让目标观众容易理解，且能保护源本赚钱功能 | 明确写“不要改” |
| 轻修 | 问题只在台词、表达、少量文化误读或英文腔 | 只能进 dialogue-polish 或局部表达修正 |
| 局部承载修正 | 某个强节点的本地化符号存在，但没有造成目标语境里的可见代价 | 回 /plan 或 /outline 修该节点承载方式 |
| 建议重做蓝图 | 用户目标和现稿目标错位，或核心赚钱功能在目标平台明显失效 | 必须用户确认后才能重做 |

只有高置信建议可以进入执行。中置信建议必须交给用户判断；低置信只记录，不推动改写。

## 不建议改的情况

以下情况优先写“保留”，不要为了显得本地化而改：

- 现稿已经有清晰目标平台语境，且观众不需要理解中式规则也能看懂代价；
- 羞辱、反杀、火葬场、付费债务已经靠可见动作成立；
- 文化符号只是表层不同，但底层关系压力在目标观众里可理解；
- 改动会削弱已经成立的付费钩子、关系债务或源本赚钱功能。

## 必须提醒用户确认的情况

以下情况不得自动执行：

- 从一个目标赛道改到另一个目标赛道；
- 从轻修升级为局部承载修正；
- 从局部承载修正升级为重做蓝图；
- 删除或替换已经成立的核心付费钩子；
- 改变主要关系、核心身份、首批信息释放顺序或结尾债务。

## Review 快查

review 本地化时只问这些问题：

1. 调整是否有用户确认或蓝图依据？
2. 建议或返修是否带文本证据和推理链路？
3. 现稿有效点是否被保留？
4. 改动是否只停在英文名、地点和文化符号？
5. 本地承载是否真的造成可见代价、关系变化或权力变化？
6. 是否为了本地化洗掉了源本赚钱功能？
7. 是否因为合规焦虑把强刺激改成温和沟通、证据说明或安全表达？
8. 是否存在未确认的方向性重构？

## 来源与边界

本文件综合本项目已有 `genre-guide.md`、`satisfaction-matrix.md`、`paywall-design.md`、`rhythm-curve.md`、`source-import`、`short_drama_form_lock_v1.md` 的执行边界整理而成。

它不新增一套海外短剧理论，只把“本地化适配”压成可执行、可验收、低误伤的承载层。
