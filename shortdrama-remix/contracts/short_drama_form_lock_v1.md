# 短剧形态锁 v1

## 目的

防止通用剧本工具把最终产物写成电视剧、网剧、小说化故事或泛型 screenplay。

本契约优先级高于通用 `screenplay_draft`、`scene_writing`、`dialogue_polish`、`quality_gate`。这些能力可以被使用，但只能服务短剧剧本文字的观看和审稿效果，不能反过来改变短剧形态。

## 最终产物定义

最终产物必须是“可见、可听、可刷、可追、可付费”的短剧剧本。

它至少满足：

1. 每集围绕一个可被观众瞬间理解的强冲突推进。
2. 前 3-5 秒必须给出明确冲突、异常、压迫、反差或身份刺激。
3. 前 10-15 秒必须让观众知道“这集为什么要看下去”。
4. 每集必须有至少一次爽点兑现或强压迫升级，不能只有铺垫。
5. 每集结尾必须形成下一集钩子，且钩子必须来自当集冲突推进，不是生硬插一句悬念。
6. 付费点附近必须有“未解决的强问题 + 已产生的情绪债”，不能只是剧情停顿。
7. 场面必须让观众看得见或听得见变化，不能靠内心独白、背景解释、作者旁白、长篇设定说明推进。
8. 对白必须短、硬、带动作压力；不能写成长台词辩论、生活流寒暄、电视剧式抒情。
9. 洗稿项目必须保住源本赚钱功能：源本让观众疼、爽、气、想追、想付费的机制，必须被新壳等价刺激动作承载。
10. 证据、文件、系统、流程、会议、鉴定、录音、公告只能作为触发物；只有当它们当场造成可见损失、反派反咬、关系撕裂、权力转移、身体风险或火葬场动作时，才算短剧戏。
11. 主刺激必须一眼可懂。观众不需要理解行业规则、合同条款、系统权限或复杂背景，就能看出谁失去了身份、身体安全、亲情、关系、权力、体面或选择权。
12. 人物状态必须写进画面和反应。角色可以冷静，但场面不能冷；至少要让观众看见对方被压住、被刺痛、被迫行动、失控或伪装破裂。
13. 视听语言必须服务冲突。动作、空间、物件、伤口、沉默、声音、音乐和镜头提示只能用于增强人物压力和追看，不做空泛装饰。

## 禁止写成的形态

以下情况一票打回：

1. **电视剧化**：长铺垫、多线群像、慢关系、慢情绪、场景信息密度低。
2. **网剧化**：有冲突但节奏松，单场戏停留太久，没有密集反转和短间隔刺激。
3. **小说化**：大量心理描写、旁白解释、抽象情绪、观众看不见的内心活动。
4. **泛型 screenplay**：格式像剧本，但没有短剧的钩子、爽点、付费点、短场景推进。
5. **只换名复刻**：保留源本事件顺序和场面外壳，只改人名、职业、地点。
6. **洗掉有效性**：为了避开相似，丢掉源本真正有效的压迫、反差、爽点和情绪债。
7. **审稿导向写作**：为了过 checklist 写得规整，但没有观众想追的冲击力。
8. **制度证据替代刺激**：把本该当场发生的羞辱、伤害、反杀、火葬场和付费钩子，写成调查、递交、公布、听证、系统权限、会议说明或文件确认。
9. **代价不可见**：剧情逻辑成立，但主要损失需要解释规则才懂；观众第一眼看不出谁痛、谁输、谁丢脸、谁被迫行动。

洗稿距离不是机械去词。通用人名、通用短句、类型套路句可以保留；只换名复刻的判断看事件顺序、场面外壳、角色关系、标志性道具、长句台词和场面调度是否组成仍可识别的源本组合。

## 短剧节奏硬指标

这些不是机械字数要求，而是审稿时必须检查的节奏线：

1. **开场钩子**：第一场不能只是介绍背景，必须直接进入危险、羞辱、误会、身份错位、强需求或强对抗。
2. **短场景推进**：每场戏只承担一个主要功能；完成后立刻推进，不在同一情绪里反复打转。
3. **高频转折**：每集至少有“压迫升级 / 反转 / 爽点兑现 / 新问题抛出”中的两个动作。
4. **信息释放**：信息只能在冲突中释放，不能用独白、旁白、说明会释放。
5. **人物功能清晰**：主角要被压迫、要选择、要反击；反派要制造具体损失；工具人不能占戏。
6. **结尾钩子**：结尾不是“下集待续”语气，而是观众必须想知道下一秒会怎样。
7. **反应链可见**：一句压迫性台词或动作后，必须有被压迫者的身体、表情、行动条件或关系位置变化；不能用“她看了他一眼，转身离开”这种空反应带过关键情绪。

## 商业刺激落地硬规则

这些规则用于防止“洗得干净但不好看”：

1. **赚钱功能先于剧情摘要**：每集先问“观众为什么继续看/付费”，再写剧情发生了什么。
2. **新壳必须承载同级刺激**：换壳后仍要有同等级压迫、误判、羞辱、反杀、火葬场、关系撕裂或命运代价；合理但不刺激的新壳不合格。
3. **反派必须主动造成新损失**：关键窗口里反派不能只等着被揭穿，必须继续施压、反咬、嫁祸、夺权、羞辱或制造风险。
4. **爽点必须可见**：爽点不能停在“真相公布”或“证据成立”，必须让某人当场失去身份、权力、体面、关系、安全或选择权。
5. **付费按钮必须卡动作**：结尾应停在未完成动作、未兑现代价或即将爆开的关系/权力断点前，不应只停在“还有一个秘密/文件”上。
6. **一眼可懂优先**：在多个新壳动作都能承载源本功能时，优先选择观众最少理解成本的版本：门锁死优于权限变更，胸牌被摘优于资格取消，当众跪下优于合同生效，亲人反咬优于证据链完成。
7. **信息释放必须有梯度**：每集要清楚本集释放什么、继续扣住什么、下一集欠什么；真相不能提前说破，也不能只靠解释堆积。

## Writer 使用顺序

writer 的输入优先级：

1. `源本有效性账本`
2. `源本赚钱功能 / 爽点钩子 / 付费压力账本`
3. `洗稿边界`
4. `新壳迁移表`
5. `新壳等价刺激动作表`
6. `创作蓝图包（商业项目形态，用户可见小 draft）`
7. `角色设定 / 欲望 / 关系反应 / 声线倾向`
8. `分集执行包 / 当前批集纲`
9. `短剧形态锁`
10. `视听语言与台词压缩要求`
11. `dialogue-polish` 台词精修：表层话 / 真实目标、声线差异、解释压缩、反应拍、去 AI 味
12. 通用 scene/dialogue/screenplay 技法

通用技法只能排在后面。只要通用技法和短剧形态冲突，以短剧形态为准。

`创作蓝图包` 和 `分集执行包` 不能是两套剧情。前者是用户确认过的自然语言理解，后者只是把同一理解压成 writer 必须执行的硬锚点。

## Reviewer 必查问题

reviewer 必须先回答这些问题，再看格式和文笔：

1. 这集像短剧吗，还是像电视剧/网剧的一场戏？
2. 第一眼有没有冲击力？
3. 观众前 10 秒为什么不划走？
4. 这一集的爽点或压迫升级在哪里？
5. 结尾钩子是不是由当集冲突自然逼出来的？
6. 源本有效性有没有被保留，还是只保留了空壳？
7. 新壳有没有成立，还是改名复刻？
8. 是否存在长台词、说明会、心理独白、生活流空转？
9. 是否把源本赚钱功能写成了制度、证据、文件、系统或会议？
10. 反派是否在关键窗口主动制造新损失，而不是只等着被揭穿？
11. 付费钩子是否卡在动作/关系/权力/命运断点，而不是卡在“还有一个秘密”？
12. 主刺激是否一眼可懂，还是需要解释规则才懂？
13. 人物状态是否写进动作和反应，而不是所有人都冷静、礼貌、站着说完？
14. 视听语言是否真实承载压力，还是只有格式化景别和音乐提示？
15. 信息释放是否有梯度：本集释放、继续扣住、下一债务是否清楚？

判断“只换名复刻”时，不得把常见人名、`don't touch me`、`go away` 等短句套路单独作为失败证据；必须证明它们和同一关系、同一场面、同一动作/道具/调度组成了高识别复刻。

如果 1-5、9-15 任一项不合格，不能进入文风润色；必须回到创作蓝图包、当前批集纲、分集执行包或正文重写。

reviewer 不能把好用的类型套路洗掉。通用短句、常见动作、常见人名和短剧类型句只要没有和源本场面组合成高识别复刻，就不应作为返工理由。

## Callback 边界

reviewer 发现问题后必须标明责任层：

- 源本功能丢失：回 `source-import` 或写稿交接包；
- 新壳承载弱：回 `creative-plan.md`；
- 单集功能弱：回 `episode-directory.md` / 分集执行包；
- 人物状态或声线没执行：回 `characters.md` 或当前集正文；
- 视听语言弱、台词 AI 味重：只允许正文层轻修，不改剧情事实；
- 洗稿高识别复刻：回禁抄边界或对应正文。

同一问题最多允许一次上游返修。返修后仍不过线，主控必须停下来说明风险，不能在用户看不到蓝图或正文前无限 callback。

polish 只拥有表达层权限：压台词、去空话、强化动作承载、补反应链。它不能改剧情、改人物关系、改真相释放顺序或新增付费窗口。

## Quality Gate 输出格式

每轮质检必须输出：

```text
短剧形态判定：通过 / 不通过
主要失败类型：电视剧化 / 网剧化 / 小说化 / 泛型 screenplay / 只换名复刻 / 洗掉有效性 / 制度证据替代刺激 / 代价不可见 / 其他
最致命问题：
证据片段：
应回退到哪一步：源本导入 / 创作蓝图包 / 当前批集纲 / 分集执行包 / 正文重写 / 对白返修
是否允许上游返修：否 / 允许一次
允许进入润色：是 / 否
```

只要 `短剧形态判定` 是“不通过”，`允许进入润色` 必须是“否”。
