# 产品架构与主控路由 v1

日期：2026-07-04

## 当前产品形态

本项目 v1 是 Codex 内运行的本地主控 agent 产品，不是网页 App，不是 CLI 工具。

用户使用方式：

```text
GitHub clone
-> Codex 打开仓库根目录
-> AGENTS.md 路由到 main controller
-> 用户自然语言提出短剧改写需求
-> 主控调度生产链
```

不要求新用户安装额外全局 skill。

## 三层架构

```text
用户层：自然语言 / rewrite-* 高级命令 / 未来网页入口
主控层：shortdrama-main-controller
生产层：source-import / short-drama-write / dialogue-polish / clean reviewer / batch-state / export
能力层：contracts / references / state
```

用户只面对主控层。生产层可以有多个 skill 或 agent，但责任必须由主控统一归口。

## 主控职责

主控负责：

- 识别用户是要新项目、蓝图、首批正文、续批、局部返修、导出还是状态查询；
- 收集必要输入；
- 判断是否可以进入正式写稿链；
- 在正确节点让用户确认；
- 调度生产链；
- 解释 reviewer 结论；
- 归因返修责任层；
- 维护 `run_log.md` 可信度。

主控不负责：

- 自己发明替代生产链；
- 把外部参考原样塞给 writer；
- 让 reviewer 生产剧情；
- 在用户看不到蓝图或正文前无限 callback；
- 绕开当前主控另造入口。

## 用户层命令

自然语言是默认入口。`/rewrite-*` 是高级别名：

| 命令 | 作用 | 内部映射 |
| --- | --- | --- |
| `/rewrite-start` | 开始一个新改写项目，导入源本，确认方向 | `source-import` + `/write-from-source` setup |
| `/rewrite-blueprint` | 生成或刷新创作蓝图 | `/plan -> /characters -> /outline` |
| `/rewrite-write` | 写当前批正文，默认 1-10 | `/episode {range}` |
| `/rewrite-polish` | 台词精修和去 AI 味 | `/dialogue-polish {range}` |
| `/rewrite-review` | 干净审稿 | `/review {range}` + clean reviewer |
| `/rewrite-continue` | 继续下一批 | `/batch-state` + 下一批生产链 |
| `/rewrite-export` | 导出用户交付稿 | `/export` |
| `/rewrite-status` | 查看当前项目状态 | 读取项目文件和 run logs |

不重命名底层内部命令。底层命令仍可在生产层使用。

## 自动推进与确认点

默认自动推进：

- 源本账本生成；
- 写稿交接包生成；
- 蓝图内部锚点生成；
- 当前批正文；
- 台词精修；
- reviewer 自检；
- 导出清理。

必须停下确认：

1. 必填信息缺失；
2. 源本不完整或不适合正式写稿；
3. 新壳方向、市场、语言、改写力度不明确；
4. 洗稿方案 / 创作蓝图需要用户确认；
5. 用户要进入下一批或大改上游。

## 多 agent 边界

- 主控 agent 是唯一用户接口。
- `source-import`、writer、polish 默认可以由主控串联执行。
- clean reviewer 必须保持干净视角。第一轮不得读主控结论、作者自检或历史赞美。
- helper agent 只用于题材研究、外部参考审计、独立审稿等边界清楚的任务。
- 默认禁止 sub-agent 再呼 sub-agent。需要更多帮助时，回到主控统一调度。

## 多批次原则

本产品不是只写 1-10 集样片。

默认理解为：

```text
设计一部完整短剧
-> 先交付当前批
-> 批次完成后刷新状态
-> 继续下一批
```

从 `/plan` 开始建立轻量全剧骨架。每批完成后生成或刷新 `batch-state.md`，再继续 11-20、21-30 或更后面的批次。

`batch-state.md` 至少追踪：

- 已释放和未释放的真相；
- 关系当前状态；
- 角色状态变化；
- 已兑现爽点；
- 未偿还追看债务；
- 不可提前消费项；
- 后续批次禁区。

## 通过标准

这轮产品入口改造完成后，不默认重跑完整剧本。验收方式是冷启动演练：

1. 新用户只读 `README.md` / `QUICKSTART.md`。
2. 用户给出一个源本路径和自然语言需求。
3. 主控能正确进入 `shortdrama-main-controller`。
4. 主控能稳定进入当前生产链。
5. 主控知道必须先做源本导入、方向确认和蓝图确认。
6. 主控知道首批是完整短剧的一部分，不是孤立样片。
7. 主控能说明产物位置和下一步。

如果要宣称新的写作能力已经质量验证，需要另开 clean run。
