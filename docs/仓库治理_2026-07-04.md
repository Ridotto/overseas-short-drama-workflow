## GitHub 替换注意

普通 commit 删除旧文件后，GitHub 的当前 `main` 文件树会变干净，但旧内容仍然存在于 Git 历史里。

如果目标只是让新用户 clone 当前 `main` 时看到干净项目，普通 commit + push 足够。

如果目标是让 GitHub 历史也彻底不含旧样本和旧版本，需要另做 orphan branch 或历史重写再 force push。这一步风险更高，必须单独确认后再做。
