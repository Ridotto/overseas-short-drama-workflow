## 0. 先给结论

当前项目不是“最新回复的地方就是 main”，也不是“所有县城只能留一个”。

更准确的状态是：

```text
几个线程分别贡献了不同价值：

root / 主控 main
  = 当前产物证据最强的主路线底盘

external-chain-rebuild / 独立worktree 3.0
  = 当前根因机制最强的压力保真补丁来源

clean-workflow-validation / 主控 2.0
  = 当前隔离验证和最终交付层暴露问题最清楚的线程

主控 / 商业项目参考 / 产品形态
  = 早期根因判断、失败回路、产品形态和短剧商业标准资产
```

所以最终不应该“选最新线程全盘覆盖”，而应该做一次**效果合成**：

```text
主底盘：root 的 shortdrama-remix 产物证据
+ 机制补丁：external-chain-rebuild 的压力保真 / 不可逆伤害 / 禁止提前泄压
+ 交付修正：clean-workflow-validation 暴露的最终正文格式污染问题
+ 历史约束：旧主控线程沉淀的 scene packet / 失败回路 / 用户反馈归因机制
= 一个新的唯一完整版本
```

一句话：

```text
不要按“最新”收口，要按“效果最好 + 根因最硬 + 风险最少”收口。
```


当前项目已经不是“完全没眉目”，也不是“只能按最新 worktree 走”。


```text

```

所以当前任务不是“停掉所有探索重新按最新走”，而是：

