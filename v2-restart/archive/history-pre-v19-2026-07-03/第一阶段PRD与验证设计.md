# 第一阶段薄 PRD · 爆款短剧机制翻译器

日期：2026-06-28
状态：方向已与用户达成共识，待跑通最小闭环

---

## 1. 产品一句话

把旧爆款短剧里"为什么能爆"的隐性机制，抽成可复用、可重组、能喂给生成的**机制卡片**；再用机制卡片指导**重构 / 生成**出更强的新本。

**它不是评分器。** 分析与评判是零件，不是产品本身。最终目的：产出海外真人 AI 短剧爆款剧本。

## 2. 第一阶段唯一用户

写本子的人（你 / 编剧）。不服务导演、监制、甲方。

## 3. 输入 / 输出

- 输入：1 部旧剧本纯文本（.txt）
- 第一阶段输出：
  1. **机制卡片**（前3集）—— 4 维：主角驱动力 / 爽点闭环 / 推进非重复性 / 卡点设计 + 伪强点检查
  2. **重构方案**（前3集）—— 不变人物/世界观/核心冲突，只动机制层

## 4. 坚决不做（第一阶段）

总分 / 雷达图 / 多维评分表 / strong-medium-weak 分级 / 一键成片 / 多 agent 编排 / 完整 spec。

## 5. 机制卡片的 4 个维度（来自 batch2 已验证判别轴）

| 维 | 问题 |
|---|---|
| D1 主角驱动力 | 主角主动推动，还是被剧情/反派/巧合推着走？ |
| D2 爽点闭环兑现 | 埋的压力是否当集兑现、且兑现后改局？ |
| D3 推进非重复性 | 靠新东西推进，还是靠重复羞辱/误会/救场刷集数？ |
| D4 卡点设计 | 每集是否卡在揭晓前一秒 / 位移点？ |

伪强点清单（命中即扣）：重复羞辱循环 / 误会依赖 / 救场依赖 / 有控制无成长 / 纯奇观无叙事增益。

---

## 6. ⚠️ 跑通闭环时发现的真问题（必须先解决）

**选定的高星样本「重生血族女继承人」前3集本身已经是强本**——四维全满足、零伪强点。

后果：**强本没有明显可重构空间。** 如果硬重构，最可能的盲测结果是"原版赢或打平"，这**证明不了机制卡片有价值**——你无法区分"是机制起作用"还是"原版本来就强"。

### 这说明第一阶段的验证对象选错了。正确的验证应该是：

> **用机制卡片去"修一个弱本"，看修完是否变强——而不是去"修一个已经很强的本"。**

弱本有明确的机制缺陷（伪强点、爽点拖延、主角被动），机制卡片能精准指出"病在哪、怎么治"，重构前后差异大，盲测才干净、才有信号。

### 修订后的最小闭环（建议）

```
取 1 个弱本（人工 2 星，如 Archmage 或 Alpha的禁忌女医）
   │
   ▼
用机制卡片诊断它的前3集："哪几条机制不达标、命中哪些伪强点"
   │
   ▼
按机制卡片做前3集重构（补主动性 / 让爽点当集兑现 / 删重复 / 修卡点）
   │
   ▼
盲测：原版前3集 vs 重构版前3集，二元问题"更想追哪个"
   │
   ▼
重构版赢 ⇒ 闭环成立，机制卡片被证明能指导创作 ⇒ 放大
```

**强本（重生血族）的机制卡片不浪费**：它是"标准答案样板"，告诉我们"修好之后应该长什么样"。

---

## 7. 验证成功标准（事先钉死）

- 盲读者不知道哪份是原版/重构版（A/B 随机）。
- 只问一个二元问题：**"看完这 3 集，你更想追下去的是 A 还是 B？"**
- 成功 = 选重构版。不问"哪个写得好"（会引向文笔评价）。

## 8. 路线图（验证通过才往下走）

| 阶段 | 做什么 | 触发条件 |
|---|---|---|
| 0（当前） | 弱本机制诊断 + 重构 + 盲测 | 进行中 |
