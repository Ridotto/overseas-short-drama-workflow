## 1. 产品定义（已定）

**一句话**：把旧爆款短剧里"为什么能爆"的隐性机制，抽成可复用、可重组、能指导创作的结构；再据此**重构 / 生成**更强的新本。

**最终目标**：产出**海外真人 AI 短剧爆款剧本**。

**它不是什么（重要，防跑偏）**：
- ❌ 不是评分器。分析与评判是**零件**，不是产品本身。
- ❌ 不是"一键成片"全链路（调研证明那是营销话术，无人真打通）。
- ❌ 不是给导演/监制/甲方看的分析报告产品。

**核心定位**：**机制翻译器 + 重构器**。难点不在工程，在"品味"（判断想不想看）和"工艺"（怎么写出想看的）。

## 2. 用户与场景（已定）

- **第一阶段唯一用户**：写本子的人（用户本人 / 编剧团队）。
- 不服务导演、监制、甲方（这是旧方案被带散的主因之一）。
- 下游消费者（导演 / AI 视频 agent）是**远期**输出适配问题，第一阶段不伺候。

## 3. 输入：preference 机制（框架已定，清单待最终确认）

用户输入 = **1 部旧剧本（.txt）+ preference**。

preference = **固定问题清单 + 自然语言补充**。

### 固定问题清单

| 类别 | 字段 | 状态 | 说明 |
|---|---|---|---|
| 必答 | ① 题材 | 已定 | 都市/历史/科幻/奇幻/狼人/悬疑/商战/霸总/家族龙傲天/玛丽苏/真假千金公子/管教熊孩子/穿越/逆袭/扮猪吃虎… |
| 必答 | ② 风格 | 已定 | 男频 / 女频 / lgbt |
| 必答 | ③ 核心钩子/爽点原型 | 建议加 | 复仇/逆袭/隐藏身份/追妻火葬场… |
| 必答 | ④ 情绪基调 | 建议加 | 虐/甜/燃/悬 |
| 必答 | ⑤ 重构自由度档位 | 已定 | 见下方"自由度档位" |
| 收集不强制 | ⑥ 目标市场地区 | 已定不强制 | 北美/东南亚爽点不同；第一版先收集不绑死创作 |
| 收集不强制 | ⑦ 角色数量 | 已定 | 默认建议 ≤4，**但用户可自定义，且与原剧本现有角色数挂钩** |
| 收集不强制 | ⑧ 合规红线/规避内容 | 已定不强制 | 海外厌女/暴力反噬；第一版轻提示，不当主约束 |

> ③④ 标"建议加"= 我推荐但用户尚未明确拍板（待定，见第 9 节）。

### 重构自由度档位（已定）

| 档位 | 含义 | 用法 |
|---|---|---|
| L1 仅优化表达 | 不动剧情，只补信息密度/镜头感/去 cliche | 基本不用 |
| **L2 改桥段** | 保留人设和主线，重写具体情节和反转 | **默认从此档起改** |
| L3 重写故事 | 只借旧本的"题材+核心钩子"当灵感，故事大改 | 按需上调 |

> 用户已确认：基本都从 L2 开始改。L2 意味着写剧本机制是"重写故事级"的重构，不是修修补补。

## 4. ★分析机制（已彻底夯实 · 七信源 + 两轮实验交叉验证）

### 设计原则
1. **维度要少**（多了塌成 medium——V1→V3 教训）。
2. **只"抽取+标记"，不下总分**（总分留给评分机制）。
3. **区分"质量维度"（进权重）和"事实/约束维度"（不进权重）**。

### 质量维度（进评分权重）

| 梯队 | 维度 | 权重 | 抽什么 | 证据 |
|---|---|---:|---|---|
| **命门** | ① 情绪节拍 | 24% | 虐/爽/甜/燃曲线、情绪切换次数、有无情绪空窗 | 5源·定性"短剧的本质" |
| **命门** | ② 钩子/卡点 | 22% | 前3秒/前30秒抓人方式、每集集尾留扣、钩子密度 | 6源·平台第一命门 |
| **命门** | ③ 反转/爽点兑现 | 19% | 反转密度+意外度(=cliche检测)、爽点闭环(埋→兑现→改局) | 7源·唯一全员命中 |
| **地基** | ④ 结构/期待线 | 14% | 是否先立核心期待、节拍骨架、转折点、主副线 | 7源·钩子的前提 |
| **地基** | ⑤ 人物欲望/驱动力 | 11% | 主角想要什么、是否强烈清晰贯穿 | 7源·要欲望不要深度 |
| **薄厚** | ⑥ 信息密度/无废戏 | 10% | 每场新信息量、有无空转/注水、因果密度 | 6源·"命脉"·实验铁证 |

> 权重数字定**梯队和相对大小**（命门>地基>薄厚），非精确系数。
> 唯一标记的不确定：①情绪(24%) vs ②钩子(22%) 谁第一——我让情绪略高（情绪是目的，钩子/反转是手段），属我的推理非信源直述。用户已同意此版。

### 负向 / 门槛维度（不加分）
- **⑦ 人物深度/弧光 → 负向项**：短剧不需要复杂成长，过度铺陈 = 拖慢节奏 = 扣分。只识别"是否过度"。
- **⑧ 台词功能 → 门槛项**：每句是否推进剧情/给信息点。达标不加分，不达标算硬伤。

### 事实 / 约束维度（抽取但不进评分）
- **⑨ 结构事实层**（确定性，代码做）：集数/场景数/角色数/中英配对/卡点位置/**人物关系网**。
  - 注：人物关系网是题材骨架（事实），不是质量维度（七信源几乎无人单独建模"关系"）。
- **⑩ AI 视频可拍性层**（轻提示，不当好坏标准）：场景数超限、角色数、群戏/复杂动作/亲密戏/儿童/动物、动作描述是否具体可拍（**镜头感降到这里**）。

### 贯穿产出
- **⑪ Cliche 清单**：标出"哪几处观众能猜到走向"。是"反转/意外度"维度的负面镜像，直接喂给写剧本机制当"不许这么写"清单。

## 5. ★评分机制（框架已定 · 细则待二次商讨）

> ⚠️ 用户已明确："评分法我们之后再商讨"。本节是**已达成的框架共识**，具体打分规则、亮点定义待定（见第 9 节）。

### 已达成的核心原则
1. **不是"全维达标"，是"命门否决 + 强项决胜"**（四信源一致：现实就是短板砍+长板选，无人算综合平均分）。
2. **维度是"诊断工具"，不是"达标清单"**（防臃肿——用户核心诉求）。
3. **要给分，但用"对照锚点本"的分，不是凭空绝对分**（凭空绝对分会塌 medium——batch2 实证 + LLM 研究 + LitBench）。
4. **极致 > 全面**：一个情绪+钩子拉满但无人物深度的本，应赢过六维都中庸的本。

### 已达成的结构框架（三/四阶段）

```
阶段1 · 命门否决（短板逻辑·二值·一票否决）
  只留全局+客观+致命的硬伤：
    ① 开篇钩子缺失（前60-90秒无冲突/悬念/钩子）
    ② 情绪曲线平坦（通篇无起伏）
    ③ 主线结构崩塌（无核心期待/主角目标自相矛盾）
  任一命中 → pass，打回重写
  【否决项资格三门槛：全局性 + 客观可验证 + 致命，缺一不可】

阶段2 · 双层诊断/评分（标记，不合成单一总分）
  A层 故事质量：6维（钩子/情绪/反转/结构/欲望/信息密度）+ cliche
  B层 生产质量：镜头感/可拍性/AI风险
  亮点单列、问题单列（像 coverage 的 strengths/weaknesses）
  细粒度问题（台词水、人设偏、小bug、合规风险）→ 全在此层标记，不否决

阶段3 · 亮点门槛（用户规则·硬性）
  至少 2 个亮点，否则判"不够吸睛"打回（即使无硬伤）
  → 独立于平均分，专防"六维平庸但无突出长板"

阶段4 · 对照排序
  单本 → 和同类"金标准锚点本"做 pairwise + 相对分
  多本 → A/B 双向 pairwise（去位置偏置）排序
  颗粒度：前重后轻，第1集 > 前3集 > 卡一付费集 > 其余
```

### 已修正的关键决定（回应用户质疑）
- **防误伤**：否决项从 7 条砍到 3 条纯客观全局项。台词水/人设偏/小bug/难拍/合规——全部降级为诊断项，不一票否决。
- **合规** → 只在诊断层标红提示，**不否决**（用户明确要求）。
- **镜头感** → 作为 B 层"生产质量"回归（不在 A 层故事质量里，因评判分两层：故事好不好看 vs 能不能拍成好视频）。
- **判"台词不符人设/注水"需要上下文** → 由分析机制先建立"设定层"（人物/关系/创作意图），评分才有判据。这印证"分析在前、评分在后"。

## 6. ★写剧本机制（框架已定 · 创作工艺待补调研）

> ⚠️ 用户洞察："写剧本是另外一块"。它 = **评判目标 + 创作工艺**，工艺这块我手里证据还不够，需专门补一轮调研后才能写实（见第 9 节）。

### 已定框架
- **性质**：重构器（不是凭空原创器）。输入旧本 + preference，输出重构版。第一版做重构不做原创——因为重构有原版做锚，可对照、可盲测、可验证机制是否有用。
- **自由度**：默认 L2 起（重写桥段和反转），可上调 L3。
- **受评分约束**：写完先过评分，不达标带着诊断意见自己改，达标才交（这是别人 skill 缺的"质检回环"）。

### 已定的工艺要求（来自两轮失败教训，但还不完整）
- **做信息密度加法，不做减法**（E1 教训：我删塌了信息量）。
- **写可拍的镜头**：特写/景别/具体动作/光效，不是"金光大盛"这种舞台说明（E1 教训：毫无镜头感 + 违反 AI 可拍约束）。
- **在套路里塞意外，躲开 cliche 清单**（E1 教训：套路拉满=最 cliche=最不想看）。
- **保留新鲜设定**（E2 启示：强本钩人的是"血月矿脉/毒礼服"这种具体新鲜设定，不是结构骨架）。

### 待补：创作工艺知识库（第 9 节待定）
知道"目标"（评判维度）≠ 知道"怎么做到"（创作工艺）。需补：钩子怎么设计、反转怎么埋（伏笔铺设→回收）、情绪怎么调动（具体场面写法）、镜头化写法、台词工艺（信息密度/潜台词/角色声线区分）、cliche 破法手册。

## 7. 三机制协作全景图（已定）

```
用户输入：旧爆款样本 + preference（固定问题 + 自然语言）
                  ↓
          ┌──────────────┐
          │ ①分析机制     │ 抽6维质量 + 事实层(设定/关系)
          │              │ + cliche清单
          └──────┬───────┘
                 ↓ （设定层 → 给评分当上下文）
          ┌──────────────┐
          │ ③写剧本机制   │ 按preference+L2自由度重构
          │              │ 信息密度↑/镜头可拍/塞意外
          └──────┬───────┘
                 ↓
          ┌──────────────┐  达标+≥2亮点 → 输出
          │ ②评分机制     │ 命门否决→双层诊断→亮点门槛
          │              │ →对照锚点排序
          └──────┬───────┘  不达标 → 带诊断意见打回 ⟲
                 │
          ⟲──────┘  （评分驱动的闭环）

约束：单线串行，第一版不上并行多 agent。
理由：写剧本是强顺序依赖（无大纲不能写、无剧本不能评），
      不是可并行任务。多 agent 是验证通过后的工程优化，现在谈是过早优化。
```

## 8. 第一阶段最小验证闭环（已定方法，待评分细则定后执行）

**做什么**：取 1 个弱本（已选 Archmage，2星）→ 用分析机制诊断前3集 → 按机制重构 → 盲测原版 vs 重构版。

**验证标准**：盲读者不知 A/B 身份，只回答"看完前3集更想追哪个"。重构版赢 = 闭环成立。

**已发生的第一次盲测结果（失败，但有价值）**：
- 用户盲选了**原版**，重构版输。
- 用户判语："A（重构版）非常 cliche，完全没看下去的欲望""信息量非常少且毫无镜头感"。
- 这次失败直接推翻了"机制卡片当生成规则"的路径，催生了现在这套（分析维度大改 + 写剧本要补工艺 + 评分要命门否决）。

**锚点本（已拆，可复用）**：重生血族女继承人（4星强本）已做完机制拆解，作为"重生复仇题材"的金标准锚点本。

## 10. 踩过的坑（防重蹈覆辙）

| 坑 | 表现 | 教训 |
|---|---|---|
| **产品定义被讨论带散** | 一会儿分析器/评分器/多agent/洗稿/全链路 | 先定产品和工作流，不先定架构和评分权重 |
| **把评判当成要先做准的"评分器"** | 反复调 V1→V5 rubric | 评判是在"分析→写→盲测"闭环里长出来的，不是先单独造 |
| **绝对打分塌成 medium** | V3 在 batch2 全给 medium | 用 pairwise / 对照锚点，不凭空绝对打分 |
| **把爆款拆成"机制规则"再套用生成** | 抽出"主角主动+每集打脸"套到弱本 | 抽出的是"套路最大公约数"，套用=生产 cliche 缝合怪。要抽"料"(具体新鲜设定)不是"规则" |
| **重构时做结构减法** | 删冗余→信息量塌缩 | 短剧靠信息密度留人，删信息=泄气。要做加法 |
| **只写舞台说明没镜头感** | "金光大盛"而非"暗绿光在纤维一闪" | 写可拍的镜头；也违反 AI 可拍约束 |
| **维度太多假精确** | 多维评分表塌成一片 | 维度要少，命门就那么几条 |
| **过早工程化 / 上多 agent** | 旧 Layer0-3 架构、agent 编排 | 闭环没验证前不碰架构 |

## 11. 证据来源（所有结论可复查）

本设计的维度/权重/评分逻辑，来自 **7 个独立信源交叉验证 + 2 轮自有实验**，刻意绕开"剧本分析/评分"这类词，从"创作端、平台要求、社区共识、生成端"反推：

| 信源 | 内容 | 代表来源 |
|---|---|---|
| S1 编剧理论 | coverage 标准、McKee/Truby/Save the Cat | StudioBinder, Script Reader Pro, Story Grid |
| S2 短剧/网文实战 | 留人要素、爽点/钩子节奏 | 红果教程, 36氪出海, Real-Reel, Reelytics |
| S3 开源 schema | 剧本质量建模字段 | MASES, CML-Bench, HANNA, clipcurator/vertical-drama |
| S4 写作软件功能 | 反推写作重点 | Final Draft, Plottr, Save the Cat Suite, Sudowrite |

**关键证据要点**：
- 情绪节拍被定性为"短剧的本质"（S2/S6）。
- 钩子是平台"第一命门""前30秒被看得最多"（S5）。
- 反转是 7 信源唯一全员命中的维度。
- 现实选本是"命门否决+强项决胜"，无人算综合分（S1/S2/S5/S6 一致）。
- pairwise+Bradley-Terry 在故事评估达 78% 人类一致 > 绝对打分 73%；绝对打分平局率 60%（LitBench）。
- 创意评分加 CoT 长推理反而降分（LitBench）。
- 评判颗粒度"前十集定生死、生死前三集"（S2 数据）。
