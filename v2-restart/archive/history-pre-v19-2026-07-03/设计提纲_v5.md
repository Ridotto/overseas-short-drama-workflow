## 第一部分 · 产品是什么（第一性定义）

### 1. 一句话
**爆款洗稿器**：输入一个源剧本（已有的爆款 / 已洗过一版的稿），按用户的换壳或集数改编需求，**保留它让观众"想付费、想追下去"的命脉，重写出一个自洽的新本子**。

### 2. 第一性原则（最重要，凌驾一切）
**它本质是"写好剧本"，只是带一个参考前提（源本）。**
- 不是机械的查表替换器 —— 换壳/改编要让新本**自洽、不生搬硬套、不假**。
- 也不是自由创作机器 —— **源本永远是锚，不脱缰**。它照着源本洗，只是洗得动脑、洗得自洽。
- 准确定性：**会动脑的洗稿器**。

### 3. 最高准则（两层，缺一不可）
```
① 本子本身要是个"好剧本"：自洽、逻辑站得住、不假、不堆砌
② 在此基础上，几个最重要的因素（钩子/爽点/卡点/情绪）写得好
→ 两者都满足，观众自然愿意付费、想追下去
```
**警告（防本末倒置）**：不能为了"付费/想看"就刻意堆砌钩子爽点。堆砌出来的是 cliche 烂本（作者曾在 Archmage 重构上犯过此错，被用户判定"非常 cliche、信息少、没镜头感"）。好剧本是地基，关键因素是地基上的加分，不是用加分替代地基。

### 4. 输入 / 输出
- **输入**：1 个源剧本（.txt）+ 用户需求（见第三部分需求问卷）。
  - 现实：用户给的本基本"不能直接用、要洗"（换壳或改编）。无论给什么，工具都先抽源本。
- **输出**：洗完的剧本（纯文本，给人读 / 用户下游环节对接）。先产出前几集，确认后续写。
  - 不输出 JSON、不输出镜头级结构、不碰视频生产。

### 5. 用户
写本子的人（用户 / 编剧）。不服务导演 / 监制 / 甲方。

### 6. 三种洗稿模式（都要会）
| 模式 | 是什么 | 常见度 | 关键动作 |
|---|---|---|---|
| **换壳重写** | 保留命脉节拍，换世界观/身份/场景/背景（天使在草房子被大臣嘲讽 → 士兵在训练场被教官罚跑） | 最常见 | 理解新壳逻辑 → 在新逻辑下重演节拍 → 保证新壳自洽（不是查词典替换） |
| **集数改编** | 保留命脉，改集数（40→60/80 扩写，或→30 缩写），重新编排节奏 | 常见 | 扩写/缩写方法 + 节奏重排，命脉曲线对齐源本 |
| **深度大改** | 180度换（如现代都市复仇→古代狼人商战） | 极少，不专门优化 | 基本接近重写，作为边界情况 |

## 第二部分 · 产品怎么工作（核心 workflow）

> 三个底层特征：**源本为锚**（全程对照源本命脉）+ **写评分离**（重写与自检不同手，防自我批准）+ **改动可追溯**（用户能看到改了哪、为什么）。
> 每个环节都是"理解→判断→创作"，不是"查表→套模板→过关卡"。

```
输入：源剧本（.txt）
   ↓
━ 环节0 · 输入校验 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  挡掉没法处理的：空稿 / 超长超短 / 不是剧本格式
   ↓
━ 环节1 · 需求对齐 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  问清楚要什么（见第三部分需求问卷：风格/题材/平台/集数/卡点/
  付费点/换壳方向/改多狠/绝对不能动的/特别想要的）
  产出：【需求单】← 全程对照
   ↓
━ 环节2 · 拆源本（分析 + 判断好坏，一份产出）━━━━━━━━━
  这一步既"提取命脉"又"判断这个本好不好"，是同一份拆解的两面。
  详见第四部分。产出：
    A 整本拆解报告（宏观命脉 + 好坏判断）
    B 逐拍 beat 卡片（微观施工单，标命脉/标可换的壳）
  产出：【拆解库】← source of truth，后续全程对照
   ↓
━ 环节2.5 · 结构保险丝（轻量兜底）━━━━━━━━━━━━━━━━
  扫骨架是否成立，只拦极少数结构塌方（默认放行，因源本多是爆款，
  结构本就好）。亮红灯→提示"结构有问题，建议大改"，交回人。
   ↓
━ 环节3 · 确定洗稿方案 ━━━━━━━━━━━━━━━━━━━━━━━━
  结合需求单 + 拆解库，定这次怎么洗：
    · 哪种模式（换壳 / 改编 / 深度大改）
    · 【minimum 保留清单】：工具自动识别"命脉候选" → 用户结合
      自己判断增删 → 锁定底线（换壳/改编时死守不丢）
    · 换壳方向：用户一般已有方向；工具可建议、用户确认
  产出：【洗稿方案 + minimum 保留清单】
   ↓
━ 环节4 · 重写（洗稿）━━━━━━━━━━━━━━━━━━━━━━━━━
  按方案重写，详见第五部分。核心是"理解→创作"，不机械：
    · 换壳：理解新壳世界观 → 在新逻辑下重演每拍 → 保证人物
      动机/行为/冲突在新背景站得住、不假
    · 改编：用扩写/缩写方法 + 节奏重排，命脉曲线对齐源本
    · 全程：创作工艺库 + 镜头感硬标准 + 去AI味 + 记录改动来源
  产出：【新本（前几集）+ 改动对照表】
   ↓
━ 环节5 · 独立自检 ━━━━━━━━━━━━━━━━━━━━━━━━━━━
  独立于重写（不同手，防自我批准）。围绕最高准则判断：
    · 好剧本判断：新本自洽吗？换壳后有没有假/出戏的地方？
    · 命脉判断：minimum 清单还在吗？画新本情感曲线 vs 源本，
      峰值位置/强度对齐吗（偏移=命脉丢了）？
    · 关键因素判断：钩子/爽点/卡点/情绪写到位了吗？
    · 镜头感 / 去AI味 / "绝对不能动的"硬约束 核对
  不过 → 带问题回环节4（自改有次数上限，防死循环）
  过 → 输出
   ↓
输出：洗完的本（前几集）+ 改动对照表
   ↓
━ 续写 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  前几集用户认可后，按已定的壳/节奏/风格分批续写（每5-10集一批），
  用户每批轻审，而非一次性续完整本。

【全程大脑】方法论 + 标准库（第六部分）——被各环节调用
【全程记忆】拆解库（source of truth）
```

## 第三部分 · 需求问卷（环节1 要问清楚的）

> 不对齐需求就洗 = 瞎洗。以下问题在动手前问清。基础规格项来自平台调研（ReelShort/DramaBox/红果等）。

**必答：**
- 题材（都市/狼人ABO/重生复仇/霸总/真假千金/逆袭/穿越… 见标准库题材表）
- 风格（男频 / 女频 / lgbt）
- 目标平台 + 目标市场地区（北美/东南亚，爽点不同）
- 集数（源本几集 → 要洗成几集，决定换壳还是改编）
- 卡点 / 付费点位置规格（这是"愿意付费"的直接抓手，必问）
- 洗稿方向：换成什么壳（用户一般已有方向）
- 改多狠：换壳平移 / 集数改编 / 深度大改
- 绝对不能动的（某人设/某条线/某卖点）
- 特别想要的（更虐/节奏更快/某个爽点）

**收集不强制：** 情绪基调（虐/甜/燃/悬）、角色数量偏好（与源本挂钩）、合规规避项（海外厌女/暴力反噬）。

## 第四部分 · 拆源本：要提取什么（环节2 详解）

> 这是整个工具的地基。"拆源本"= 一份产出同时干两件事：**提取命脉**（换壳/改编要守的）+ **判断好坏**（这个本值不值、强在哪弱在哪）。参考 oh-story 拆文报告结构（8大板块）+ 我们的6维分析。

### A. 整本拆解报告（宏观——看清整个本靠什么爆）
```
① 故事核：设定 / 核心爽点行动 / 一句话概括
② 结构划分：分几段、每段功能、结构类型（对话驱动/隐忍布局/倒计时告别…）
③ 情感曲线（整本）：逐节点 ±9、曲线形态（波浪/倒V）、极值在哪个位置%、
   翻转次数  ★命脉之命脉——换壳/改编后这条要对齐
④ 爆点分析：核心爆点是什么、是否叠加爆点（表层爽 + 深层意难平）
⑤ 主反转：类型、铺垫线索几条、铺垫:释放比、时机位置%、
   惊喜度/合理性/情绪冲击 打分
⑥ 钩子体系：开头前3句怎么钩、每集集尾怎么收、钩子回收检查
⑦ 人物：角色功能（叙事角色 vs 行动角色）、可删除性、人物关系网
⑧ 写作手法：用了哪些（信息差锁/物件钩子/童言补刀…）、核心手法 Top3
⑨ 题材赛道 + 话题性/共鸣点（为什么观众有共鸣、会传播）
─── 同时产出"好坏判断"（用6维 + 五维评分，靠山=付费/想看）───
   情绪节拍 / 钩子卡点 / 反转爽点 / 期待线 / 人物欲望 / 信息密度
   + 整体诚实结论：这个本最重要那几个点强不强（强=好，那几点弱=差，
     别的高分救不回来）
```

### B. 逐拍 beat 卡片（微观——换壳/改编的施工单位）
```
一个 beat = 一个剧情节拍（如"女主被当众羞辱"是一拍）
每张卡片：
  【命脉·锁死，换壳改编不动】
    位置（第几集/场）/ 功能（铺垫/羞辱/打脸/反转/卡点）/
    情绪值±9 / 钩子爽点类型 / 这拍在整本爆点链里的角色
  【壳·可换，换壳时替换】
    涉及角色 / 场景地点 / 原文白描 / 篇幅
  【可动性标记】
    核心命脉（锁死） or 过渡铺垫（改编时可增减）
```

### 为什么分宏观+微观两层
```
只有微观beat → 能换壳，但改编集数会迷失（不知道整本曲线长啥样，乱加乱砍）
加宏观报告 → 知道整本靠什么爆、命脉是哪几个
  换壳：微观逐拍换皮，宏观曲线对齐
  改编：宏观曲线/爆点链不动，微观增减过渡beat
  自检：画新本曲线 vs 源本，对齐 = 命脉没丢
```

## 第五部分 · 重写：怎么洗（环节4 详解）

### 5.1 换壳重写（不是查词典替换）
```
错误做法：建对照表 → 逐词替换角色/场景  ← 生搬硬套，会假
正确做法：
  1. 理解新壳的世界观逻辑（军营里怎么羞辱人？不是"嘲讽血统"，
     是"当众体罚/羞辱军衔"）
  2. 在新逻辑下重新演绎每一拍的功能（"羞辱"这拍→军营版的羞辱）
  3. 保证新壳里人物动机/行为/冲突站得住、不假（这是"好剧本"底线）
  4. 命脉锁死：卡点位置/情绪值/钩子爽点类型 1:1 不动
```

### 5.2 集数改编（扩写/缩写 + 节奏重排）
```
扩写方法（健康扩写，非注水）：
  · 支线人物深化（工具人→独立支线，与主线在高潮前合流）
  · 爽点铺垫拉长（1集打脸→压抑2集+积蓄1集+打脸2集）
  · 多层反转加码（1次揭秘→假身份→半真相→真相3段）
  · 情感线细腻化（定情2集→误会拉扯6集）
  健康 vs 注水标准：每集至少1个新冲突/爽点/反转，否则就是水

缩写方法（压不丢爽点）：
  · 砍/合并支线（只留帮主线的部分）· 场景合并（铺垫3集→1集闪回）
  · 爽点提速（压抑-爽周期5集→2集）· 砍对白水分
  保留：核心爽点 + 高光名场面

节奏重排（短剧四段配比）：
  铺垫25% / 冲突升级37.5% / 危机大高潮25% / 收尾12.5%
  高潮间隔10-15集1个（扩写不能稀释到20集才1个）
  情绪曲线：扩写时谷拉长但峰密度不变；缩写时谷压缩峰保留

与命脉结合：核心爽点必须保留，集数变化只改"到达爽点的路径"；
  改编后每10集画情绪曲线对比源本，峰值位置/强度一致 = 命脉未丢
```

### 5.3 创作工艺库（怎么把每场写好——见第六部分）
信息差锁 / 物件钩子 / 身体反应替代心理描写 / 围观反应链替代当面打脸 / 延迟激活链 / 视角欺骗 / 打脸三段式（铺垫40%+反击30%+释放30%）…

### 5.4 硬标准
- **镜头感**：情绪标签句 hard-fail（"心中涌起暖意"→改成可拍的动作/沉默）。
- **去AI味**：禁用词表（"不是A是B""眼中闪过一丝"…）+ 去非视听语言（删上帝视角/内心独白）。
- **改动可追溯**：记录"原第X集X场 → 保留/换壳成/改写成/删/合并成什么"，产出改动对照表。

## 第八部分 · 诚实交底：作者尚未逐字细读、Codex review 时应核实的内容

## 附 · 版本演进与已纠正的错误（防止 review 时重提旧坑）

| 版本 | 当时定位 | 被纠正的错误 |
|---|---|---|
| v1-v2 | "分析器/评分器/精洗器" | 产品定义散；把评判当成要先做准的评分器；分析维度堆砌 |
| v3 | "精洗器" | **错误声称 Dramatron 能逆向拆解+级联**（独立review抓出，已删）；缺输入校验/可追溯等环节 |
| v4 | "精洗器" | 定位仍偏"精洗执行层"，**未理解用户实际做的是换壳/集数改编**；分析/评分是旧定义残留 |
| **v5（本版）** | **会动脑的洗稿器** | 回到第一性原则（写好剧本）；明确三种洗稿模式；换壳=理解重演非替换；拆源本分宏观+微观两层；评分改自检；诚实交底未读内容 |

**核心认知（贯穿全程的教训）**：
- 这是洗稿器，源本是锚不脱缰；但换壳/改编要自洽、不机械、不堆砌。
- 一切服务"观众付费+想看"，但本质是先写好剧本，不是堆钩子爽点。
- 不要落入传统电影剧本逻辑（人物弧光/主题深度，短剧不要）。
