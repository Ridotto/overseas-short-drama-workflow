## 第一部分 · 产品是什么

### 1. 一句话
**会动脑的海外短剧洗稿器**：输入一个源剧本，按用户的换壳/集数改编需求，**保留它让观众"想付费、想追下去"的命脉，重写出一个自洽的新本子**。

### 2. 第一性原则（凌驾一切）
**本质是"写好剧本"，只是带一个参考前提（源本）。**
- 不是机械查表替换器 —— 换壳/改编要让新本**自洽、不生搬硬套**。
- 不是自由创作机器 —— **源本永远是锚，不脱缰**。
- 准确定性：**会动脑的洗稿器**。

### 3. 最高准则（两层，缺一不可）
```
① 本子本身是"好剧本"：自洽、逻辑站得住、不假、不堆砌
② 在此基础上，关键因素（钩子/爽点/卡点/情绪）写得好
→ 观众自然愿意付费、想追下去
```
**防本末倒置**：不能为"付费/想看"刻意堆砌钩子爽点——堆砌出来是 cliche 烂本。好剧本是地基，关键因素是加分，不能用加分替代地基。

### 4. 输入 / 输出
- 输入：1 个源剧本（.txt）+ 用户需求。用户给的本基本要洗（换壳或改编）。
- 输出：洗完的剧本（纯文本）。先产出首批集数，确认后续写。不输出 JSON / 不碰视频。

### 5. 用户
写本子的人（用户 / 编剧）。

### 6. 三种洗稿模式
| 模式 | 是什么 | 常见度 | 关键 |
|---|---|---|---|
| 换壳重写 | 保留命脉，换世界观/身份/场景 | 最常见 | 理解新壳逻辑→等效重演→保证自洽 |
| 集数改编 | 保留命脉，改集数(40→80/30)，重排节奏 | 常见 | 扩/缩写方法 + 节奏重排，命脉体验对齐 |
| 深度大改 | 180度换（极少） | 极少 | 接近重写，边界情况 |

## 第二部分 · 命脉怎么理解（关键修正，防机械对齐）

> ⚠️ v5 曾写"卡点位置/情绪值/钩子类型 1:1 不动"——这是机械对齐，错。v6 修正：

```
命脉锁的不是【数字和标签】，是【观众体验功能 + 情绪作用】。

错误（机械）：第3集卡点、情绪值-7、身份反转钩子 → 新本必须一模一样照搬
正确（等效重演）：
  这一拍在源本里的作用是"让观众为主角揪心 + 留下'她会怎么翻盘'的悬念"
  → 新壳里只要达到同样的观众体验/情绪作用即可，
    具体用什么卡点形式、什么钩子类型，由新壳逻辑决定
  → 若新壳逻辑不支持原 beat（军营没有"血统羞辱"），
    允许等效重演（改成"当众羞辱军衔"达到同样羞辱效果），
    不硬搬原形式
```

**判断命脉是否保住的标准 = "观众的体验和情绪走向是否复现"，不是"数字标签是否对齐"。**
（自检里画情绪曲线对比，看的也是"峰谷的体验作用是否对应"，不是数值必须相等。）

## 第三部分 · 产品怎么工作（workflow）

> 三特征：源本为锚 + 写自检分离（防自我批准）+ 改动可追溯。每环节"理解→判断→创作"，不机械。
> 节点中"★需用户确认"的，参考 shortdrama-pipeline 的产品流程：关键节点人工确认后才续。

```
输入：源剧本
 ↓
环节0 输入校验 ── 挡空稿/超长短/非剧本格式 → 产出：通过 or 拒绝(附原因)
 ↓
环节1 需求对齐 ── 问清需求(第四部分问卷) → 产出：【需求单】★需用户确认
 ↓
环节2 拆源本 ── 提取命脉 + 判断好坏(同一份产出)
              → 产出：【源本拆解报告】+【逐拍 beat 卡】
 ↓
环节2.5 结构保险丝 ── 扫骨架是否塌(默认放行) → 亮红灯则提示大改、交回人
 ↓
环节3 定洗稿方案 ── 定模式 + 锁 minimum 命脉 + 定换壳方向
              → 产出：【minimum 保留清单】+【洗稿方案】★需用户确认
 ↓
环节4 重写 ── 按方案洗(换壳等效重演 / 改编扩缩 + 节奏重排)
            全程：创作工艺 + 镜头感 + 去AI味 + 记录改动
            → 产出：【新本(首批集)】+【改动对照表】
 ↓
环节5 独立自检 ── 三级标准判定(第六部分)
            → 产出：【自检报告】
            hard fail → 必返工回环节4(自改有上限)
            warning → 标出建议修
            user decision → ★需用户判断取舍
 ↓
输出：洗完的本(首批集) + 改动对照表 + 自检报告
 ↓
续写 ── 用户认可首批后，分批续写(每5-10集)，每批用户轻审 ★需用户确认

【全程大脑】方法论+标准库(第七部分)  【全程记忆】拆解库(source of truth)
```

**用户决策点汇总（★）**：需求单确认 / 洗稿方案+命脉清单确认 / 自检 user decision 项 / 首批交付确认 / 每批续写确认。

## 第四部分 · 需求问卷（环节1）

**必答**：题材 / 风格(男频·女频·lgbt) / 目标平台+市场 / 集数(源本几集→洗成几集) / 卡点·付费点规格 / 换壳方向 / 改多狠(换壳·改编·大改) / 绝对不能动的 / 特别想要的。
**收集不强制**：情绪基调 / 角色数偏好 / 合规规避项。
**需求冲突时**（如"既要保留全部支线又要砍到30集"）：标为冲突，交用户决策，不擅自取舍（见待定项）。

## 第五部分 · 七个交付物（每个：给谁看 / 何时产出 / 字段 / 如何进下一环 / 什么算不合格）

> 这是 v6 的核心补强。交付物是 workflow 的血液，必须定清楚流转和达标。

### 5.1 需求单
- **给谁看**：用户确认 + 后续所有环节对照。
- **何时产出**：环节1。
- **字段**：题材/风格/平台市场/源本集数→目标集数/卡点付费点规格/换壳方向/改动力度/不能动的/特别想要的/(冲突标记)。
- **进下一环**：用户确认后，作为环节2-5 的对照基准。
- **不合格**：必答项缺失 / 存在未解决的需求冲突 / 用户未确认。

### 5.2 源本拆解报告（宏观）
- **给谁看**：内部(喂环节3-5) + 用户可选看(判断本值不值得洗)。
- **何时产出**：环节2。
- **字段**：故事核 / 结构划分 / 整本情感曲线(峰谷+体验作用) / 爆点分析 / 主反转 / 钩子体系 / 人物功能与关系 / 写作手法 / 题材赛道与共鸣点 / 好坏判断(6维 + 诚实结论：最重要那几点强不强)。
- **进下一环**：环节3 据此锁命脉、定方案。
- **不合格**：命脉(情感曲线/爆点/钩子)没识别出来 / 好坏判断回避结论 / 把源本烂点当优点。

### 5.3 逐拍 beat 卡（微观）
- **给谁看**：内部(环节4 重写施工单)。
- **何时产出**：环节2。
- **字段**：
  - 命脉(锁体验作用，不锁数字)：位置 / 这拍的**观众体验功能** / **情绪作用** / 在爆点链里的角色。
  - 壳(可换)：涉及角色 / 场景 / 原文白描 / 篇幅。
  - 可动性：核心命脉(等效重演时必须复现作用) or 过渡铺垫(改编时可增减)。
- **进下一环**：环节4 逐拍换壳/改编。
- **不合格**：把"形式数字"当命脉锁死(机械) / 没区分命脉与可换壳。

### 5.4 minimum 保留清单
- **给谁看**：用户确认 + 环节4·5 死守。
- **何时产出**：环节3(工具识别候选→用户增删→锁定)。
- **字段**：这次洗稿绝对不能丢的命脉项(每项写：是什么观众体验/情绪作用、为什么不能丢、对应源本哪些 beat)。
- **进下一环**：环节4 重写时守住、环节5 自检时核对。
- **不合格**：清单是"数字标签"而非"体验作用" / 用户未确认 / 与需求单矛盾。

### 5.5 洗稿方案
- **给谁看**：用户确认 + 环节4 执行。
- **何时产出**：环节3。
- **字段**：洗稿模式 / 换壳方向(新世界观设定) / 集数改编计划(扩缩哪些、节奏怎么重排) / 源本各部分处置(保留/修复/丢弃) / 首批交付几集。
- **进下一环**：环节4 据此重写。
- **不合格**：没说清"源本哪些保留/修复/丢弃" / 换壳方向新壳逻辑自相矛盾 / 与 minimum 清单冲突。

### 5.6 改动对照表
- **给谁看**：用户(建立信任，知道改了哪、为什么)。
- **何时产出**：环节4 边写边记。
- **字段**：原第X集X场/beat → 处置(保留/换壳成/改写成/删/合并/新增) → 为什么(对应需求或诊断)。
- **进下一环**：随新本一起交付 + 环节5 核对可追溯性。
- **不合格**：有改动追溯不到源本 beat 或用户需求 / 缺失大段改动记录。

### 5.7 自检报告
- **给谁看**：内部(决定返工) + 用户(看 user decision 项)。
- **何时产出**：环节5。
- **字段**：按三级列出每条(hard fail/warning/user decision) + 命脉对齐情况(情感曲线体验对比) + 改动可追溯性核对。
- **进下一环**：hard fail→返工；warning→建议修；user decision→交用户。全过→交付。
- **不合格**：自检本身漏检第六部分必查项 / 与重写同一手(没做到独立)。

## 第六部分 · 自检三级标准（由旧规则改造，非新评分器）

> **复用** v2 旧自检/规则体系的内容，**改造**成洗稿自检三级。不做大而全分数表，不机械套模板。

### 三级动作定义
- **hard fail**：必须返工，否则不能交付。
- **warning**：建议修，不自动卡死，标给用户/重写参考。
- **user decision**：需用户判断取舍。

### 必查项（来自 Codex review 第5条 + 旧规则改造）
| 检查项 | 级别 | 判定（来源） |
|---|---|---|
| 源本命脉有没有丢（情感曲线体验、核心爆点、minimum清单项） | **hard fail** | 旧"命门否决"改造：命脉丢=必返工 |
| 新壳里人物动机/行为/冲突站不站得住、假不假 | **hard fail** | "好剧本"底线，假=必返工 |
| 是不是查词典式换皮（生搬硬套、新壳逻辑不通） | **hard fail** | 第一性原则底线 |
| 钩子/爽点/卡点是否服务剧情、还是堆砌 | **hard fail** | 防本末倒置 |
| 台词/叙述有没有 AI 味/解释味/情绪标签句 | **hard fail**（情绪标签句）/ warning（轻度AI味） | 旧"去AI味/镜头感hard-fail"改造 |
| 是否继承了源本的烂点（源本本就弱的地方照搬了） | **warning** | 新增：洗稿不能连烂点一起洗过来 |
| 改动是否能追溯到源本 beat 或用户需求 | **warning** | 可追溯性 |
| 信息密度/节奏 是否有注水或拖沓 | **warning** | 旧"可量化质检"改造 |
| 够不够"吸睛"（是否有足够亮点、用户会不会想追） | **user decision** | 旧"亮点门槛"改造：交用户判断 |
| 换壳方向/某处取舍是否符合用户口味 | **user decision** | 主观偏好交用户 |

**原则**：hard fail 是客观底线(命脉丢/假/换皮/堆砌/AI味)；warning 是建议；主观的(够不够吸睛、合不合口味)交 user decision。**不合成总分、不排序。**
