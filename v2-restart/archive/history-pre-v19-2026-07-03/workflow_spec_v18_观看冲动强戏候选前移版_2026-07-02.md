> 本版修订依据：`生产链能力执行对账_2026-07-02.md`、`0xsline短剧标准库逐文件接入审计_2026-07-01.md`、`0xsline底盘接入架构审计_2026-07-01.md`、`样本验证/飞书三星样本/PRD_v31_clean_v17_0xsline_episode/12_V31_vs_V5_V30_商业平台稿_主控对比.md`  
> 验证方式：必须跑同一飞书三星样本首批 1-10 集，并与 V5 / V22 / V24 / V25 / V28 / V30 / V31 对比。  

本版相对 v17 只改一件事：

```text
把“观看冲动 / 爽感类型 / 强戏候选空间”从 S5 前移到 S3/S4。
```

v17 已证明：0xsline `/episode` 接到 S5/S6 后，writer 的写作动作会变强；但 V31 也证明，如果 S3/S4 给到 writer 的候选承载仍偏冷、偏机构、偏制度，S5 只能在冷材料上加热，难以超过 V5 和商业平台稿的本能短剧刺激。

v18 的根因判断：

```text
不是 writer 完全不会写，而是“强戏候选空间”出现太晚。
S3/S4 仍在优先交付故事功能和安全承载，尚未把观众观看冲动变成上游可继承的生产变量。
```

所以本版不加 reviewer 规则，不整套接入 0xsline，不把 V5 当目标复刻。  
本项目仍是洗稿工具，0xsline 只作为 writer 写作当下的短剧发动机；S3/S4 负责把源本有效性和新壳条件变成可供 writer 选择的强戏候选空间。

## 0. 执行需求卡

### 0.1 用户可见目标

把一个已被市场验证或值得参考的短剧源本，洗成一个新壳下仍然好看、能追、能付费的新剧本。

交付物是剧本文字，不是分析报告。

### 0.2 本 spec 解决什么

v18 只解决一件事：

```text
把 PRD、原项目能力、外部短剧手艺和历史样本里已确认有用的能力，
接成一条可执行、可交接、可回滚的写稿链。
```

它不是新 PRD，不是资料池，不是短剧规则大全。

### 0.3 明确不做什么

- 不从零原创。
- 不照搬 0xsline / short-drama 的原创工作台流程。
- 不把 V5 当目标样本复刻。
- 不把短剧感理解成钩子堆砌或强度硬加。
- 不把短剧手艺写成固定模板；本 spec 只要求关键创作动作必须发生，不规定每集使用同一桥段、同一比例或同一顺序。
- 不继续修 V25 正文。
- 不让 reviewer 生产短剧感。
- 不让 run log 证明剧本质量。
- 不创建真实 `.codex/skills/SKILL.md`。
- 不进入网页 App 或工程实现。

### 0.4 通过标准

v18 只有跑样本后才能正式化。静态通过不等于剧本通过。

总通过轴只有一个：

```text
观众看完当前集 / 当前批次后，是否更想继续看，是否愿意为下一段答案付费。
```

视听语言、短剧感、钩子、爽点、拉扯、反派压力、DNC、台词和连续性都不是孤立 checklist。它们都必须服务这个追看 / 付费效果。  
如果某项能力“有了”，但没有让观众更想追，仍然视为没有真正生效。

追看 / 付费效果不是钩子堆砌，而是主追问统住，压放、兑现和追问节奏成立。

本 spec 的生产约束不是“模板”。  
它只要求每个写作单元都能说清：观众在等什么、压力从哪里来、谁被迫行动、爽点如何压放、结尾留下什么未完成追问。  
具体桥段、动作、空间、台词和卡点形式必须由源本有效性、新壳设定、人物动机和当前剧情自然长出来。

样本通过必须同时满足：

- 没洗飞源本有效性；
- 不是改名复刻；
- 新壳下自己站得住；
- 观众有继续追的理由；
- 有付费前后的追问和兑现义务；
- 高压节点像戏，不像报告、会议、文件播报；
- 台词短、准、有功能；
- 动作、身体、空间、声音、物件进入正文；
- 钩子、爽点、拉扯、反派压力、DNC、连续性没有被短剧感修复挤掉。

## 1. 产品锚点与非目标

### 1.1 产品锚点

本项目是洗稿器，不是原创短剧生成器。

简写口径：洗稿，不是原创。

核心判断：

```text
新本没有洗飞源本有效性；
新本不是源本改名复刻；
新本作为一个新短剧自己站得住。
```

v18 的关键口径：

```text
源本有效性必须通过新事件载体和新短剧场面承载出来。
不能只迁移剧情功能，也不能只换名词。
```

### 1.2 三条红线

| 红线 | 含义 | 失败回滚 |
|---|---|---|
| 洗飞 | 源本让人追下去、愿意付费的有效性丢了 | S1/S2/S3 |
| 改名复刻 | 源本事件链和强桥段整串保留，只换身份、名词、地点 | S2/S3/S4 |
| 新本不站住 | 新壳不自然、人物动机弱、连续性乱、剧本不像短剧 | 对应失败层 |

### 1.3 非目标

- 不是评分器：不使用 50 分制或总分幻觉。
- 不是模板器：不把开场、爽点、付费点固定成硬比例。
- 不是强度堆叠器：不要求每场都爆、每句都狠、每个节点都反转。
- 不是千篇一律生产器：不能因为有生产约束，就把每集写成同一种开场、同一种打脸、同一种卡点。
- 不是 reviewer 驱动返修机：reviewer 只兜底和归层。
- 不是视频分镜工具：正文必须可看可听，但不写镜头工程。
- 不是用户要理解的多 agent 展示产品：用户只和主控 agent 对话。

## 2. 总执行链

v18 固定 S0-S11：

```text
S0 需求确认
-> S1 源本拆文
-> S2 洗稿边界
-> S3 新本故事架构
-> S0b 洗稿方案二次确认
-> S4 场面承载原料
-> S5 writer 批次 / 单集 / 场景组施工稿
-> S6 screenplay 正文
-> S7 台词 / 视听 polish
-> S8 continuity / 信息边界
-> S9 作者自检与返修诊断
-> S10 干净 reviewer
-> S11 主控拍板
```

唯一交接原则：

```text
A 环节产出 X；
B 环节只吃 X；
B 环节产出 Y；
C 环节只吃 Y。
```

下游不能回头翻上游厚材料。  
如果下游必须回头翻，说明上一层产物不合格，应回上一层重做。

长篇 / 长批次执行原则：

```text
S3 控制全剧、阶段、批次和分集功能；
S4 给当前批次或当前高压节点准备可写戏原料；
S5 只排当前批次 / 当前集 / 当前场景组的施工稿；
S6 只写当前施工稿对应的正文。
```

如果用户要 100 分钟、100 集或多批次交付，不能让 S5 一次性写完整本施工稿。  
必须先由 S3 产出全剧阶段骨架和批次地图，再按批次进入 S4-S8，并在每批结束后更新 story memory 和下一批 DNC。

## 3. 全链工位合同

### S0 需求确认

**只负责：** 拿到可开工边界。

**输入：**

- 用户需求；
- 源本入口；
- 用户已知的新壳、集数、语言、市场、改写力度和禁区。

**必须输出：** `最小需求确认`

必须回答：

- 源本是什么；
- 新壳是什么；
- 目标市场是什么；
- 输出语言 / 审稿语言是什么；
- 集数是否改变；
- 改写力度按哪些变量控制；
- 用户禁区是什么；
- 正文用途是什么：用户审稿、导演审稿、AI 视频生成、字幕或双语。

**禁止：**

- 分析源本；
- 设计剧情；
- 默认深度重构；
- 默认英文或中文，不确认输出需求。

**下游只能读：**

- S1 读取 `最小需求确认 + 源本`；
- S2 可读取需求边界。

**失败回滚：**

- 目标市场、语言、集数、用途不清，回 S0。

**S0b 洗稿方案二次确认：**

S0b 不单独编号为新工位，但必须发生在 S3 之后、S4 之前。

S0b 必须把以下内容给用户 / 主控确认：

- 源本最需要保住的追看 / 付费效果；
- 改写力度和变量边界；
- 新本故事核、题材、世界观和目标市场语感；
- 全剧阶段骨架和首批 / 当前批次范围；
- 明确不能提前消费的信息；
- 明确不能复用的源本高辨识外形；
- 输出语言和正文用途。

如果 S0b 没有确认，不能进入 S4。  
如果用户在 S0b 改方向，回 S0/S2/S3，不从 S4 开始硬拧。

---

### S1 源本拆文

**只负责：** 拆源本为什么能追、能付费。

**输入：**

- `最小需求确认`；
- 源本。

**必须输出：** `源本有效性包 / Source Bible`

必须拆出：

- 核心期待；
- 留存骨架；
- 钩子 / 卡点；
- 付费节点；
- 爽点和压抑释放；
- 源本强弱节奏：哪些地方该压、哪些地方该放、哪些地方需要呼吸；
- 情绪密度 / 信息密度 / 情节密度；
- 人物关系和拉扯；
- 信息差 / 真相揭露；
- 反派压力和反扑燃料；
- Do Not Consume；
- 水分 / 烂点；
- 源本高辨识外形。

**禁止：**

- 设计新壳；
- 写新剧情；
- 把剧情摘要当源本有效性；
- 把源本强桥段直接当可继承内容。

**下游只能读：**

- S2 读取源本体验功能、高价值节点、高辨识外形和 DNC。

**失败回滚：**

- 如果后续发现“观众追什么、为什么付费”没拆出来，回 S1。

---

### S2 洗稿边界

**只负责：** 区分必须继承的体验和必须替换的外形。

**输入：**

- `源本有效性包`。

**必须输出：** `洗稿边界包 / Boundary`

必须包含：

- 强继承体验；
- 等效迁移体验；
- 自由改写变量；
- 必须删除 / 修复；
- forbidden shape；
- 新壳不能承载的源本外形；
- 改写力度的变量说明。

**禁止：**

- 写具体新剧情；
- 为了防同构洗飞源本；
- 把“不能复刻外形”误读成“不能用短剧套路”。

**下游只能读：**

- S3 读取继承项、自由项、禁用外形和改写边界。

**失败回滚：**

- 源本强桥段整串回流，回 S2/S3/S4；
- 防同构压过防洗飞，回 S2。

---

### S3 新本故事架构

**只负责：** 搭新壳故事、真相、阶段、分集骨架、追看强度地图，并把高价值节点的观看冲动变成上游生产变量。

**输入：**

- `洗稿边界包`。

**必须输出：** `新本故事开发包`

必须包含：

- 新本故事核；
- 题材 / 世界观边界；
- 目标市场语感；
- 真相账本；
- 角色错信；
- 主追问和阶段追问；
- 全剧阶段骨架 / 粗纲；
- 批次地图：每批覆盖哪些集 / 分钟、承担什么追看功能、在哪个问题上停；
- 分集功能 / 单集提纲；
- 人物行动方式；
- DNC 排程；
- `追看强度地图`；
- `观看冲动总线`；
- `爽感类型 / 情绪债务`；
- `强戏候选边界`。

`追看强度地图` 必须把原本散落的强弱分配、付费追问、反派压力合成一张生产地图，而不是三张表。  
它必须覆盖当前交付范围，并回答：

- 哪些集 / 段落是波峰，哪些是呼吸段，哪些是铺垫段；
- 每个波峰兑现什么旧问题，同时留下什么新问题；
- 每个付费前后，观众为什么愿意继续等答案，下一段必须兑现什么；
- 当前主角被什么人、关系、资源、误判或信息差继续压住；
- 反派下一步还能靠什么资源、误判、筹码或关系继续作死；
- 哪些真相、身份、证据、关系变化不能在本批提前消费；
- 哪些地方必须克制，避免连续堆钩子导致疲劳。

注意：`追看强度地图` 不是固定公式。  
它不要求每集都有同等强度，也不要求每集都有反转。它只负责让强弱有节奏、兑现有债务、反派有燃料、观众有继续追的理由。

`观看冲动总线` 是 v18 的关键前移动作。

它必须把每个高价值节点从“剧情发生什么”改写成“观众为什么要继续看”，但不写具体桥段。  
每个高价值节点至少回答：

- 观众此刻最想看什么：反击、打脸、揭穿、后悔、救赎、复仇、关系撕裂、身份兑现、真相击穿，还是下一集答案；
- 本节点属于哪种爽感 / 情绪债务：先压后放、公开失位、关系拉扯、反派作死、火葬场、身份逆转、身体危险、当众羞辱、主角夺权、真相刺痛；
- 这个冲动不能被什么低强度实现替代：文件递交、屏幕播报、会议讨论、系统提示、纯解释对白、角色冷静承认；
- 最低强度地板是什么：至少要有一个现场动作、身体状态、空间压迫、物件变化、公开后果或关系位移；
- 哪些源本高辨识外形不能复用；
- 哪些信息不能提前消费。

注意：S3 只定义“观众冲动”和“强度地板”，不写完整场面。  
它不规定每集都要枪、车祸、下跪、耳光、爆炸或血。它只保证 S4/S5 不能把高价值节点降级成冷证据、冷流程或冷说明。

`强戏候选边界` 必须说明：本节点更适合从哪类承载里找戏，例如人际压迫、身体风险、公开失位、关系背叛、资源夺回、身份兑现、反派反扑、空间封锁、物件剥夺。  
它只给候选方向，不替 S4 做最终承载选择。

**禁止：**

- 写具体高压场完整外形；
- 锁死源本外形；
- 写正文级动作串；
- 把题材设定写成百科；
- 把短剧固定比例压过源本节奏。
- 把每集都设计成高压爆点，导致主线散、兑现不足或观众疲劳。
- 把 `追看强度地图` 写成“每集都要开场钩子+中段打脸+结尾反转”的固定模板。

**下游只能读：**

- S0b 读取用户需要确认的故事方向和批次范围；
- S4 读取当前批次 / 当前集需要的故事状态、分集功能、角色行动方式、`追看强度地图`、`观看冲动总线`、`强戏候选边界`、DNC 和新壳边界。

**失败回滚：**

- 主追问、真相账本、错信、阶段骨架、批次地图不清，回 S3；
- 人物动机与行为矛盾，回 S3/S5；
- `追看强度地图` 没有波峰 / 呼吸 / 兑现 / 追问 / 反派燃料，回 S3；
- 高价值节点只有剧情功能、没有具体观看冲动和强度地板，回 S3；
- 反派没有继续作死的资源或误判，回 S3，不先丢给 S5 临场补。

### S4 场面承载原料

**只负责：** 把当前批次 / 当前集 / 当前高压节点的故事功能和观看冲动，变成可供 writer 选择的强戏候选空间。

**输入：**

- `新本故事开发包`。

**必须输出：** `场面承载原料包`

必须包含：

- 当前批次或当前写作单元；
- 当前批次承接上一批什么状态，结束后必须留下什么追看问题；
- 这场承载源本哪个体验功能；
- 新壳里靠什么权力、关系、空间、物件或行动承载；
- 场面承载选择：至少给出 2 个可选承载方向，并说明最终选择哪一个、为什么它比其他选项更能承载源本体验；
- 强戏候选空间：高价值节点至少给出 3 个候选承载，并说明哪一个太冷、哪一个太像源本、哪一个最可能制造观看冲动；
- 谁压谁、谁旁观、谁被迫行动；
- 压力如何形成：靠人、关系、资源、空间、身体风险、公开羞辱、错信还是信息差；
- 证据 / 文件 / 屏幕 / 系统在场面里的地位：只能是触发器或辅助，不能是高压节点本体；
- 释放后造成什么可见后果；
- 禁止复用的源本外形；
- 本写作单元 DNC。

`场面承载选择` 的判断口径：

```text
删掉文件、屏幕、报告、系统提示之后，
这场还剩不剩人和人之间的现场压力、被迫行动和可见后果？
```

如果答案是否，这个承载方向不能进 S5。  
如果证据必须存在，它必须触发人物选择、身体动作、公开关系变化或权力后果，而不能自己替代戏。

这不是要求每场都变成打架、下跪、喊话或羞辱。  
低压过渡可以克制，但高价值节点不能只靠证据递交、会议讨论或系统播报完成。

v18 对 `强戏候选空间` 增加一个硬要求：

```text
高价值节点不能只交一个“安全可写方案”给 S5。
S4 必须把可选演法空间打开，让 writer 能在 S5 做真正的短剧选择。
```

高价值节点的候选承载至少包含：

1. `冷安全候选`：通常是文件、屏幕、会议、系统、报告、口供等。它可以存在，但必须被识别为偏冷，默认不能作为高压节点本体；
2. `新壳自然候选`：从新设定、职业、空间、身份、资源、人物关系里自然长出的承载；
3. `高冲动候选`：最能制造观众即时观看冲动、可见后果和下一段追问的承载。

选择口径按顺序判断：

- 是否保住源本体验功能；
- 是否制造 S3 指定的观看冲动；
- 是否有现场动作、身体状态、空间压迫、物件变化、公开后果或关系位移；
- 是否避开源本高辨识外形；
- 是否符合新壳自然性和人物动机；
- 是否可看可听，适合 AI 短剧成片。

如果 3 个候选都只能靠机构、系统、文件、会议或屏幕成立，S4 不能放行，必须回 S3/S4 重新找承载。  
如果为了刺激而破坏人物动机、新壳自然性或 DNC，也不能放行。

**禁止：**

- 排完整戏；
- 写完整节拍；
- 写台词；
- 写成片最后一帧；
- 用文件、屏幕、会议、系统替代人和人的现场冲突。
- 把候选方向全部写成同一种短剧套路，或者为了“更刺激”破坏新壳自然性。
- 只交一个最终承载给 S5，让 writer 没有强戏选择空间。

**下游只能读：**

- S5 只读 `场面承载原料包`。

**失败回滚：**

- 新壳承载弱、只有证据 / 文件 / 系统 / 会议，回 S4；
- 没有给出承载选择，只给一个默认方案，回 S4；
- 高价值节点没有给出强戏候选空间，或候选全部偏冷，回 S4/S3；
- 证据从辅助变成高压节点本体，回 S4；
- S4 写成半成品正文，回 S4 重做原料。

### S5 writer 批次 / 单集 / 场景组施工稿

**只负责：** 把 S4 原料排成当前批次、当前集或当前场景组的短剧戏。S5 是创作工位，不是填表工位。

v18 继承 v17 硬线：

```text
S5 必须按 0xsline /episode 的写作动作运行。
```

含义不是照搬 0xsline 题材模板，而是 writer 在写当前集 / 当前场景组时，必须加载对应短剧手艺，并完成一次“强戏演法选择”。

S5 的核心任务不是把剧情功能翻译成剧情，而是：

```text
把剧情功能变成最能制造观看冲动的短剧演法。
```

v18 增加一条上游保护：

```text
S5 不负责从零发明强戏候选空间。
S5 负责在 S4 给出的强戏候选空间里，选择、重组、发动成戏。
```

如果 S4 只给了一个冷安全方案，或者所有候选都偏机构、偏证据、偏系统，S5 不能硬写正文，必须打回 S4/S3。

**输入：**

- `场面承载原料包`。

S5 不直接读取：

- 源本拆文包；
- 洗稿边界包；
- 新本故事开发包；
- story memory 全文；
- 外部标准库全文；
- reviewer report；
- 作者自检；
- run log；
- 旧版对比；
- 临时诊断。

**必须输出：** `writer 施工稿 / 写作卡`

写作单元选择：

- 首批 1-10 集验证：通常按集写施工稿，高压节点可细到场景组；
- 100 分钟 / 多批次：按批次进入 S4-S8，每批内部再按集或场景组拆；
- 低压过渡：可以用场景组施工稿，不强迫每场都写成高压戏；
- 绝对波峰、付费点、身份揭露、火葬场、重大反转：必须细到具体场景组。

`writer 施工稿 / 写作卡` 必须读起来像戏的粗稿，不像任务单。  
它必须把当前写作单元排成一串可以写正文的行动 / 反应 / 后果，而不是只写“本集要揭露、要打脸、要卡点”。

v18 沿用 v17 的四段结构，`writer 施工稿 / 写作卡` 至少包含四段，四段都必须是当前单元的具体内容，不允许写方法论：

1. `本集观看冲动`：观众这一集为什么想继续看。必须写成一句具体追问或情绪冲动，例如“想看她如何当众夺回位置”，不能写“提升短剧感”。
2. `单集微结构`：按开场 / 中段升压 / 爽点压放 / 结尾卡点排成连续戏，不写成字段表。
3. `强戏演法选择`：高价值节点必须基于 S4 的强戏候选空间做选择或重组，比较哪一个更能制造观看冲动、哪一个太冷、哪一个太像源本，最后选定一个。
4. `正文写作卡`：把选定演法写成可直接进入 S6 的行动链、情绪链、信息边界和最后一拍。

`强戏演法选择` 是 v17 的关键新增动作。  
v18 的变化是：这个动作不再完全压给 writer 临场救火。S3 先定义观看冲动，S4 先给强戏候选空间，S5 再做创作选择。

示例口径：

```text
功能：男主发现自己害死孩子。

弱演法：看报告、听录音、屏幕播报。
可用演法：他被迫亲手打开旧急救门，里面留下当年未送出的婴儿手环。
更强演法：他当众读出自己签过的拒救命令，女主不解释，只让他把那枚手环放到董事席上。

选择：选更强演法，因为它让男主主动承认、当众失位，并把证据变成身体动作和公开后果。
```

这不是要求每集都用手环、董事席、下跪或枪响。  
它要求 writer 在当前剧情条件下主动比较演法，而不是默认采用最安全、最像报告的实现。

如果 S4 的候选空间没有提供“可用演法 / 更强演法”的材料，S5 必须记录为上游失败，不得自己补成孤立桥段。  
S5 可以重组 S4 候选，但不能跳出 S2/S3/S4 的洗稿边界、DNC、新壳自然性和人物动机。

以下问题必须能从施工稿里看出来，但不要求按编号逐项填空。  
推荐用连续的“戏的粗稿”写，必要时再附 2-3 条边界提醒。

必须能看出：

1. 当前写作单元观众正在等什么；
2. 开场第一个可见动作 / 声音 / 冲突是什么；
3. 中段如何升压；
4. 爽点前如何压低；
5. 谁做动作，谁被迫反应，谁失去什么，谁看见了；
6. 关系 / 权力 / 信息如何变化；
7. 结尾停在哪个未完成动作、声音或画面，下一集 / 下一段必须兑现什么；
8. 哪些信息不能提前消费；
9. 禁止复用的源本外形；
10. 当前写作单元短剧手艺如何使用。

高价值节点必须写成“行动链”，但不规定固定拍数。  
行动链至少要能看出：

- 压力从哪里进入现场；
- 人物如何被迫做选择或露出状态；
- 爽点释放前，主角或观众期待被压低到哪里；
- 释放后谁失去脸面、位置、资源、关系或主动权；
- 下一段还欠观众什么答案。

低压过渡不用硬凑同等强度，但仍要服务主追问、人物状态或下一场压力，不能变成低信息聊天。

**短剧手艺必须在写稿当下使用：**

v18 继续使用 0xsline `/episode` 的阶段加载方式。S5 不读完整 0xsline 项目，只在写当前单元时加载和当前动作有关的参考：

| 0xsline reference | v18 接入方式 | 注意 |
|---|---|---|
| `opening-rules.md` | 用来写当前集开场第一拍，尤其第 1 集和每批开头 | 不是固定开场模板 |
| `rhythm-curve.md` | 用来排当前集微结构：开场抓人、中段升压、结尾释放或卡断 | 不照搬固定比例 |
| `hook-design.md` | 用来确定本集钩子类型和未回答问题 | 下一集必须有承接义务 |
| `satisfaction-matrix.md` | 用来决定爽点类型、压低动作、释放动作、可见后果 | 不能硬堆爽点 |
| `paywall-design.md` | 只在 S3/S4 标记为付费 / 强留存节点时使用 | 不照搬付费集数比例 |
| `villain-design.md` | 只在当前单元需要反派施压或反扑时使用 | 不从零发明反派设定 |
| `genre-guide.md` | 用于目标市场、题材语感、台词气质 | 不改变用户已定新壳 |

| 外部能力 | 在 S5 里的生产动作 |
|---|---|
| `opening-rules` | 写出开场第一拍，不写“开场要强” |
| `rhythm-curve` | 写出当前写作单元开场、中段、末尾压放 |
| `hook-design` | 写出未回答问题、卡住动作、下一集兑现义务 |
| `satisfaction-matrix` | 写出压低动作、释放动作、可见后果 |
| `paywall-design` | 写出付费追问和兑现义务，不照搬固定比例 |
| `villain-design` | 写出反派资源、误判和下一步反扑 |
| `genre-guide` | 写出题材语感和角色说话方式 |

这些外部能力只作为写作时的手艺，不作为固定桥段模板。  
S5 不能为了“用了 hook-design / satisfaction-matrix”而硬塞反转、硬塞羞辱或硬塞金句；如果某项手艺与当前剧情冲突，必须说明本单元如何用更自然的方式达到同一追看效果。

S5 对 0xsline 的使用边界：

```text
可以直接用：/episode 的写作动作、阶段加载 reference、单集微结构、钩子/爽点/节奏/台词的写作手艺。
不能直接用：原创题材生成、固定比例、固定爽点密度、总分 review、合规优先的创作决策。
```

**禁止：**

- 写流程证明；
- 复读上游字段；
- 写成检查表；
- 写完整正文；
- 照抄源本高辨识外形；
- 把外部短剧手艺直接变成固定套路；
- 为了制造钩子堆叠无兑现的新问题；
- 让 S6 正文 writer 自己回头悟材料。

**下游只能读：**

- S6 只读 `writer 施工稿 / 写作卡`。

**失败回滚：**

- 施工稿像任务单，没有行动链和压放，回 S5；
- S1/S2/S3 有的核心能力没进入 S5，回对应层或 S5；
- S4 没有给强戏候选空间，导致 S5 只能在冷承载上加热，回 S4/S3；
- S5 只是列出“开场 / 爽点 / 卡点”标签，没有把它们排成戏，回 S5；
- S5 为了追求刺激破坏新壳自然性或人物动机，回 S3/S5；
- S6 还需要回头读 S1-S4，说明 S5 失败，回 S5。

### S6 screenplay 正文

**只负责：** 把施工稿写成可看可听的剧本正文。

**硬线：writer 只读 S5 施工稿。**

v18 继承 v17 硬线：

```text
S6 只能执行 S5 已选定的强戏演法。
```

S6 不是重新选剧情，不是把 S5 的内容概括成对白，也不是把 S5 的动作链改回证据链。

如果 S5 已经选定“当众失位 + 物件触发 + 未完成追问”的演法，S6 必须把它写成现场动作、人物反应、空间变化、台词攻防和结尾卡点。
不能写成：

```text
某文件显示真相；
众人震惊；
角色解释；
本集结束。
```

**输入：**

- `writer 施工稿 / 写作卡`。

**必须输出：** `screenplay 正文初稿`

必须做到：

- 剧本页上的内容基本就是最终 AI 成片能看见、听见的东西；
- S5 的 `本集观看冲动` 在正文里能被读者直接感受到；
- S5 的 `强戏演法选择` 没有被正文写回安全、冷、说明性的版本；
- 人物通过动作、身体、空间、声音、物件和当众后果表达情绪；
- 台词短、准、有功能；
- 每集 / 每个交付段落结尾卡在答案前一拍；
- 高压节点像戏，不像报告、会议或证据播报。

**禁止：**

- 内部标签；
- 规则名；
- 工位名；
- 作者解释；
- 抽象心理替代画面；
- 站桩聊天；
- 多轮低信息对白；
- 回头翻 S1-S4。

**下游只能读：**

- S7 只读 `screenplay 正文初稿`。

**失败回滚：**

- 正文像说明书、报告、站桩聊天、不可视听，回 S6/S7；
- 正文完成了剧情功能，但没有制造 S5 指定的观看冲动，回 S5；
- 正文把 S5 选定的强戏演法写弱、写冷、写成解释，回 S6；如果 S5 本身选择弱，回 S5；
- 如果问题来自施工稿无戏，回 S5。

### S7 台词 / 视听 polish

**只负责：** 修表达，不重设主结构。

**输入：**

- `screenplay 正文初稿`。

**必须输出：** `台词视听 polish 稿`

必须处理：

- 删水词；
- 删解释腔；
- 强化角色声线；
- 把能用动作、声音、身体、空间表达的内容从对白里拿出来；
- 确保台词承担攻击、防守、逼迫、拒绝、试探、威胁、揭露、反击或选择；
- 确保高压场面有可见状态变化。

**禁止：**

- 重设主线；
- 替 S4/S5 重新造场面；
- 为了金句硬塞不合角色的台词；
- 把 polish 写成二次创作大改。

**下游只能读：**

- S8/S9 读取 `台词视听 polish 稿`。

**失败回滚：**

- 台词水、同声、解释腔，回 S7；
- 如果表达问题源于场面无动作，回 S5/S6。

### S8 continuity / 信息边界

**只负责：** 查事实、错信、DNC、伏笔、状态。

**输入：**

- `台词视听 polish 稿`。

**必须输出：** `story memory checkpoint`

必须检查：

- 真相账本；
- 角色错信；
- 已播 / 未播；
- DNC；
- 伏笔；
- 关系 / 权力状态；
- 同一信息是否重复当主刀；
- 后续火葬场、反转、关系击穿是否被提前消费。
- 当前批次结束后，下一批开场还能接什么追问、关系状态和反派压力。

**禁止：**

- 审美评价；
- 短剧感打分；
- 逐句改正文。

**下游只能读：**

- S9/S11 读取状态和风险。

**失败回滚：**

- 真相矛盾、DNC 失效、重复信息主刀、状态漂移，回 S3/S8；
- 批次交接后下一批无追问、无反派压力或状态断裂，回 S3/S8；
- 如果 S8 记录问题但放行，必须说明为什么它不是信息边界问题。

### S9 作者自检与返修诊断

**只负责：** 主创侧先判明显失败和回滚层。

**输入：**

- `台词视听 polish 稿`；
- `story memory checkpoint`。

**必须输出：** `作者自检与返修诊断`

必须判断：

- 是否洗飞；
- 是否复刻；
- 是否短剧降级；
- 是否正文长剧化；
- 是否状态漂移；
- 是否提前消费；
- 失败应回哪层；
- 返修时必须保留什么；
- 返修时禁止破坏什么。

**禁止：**

- 把硬风险写成“看主控偏好”；
- 把问题丢给 reviewer 兜底；
- 直接给正文逐句补丁；
- 为每个“还能更好”开返修。

**下游只能读：**

- S10/S11 读取结论、最终稿和必要边界。

**失败回滚：**

| 问题 | 回滚层 |
|---|---|
| 源本体验丢失 | S1/S2 |
| 高辨识外形回流 | S2/S3/S4 |
| 场面承载不住 | S4 |
| 施工稿像任务单 | S5 |
| 正文像说明书 | S6/S7 |
| 真相 / 信息乱 | S3/S8 |
| 台词水 | S7 |

### S10 干净 reviewer

**只负责：** 独立兜底，查红线和明显漏损。

**硬线：reviewer 不生产短剧感。**

**输入：**

- 用户需求摘要；
- 洗稿边界摘要；
- 最终稿；
- 必要源本片段。

第一轮 reviewer 不读：

- 作者自检；
- 主创解释；
- 旧版对比；
- run log；
- 临时诊断。

**必须输出：** `reviewer report`

必须包含：

- 致命阻断；
- 应改风险；
- 可选优化；
- 问题所属层级；
- suggestion。

**Reviewer 只查：**

- 洗飞；
- 改名复刻；
- 提前消费；
- 真相矛盾；
- 明显强度漏损；
- 正文长剧化；
- 状态漂移；
- 触碰红线。

**禁止：**

- 生产短剧感；
- 逐句代写；
- 无限挑刺；
- 因“还能更好”打回；
- 把个人口味升成致命阻断。

**下游只能读：**

- S11 读取 issue、严重度、回滚层和 suggestion。

**失败回滚：**

- reviewer 越权生产短剧感或无限挑刺，回 S10/S11 修 reviewer 口径。

### S11 主控拍板

**只负责：** 对比、归因、反馈回流、交付判断。

**输入：**

- 最终稿；
- 作者自检；
- reviewer report；
- run log；
- V5 / V22 / V24 / V25 / V28 / V30 / V31 / 上一版对比材料；
- 用户 / 导演反馈。

**必须输出：** `主控版本对比与拍板`

必须判断：

- 是否可交付；
- 为什么；
- 比 V22 / V24 / V25 / V28 / V30 / V31 是否明显提升；
- 是否至少保住 V5 的追看强度；
- 失败应回哪一层；
- 用户 / 导演反馈属于哪一层。

**禁止：**

- 只看 run log；
- 只看 reviewer；
- 只和烂版比；
- 把每条反馈变成新规则；
- 让 reviewer 对用户拍板。

**下游只能读：**

- 用户只看交付结论和必要依据。

**失败回滚：**

- 主控无法归因，回 S11；
- 如果问题属于上游，回对应层，不重开全链，除非触碰产品红线。

## 6. Reviewer 降权协议

Reviewer 是兜底，不是发动机。

### 6.1 Reviewer 可以打回

只有以下问题可以触发 `revise`：

- 伤到三条产品红线；
- 核心真相或主要人物动机自相矛盾；
- 明显洗飞；
- 明显改名复刻；
- 明显短剧降级；
- 明显提前消费；
- 明显状态漂移；
- 高压节点完全没有现场戏。

### 6.2 Reviewer 不能打回

以下只能列为风险或可选优化：

- 某句台词还能更狠；
- 某个桥段可以更刺激；
- 某个套路比较常见但没有复刻具体源本外形；
- 某个节点不如 V5 有标志物；
- 个人觉得还可以更短剧。

### 6.3 Reviewer 输出格式

每个 issue 必须写：

```text
问题：
严重度：致命阻断 / 应改风险 / 可选优化
失败层：
证据：
建议回滚：
不能破坏：
```

没有失败层和建议回滚的 issue，不允许主控直接转给 writer。

## 7. Run log 规则

run log 是执行记录，不是质量证明。

### 7.1 run log 必须记录

- 本轮使用哪个 workflow spec；
- 每层产物路径；
- 每层交给下游什么；
- 是否出现下游回头翻上游；
- writer 是否读了不该读的材料；
- reviewer 是否读了作者自检；
- 哪些步骤只是 execution complete；
- 最终创作判断是什么。

### 7.2 run log 禁止

- 证明剧本质量；
- 给 writer 当输入；
- 混入正文；
- 替代主控横向对比；
- 因为步骤齐全就判 pass。

run log 顶部必须写：

```text
run log 完成不等于剧本可交付。
工位 done 只代表该工位有产物，不代表剧本质量通过。
```

run log 还必须记录本轮输出范围：

- 是首批 1-10 集；
- 还是 100 分钟 / 多批次中的第几批；
- S0b 是否确认过洗稿方案；
- 当前批次承接上一批什么状态，并给下一批留下什么追问。

## 8. 样本验证协议

v18 写完后，必须先跑同一飞书三星样本首批 1-10 集。

首批验证通过只证明首批链路成立。  
如果用户目标是 100 分钟 / 多批次交付，还必须继续做一次批次交接验证：检查 S3 批次地图、S8 checkpoint 和下一批 S4/S5 是否能接住追问、状态、DNC 和反派压力。

### 8.1 对比对象

- V5：场面发动机和追看强度正向证据；
- V22：较稳但偏平；
- V24：链路完整但 writer 渲染感明显；
- V25：skill 工位链方向有效但未达标；
- V28：深度重构距离提升但强度轴未稳定提升；
- V30：v16 小修 clean 样本，场面链比 V28 强但仍偏机构；
- V31：v17 `/episode` 接入 clean 样本，writer 变强但 S3/S4 候选空间偏冷；
- 本版新样本。

### 8.2 验收问题

必须回答：

1. 是否没有洗飞源本追看功能；
2. 是否没有改名复刻；
3. 是否比 V25 更少旧外形回流；
4. 是否比 V25 更稳定地产生动作后果链；
5. 是否比 V28 更稳定地产生中后段强度，而不是只拉开外形距离；
6. 是否至少保住 V5 的场面发动机优势；
7. 0xsline 的开场、节奏、钩子、爽点是否真的进入 S5/S6；
8. S3 是否把高价值节点的观看冲动、爽感债务和强度地板定义清楚；
9. S4 是否给出强戏候选空间，而不是只交一个安全承载；
10. S8 是否拦住重复信息消费；
11. S9 是否在主创侧打回硬风险；
12. reviewer 是否仍然降权；
13. run log 是否没有被误读成质量通过；
14. 主线期待、人物关系、拉扯、反派压力、信息差、情绪密度和 story memory 是否保住；
15. 是否没有为了短剧感牺牲新壳自然性、源本有效性或海外市场语感。

### 8.3 判定

只有三种判定：

| 判定 | 含义 |
|---|---|
| `workflow candidate pass` | 样本产物显著强于 V22/V24/V25/V28/V30/V31，至少保住 V5 追看强度，可考虑升为当前默认 workflow |
| `workflow execution complete / creative revise` | 流程跑完，但产物仍不达标；必须根据失败层返修 workflow |
| `workflow invalid` | 流程本身使 writer 变差、丢能力或复发旧问题，不能继续小修 |

## 9. 静态对齐检查

在跑样本前，主控必须先检查：

- PRD 成功标准是否都能追到 S0-S11；
- 追看 / 付费总效果是否是所有小能力的共同验收轴；
- 每个历史失败类型是否有前置生产动作；
- S0b 二次确认是否存在，且发生在 S3 之后、S4 之前；
- 长篇 / 多批次是否有 S3 全剧阶段骨架、批次地图和 S8 批次 checkpoint；
- S3 是否产出 `追看强度地图`，而不是只写阶段功能；
- S3 是否产出 `观看冲动总线` 和 `强戏候选边界`，而不是只写剧情功能；
- S4 是否做了 `场面承载选择`，并确认证据 / 文件 / 屏幕只是辅助；
- S4 是否做了 `强戏候选空间`，并明确冷安全候选不能默认成为高压节点本体；
- writer 是否只读 S5 施工稿；
- S5 是否把源本有效性、钩子、爽点、拉扯、反派压力和 DNC 排成行动链，而不是任务单；
- S6/S7 是否负责 page-to-screen、台词功能和视听表达；
- S8 是否只查连续性和信息边界；
- S9 是否承担返修诊断；
- S10 是否保持降权；
- S11 是否做横向对比和用户反馈归层；
- run log 是否不进入 writer 输入。

如果以上任一项无法追到具体层级，不能跑样本。
