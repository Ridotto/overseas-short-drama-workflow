## 1. 产品定义（已定）

**一句话**：输入一部旧爆款短剧 + 偏好，**诊断它的机制 → 重构出更强的新剧本**。最终目标：产出**海外真人 AI 短剧爆款剧本**。

**最终输出 = 剧本（纯文本，给人读 / 给用户的下游环节对接）。**
- ❌ 不输出 JSON、不输出镜头级结构化、不碰视频生产（用户下游已有环节对接）。
- ❌ 不是评分器、不是一键成片、不是给甲方的分析报告。
- ✅ 是"机制翻译器 + 重构器"。难点在**品味**（判断想不想看）和**工艺**（怎么写出想看的）。

## 2. 用户与场景（已定）

- 第一阶段唯一用户：**写本子的人**（用户 / 编剧）。
- 不服务导演、监制、甲方。下游消费者（AI 视频环节）由用户自己对接，本工具不伺候。

## 4. ★分析机制（已夯实 · 7信源+2实验+6项目交叉验证）

### 4.1 设计原则
1. 维度要少（多了塌成 medium）。2. 只抽取+标记，不下总分。3. 区分质量维度 / 事实约束维度。

### 4.2 质量维度（6维 + 权重）

| 梯队 | 维度 | 权重 | 抽什么 |
|---|---|---:|---|
| 命门 | ① 情绪节拍 | 24% | 虐/爽/甜/燃曲线、情绪切换、情绪空窗。**量化：逐集情绪强度 −9~+9 曲线，标极值位置%**（借 oh-story） |
| 命门 | ② 钩子/卡点 | 22% | 前3秒/30秒抓人、每集集尾留扣、钩子密度。**按第13节钩子5类×5子模式打标** |
| 命门 | ③ 反转/爽点兑现 | 19% | 反转密度+意外度（=cliche检测）、爽点闭环（埋→兑现→改局）。**按第13节爽点矩阵打标** |
| 地基 | ④ 结构/期待线 | 14% | 先立核心期待、节拍骨架、转折点、主副线。**按第13节节奏四段配比核对** |
| 地基 | ⑤ 人物欲望/驱动力 | 11% | 主角想要什么、是否强烈清晰贯穿 |
| 薄厚 | ⑥ 信息密度/无废戏 | 10% | 每场新信息量、有无空转/注水 |

> 负向项：人物深度/弧光（过度则扣分）。门槛项：台词功能。
> 事实层（代码做）：集数/场景/角色/中英配对/卡点位置/人物关系网。

### 4.3 ★分层诊断（新增 · 借 how-to-make-script 的 rewrite_report）

**这是 v1 缺的核心环节，治"症状≠病因"（我重构 Archmage 失败的根因）。**

诊断不许跳层，必须先定位问题在哪一层：
```
概念病(concept) → 结构病(structure) → 场景病(scene) → 对白病(dialogue)
```
每个问题分三级：`must-restructure 必须重构` / `suggested 建议` / `observable-risk 风险`。

规则：
- **症状≠病因**："主角不讨喜"可能真正是"目标不清/世界压力不够/开场framing错"。聚类反馈→反推底层病灶。
- **定位到"第几集第几场"**，不笼统（借商业产品 Greenlight 页码批注 / 海马轻帆场次定位）。
- 硬失败 vs 加权弱项分离（hard fail 不能被一堆小风格建议淹没）。

### 4.4 cliche 清单（贯穿产出）
标出"哪几处观众能猜到走向"，喂给写剧本机制当"不许这么写"清单。

## 5. ★写剧本机制（框架定 + 工艺已补全）

### 5.1 性质
- **重构器**（非凭空原创）：输入旧本+preference，输出重构版纯剧本。
- 默认 L2 起改，可上调 L3。受评分约束：写完过评分，不达标带诊断意见自改，达标才交。

### 5.2 ★重构管线（借 Dramatron 逆向抽取再正向重生）
```
旧剧本 → 标记解析逆向抽成：logline / 人物表 / beat表 / 地点表
       → 按新设定（preference + 诊断结论）逐层重生
       → beat表做主线锚，每场对白注入"全局摘要+上一场beat"保连贯
       → 选择性上下文注入（每场只塞出场角色+当前地点），省token防串味
改进 Dramatron 短板：加"下层改动回写上层"的校对环 + 每层人审，避免它自承的"套路化"。
```

### 5.3 ★创作工艺库（借 oh-story 11手法 + 调研创作手册）
知道"目标"（评判维度）≠ 知道"怎么做到"。可直接用的手法菜单：
- **信息差锁**：POV锁死一方，另一方情感只用"物件+身体反应"漏给观众（控制2-3处）。
- **物件钩子**：一个有来历的小物件一次性引爆，胜过十句心理描写。
- **身体反应替代心理描写**：拳关节发白/双腿发颤，情绪外化（也是去AI味）。
- **围观反应链替代当面打脸**：弹幕/评论从嘲讽到泪目（短视频时代尤契合）。
- **延迟激活链**：同一道具跨集三次功能跃迁（喜剧道具→伏笔→金手指）。
- **视角欺骗**：用主角主观误判带观众一起低估，反转更炸。
- 钩子设计 / 反转铺设→回收 / 打脸三段式（铺垫40%+反击30%+释放30%）→ 见第13节。

### 5.4 ★镜头感硬标准（借 how-to-make-script 情绪标签句 hard-fail）
- **情绪标签句 hard-fail**："感到一阵悲伤""心中涌起暖意"必须改成摄影机能拍的动作/沉默。
- 一段对白命中≥3条毛病（四字词堆砌/解释性过渡/对仗对白/语域错位）→ 判 fail 必须重写。
- 把"有没有镜头感"从主观评价变成机器可判定的硬门槛。

### 5.5 ★去AI味标准（借 oh-story 禁用词表+7Gate）
- 禁用词（分级）：★★★★★"不是A，是B"；★★★★"眼中闪过一丝""嘴角勾起一抹""心头一震""缓缓""仿佛"。
- 7 Gate：禁用词/句式/心理告知/节奏均匀/对话腔/结尾升华/解释腔。
- **去非视听语言**：删上帝视角、内心独白（借海马轻帆"小说转剧本"NLP）。←我重构塞OS独白是反面教材。
- 删除比例上限：轻≤15% / 中≤25% / 重≤35%，不得整段删、不得删伏笔。

## 6. ★评分机制（框架定 · 打分法待专题敲定）

### 6.1 已达成的核心原则
1. **命门否决 + 强项决胜**，不是全维达标（4信源一致：现实就是短板砍+长板选）。
2. 维度=诊断工具，不是达标清单（防臃肿）。
3. **极致 > 全面**。
4. 给分，但用"对照锚点本"的相对分，不凭空绝对分（凭空会塌 medium）。

### 6.2 结构框架
```
阶段1 命门否决（3条全局+客观+致命硬伤）：开篇钩子缺失/情绪曲线平坦/主线结构崩塌
       + 可量化工程门（借 shortdrama-pipeline，用计数器零成本）：
         单镜头≤1句对白/≤3人；shot时长/集时长偏差等（适配剧本层）
阶段2 双层诊断（标记，不合成单一总分）：
       A层故事质量(6维+cliche)  B层生产质量(镜头感/可拍性)
       亮点单列、问题单列（带 suggestion 改法，借 shortdrama-pipeline）
阶段3 亮点门槛（用户规则·硬性）：至少2个亮点，否则"不够吸睛"打回
阶段4 对照排序：单本→和同类锚点本比 / 多本→A/B双向pairwise
       颗粒度前重后轻：第1集>前3集>卡一集
```

### 6.3 ⏳ 待专题敲定：更好的打分法（已找到方向，未拍板）
调研发现比"锚点+pairwise"更好的组合（EQ-Bench式三段混合）：
- **Boolean清单逐集评**（CheckEval）：不问"想不想看"，问"前3句有钩子吗""每集尾留悬念吗""主角有即时目标吗"→ 主观变客观、单样本可用、治 medium。
- **期望-惊奇建模**（StoryExpectations，Wattpad 30k章验证）：量化"意料之外又情理之中"，对"continue-to-read"解释力+31% → **爆款潜质代理指标**。
- rubric绝对分（概率加权）→ 播种Elo → 邻近锚点稀疏pairwise → Glicko-2聚合。
- 单样本最佳实践：锚点比+概率加权 / G-Eval多次采样 / Boolean清单 / 逐集过程评分。

> ⚠️ "什么算一个亮点"也待定。调研给的可落地判定（满足任一记1亮点）：①强反转 ②爽点峰值 ③可截图/转发的名场面 ④某执行维度明显强于锚点 ⑤强人设/强钩子。

## 7. ★连续性管理（新增环节 · 借 oh-story，可直接抄模板）

**v1 完全没有这一环。多集短剧最容易在伏笔断线/角色状态/时序上翻车。**

文件驱动，4 个追踪文件（把 oh-story 的"章"改成"集"）：

```
伏笔.md      | ID | 伏笔内容 | 埋设集 | 预计回收集 | 状态 | 重要度 |
角色状态.md   每角色：当前身份/能力/关系/公众形象 + 状态变更记录(逐集追加)
时间线.md    | 集 | 故事时间/事件 |（处理"同时/次日/几小时内"相对时间）
上下文.md    断点恢复：最后完成集/进度/活跃伏笔数/感情线/下一集要点
```

配套自动化（建议移植）：
- **缺分集大纲禁止写台词**（guard hook）。
- 开会话即检测设定缺口/伏笔断线。
- consistency-checker：跨集事实冲突扫描 + 分级报告。

      单线串行，第一版不上并行多agent。

## 9. 第一阶段最小验证闭环（已定）

- 取 1 弱本（Archmage，2星）→ 分层诊断前3集 → 重构 → 盲测原版 vs 重构版。
- 标准：盲读者不知A/B身份，只答"更想追哪个"。重构版赢=闭环成立。
- 锚点本（已拆）：重生血族女继承人（4星强本）= 重生复仇题材金标准。
- **第一次盲测已失败**（重构版输："非常cliche、信息少、无镜头感"）→ 催生了本版所有补强。**下一次重构必须带上：分层诊断 + 创作工艺库 + 去AI味 + 镜头感hard-fail。**

## 13. 短剧标准库（已提取自 short-drama + 调研，可直接查用）

### 钩子 5类 × 5子模式
| 类型 | 子模式 | 叠加规则 |
|---|---|---|
| 悬念钩 | 身份/结果/选择/来者/发现 | 可叠：悬念+情绪、反转+危机、信息+悬念、危机+情绪 |
| 反转钩 | 身份/局势/关系/时间/动机 | 禁叠：反转+反转、信息+信息、3种以上 |
| 情绪钩 | 甜蜜中断/心碎中断/误会引爆/决绝中断/重逢中断 | 频率：反转钩全剧≤5-7次 |
| 信息钩 | 证据揭露/秘密泄露/记忆闪回/文件发现/线索串联 | 连续3集不用同类钩 |
| 危机钩 | 突袭/背刺/暴露/倒计时/绝境 | 卡点集=最强钩+最高情绪 |

### 爽点矩阵 5类
身份碾压(军衔/证件/财富/能力/关系) · 打脸复仇(原话奉还/证据反杀/局中局/群众审判/实力碾压) · 逆袭翻盘 · 情感爆发(告白/真相/误会冰释/重逢/牺牲揭露) · 悬念揭秘。
- 每类5档★强度；同类不连续、情感与力量交替、大爽点前小铺垫、全剧≥3种。
- 主副辅配比 60%/30%/10%。打脸三段式：铺垫40%+反击30%+释放30%。

### 节奏四段配比（以60集为例）
起势15%(1-9,★★每2-3集) → 攀升30%(10-27,★★★每5-7集) → 风暴35%(28-48,★★★★每3-5集) → 决战20%(49-60,★★★★★几乎每集)。
- 单集微结构：前30秒钩子段 → 中1-1.5分钟冲突升级(禁信息真空) → 末30秒爽点/钩子段。
- 致命：连续3集无推进/高潮间隔>8集/最强高潮不在末段。

### 反派四层
小反派(前15%炮灰) → 中反派(15-45%,三段式:小胜→受挫→翻盘) → 大反派(45%显露,2-3集) → 隐藏反派(决战揭露,需≥3处可回溯伏笔)。
- 三原则：可恨/可信(为何作恶+能力来源+为何不早动手)/递进。

### 开场 6 模板
直接打脸 / 身份反转 / 极端困境 / 倒叙悬念 / 高甜钩子 / 重生起始。
- 黄金法则：第1秒画面冲击、第3秒核心冲突、第5秒好奇心；前30秒完成 冲突+主角处境+第一个钩子。

### 海外赛道（最对口本项目）
- **S级爆款**：Billionaire / Werewolf-Alpha / Flash Marriage / Secret Baby。
- **A级**：Revenge / Hidden Identity / Rejected Mate / Mafia。
- **8条中式叙事陷阱替代**：孝道绑架→经济控制、跪地道歉→语言道歉…（海外厌女/暴力反噬风险）。

### 留存卡点位置
首卡8-12集 / 二卡18-25集 / 三卡35-45集 / 终极卡倒数3-5集；最少间隔5集，前10集≤2个。
- 5禁忌：无悬念硬切/虚假悬念/信息提前泄露/同悬念连卡/卡无关小事。
- 注：海外 ReelShort 无传统付费墙，此表转为"关键留存集"使用。

## 14. 踩过的坑 + 证据来源

### 踩过的坑
| 坑 | 教训 |
|---|---|
| 产品定义被讨论带散 | 先定产品和工作流，不先定架构和评分权重 |
| 把评判当成要先做准的评分器 | 评判在"分析→写→盲测"闭环里长出来 |
| 绝对打分塌成 medium | 用 pairwise/对照锚点 |
| 把爆款拆成"机制规则"再套用生成 | 抽"料"(具体新鲜设定)不是"规则"，否则=cliche缝合怪 |
| 重构做结构减法 | 短剧靠信息密度留人，要做加法 |
| 只写舞台说明没镜头感 | 写可拍镜头；情绪标签句 hard-fail |
| 塞OS内心独白 | 去非视听语言（上帝视角/内心独白） |
| 症状≠病因 | 分层诊断（概念/结构/场景/对白），反推病灶 |
| 维度太多假精确 / 过早上多agent | 维度要少；闭环没验证前不碰架构 |
