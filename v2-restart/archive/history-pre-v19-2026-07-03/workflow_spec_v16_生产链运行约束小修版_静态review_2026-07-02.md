# workflow_spec_v16 生产链运行约束小修版 静态 review

> 日期：2026-07-02  
> 对象：`workflow_spec_v16_生产链运行约束小修版_2026-07-02.md`  
> 范围：静态 review，只查 spec 是否把生产链假执行点归位。未跑样本，未审最终剧本。  

---

## 1. 结论

这版是轻修，不是大重构。

它没有改 PRD，没有新建工位，没有加 reviewer 规则，也没有把短剧写法改成固定模板。

它做的事是把 `生产链能力执行对账_2026-07-02.md` 里发现的 6 个假执行点合并成 3 个生产约束：

| 原假执行点 | 合并后的生产约束 | 所在层 |
|---|---|---|
| 强弱分配、付费追问、反派燃料 | `追看强度地图` | S3 |
| 谁被迫行动、证据只能辅助、候选场面优选 | `场面承载选择` | S4 |
| 逐拍演出、短剧手艺加载 | `writer 行动链 / 戏的粗稿` | S5 |

这样改的目的不是让剧本千篇一律，而是让 writer 在写正文前真正拿到可创作的戏，而不是只拿到剧情结论。

---

## 2. 这版解决了什么

### 2.1 防止强度只停在口号

旧 v16 写了强弱分配、付费追问、反派压力，但它们还是散项。  
小修版把它们合成 `追看强度地图`，要求 S3 在进入 S4 前先决定：

- 哪些段落是波峰，哪些是呼吸；
- 每个波峰兑现什么旧问题，留下什么新问题；
- 每个付费前后欠观众什么答案；
- 反派后续还能靠什么燃料继续作死；
- 哪些地方不能为了刺激连续堆钩子。

这能直接回应 V28 暴露的问题：深度重构推开了外形距离，但中后段强度没有稳定上来。

### 2.2 防止 S4 默认滑向证据链

旧 v16 已经说不要文件 / 屏幕 / 会议替代人戏，但 V28 后半段仍然滑向证据链。  
小修版要求 S4 做 `场面承载选择`：

- 至少给两个候选承载方向；
- 说明为什么选这个方向；
- 判断证据 / 文件 / 屏幕是否只是辅助；
- 如果删掉证据和屏幕，场面还剩不剩人际压力、被迫行动和可见后果。

这不是禁用证据。它只是防止证据成为高压戏本体。

### 2.3 防止 S5 继续写成任务单

旧 v16 的 S5 已经有施工稿，但 V28 证明它仍可能变成“开场 / 中段 / 释放 / 结尾”的半提纲。  
小修版要求 S5 读起来像戏的粗稿：

- 压力从哪里进入现场；
- 谁做动作；
- 谁被迫反应；
- 谁失去什么；
- 谁看见了；
- 下一段还欠观众什么答案。

同时明确：这些问题必须能从施工稿里看出来，但不要求逐项编号填空，避免把施工稿又做成表格模板。

---

## 3. 没有做什么

- 没有把 reviewer 变重；
- 没有新增 S12 / 新 agent / 新资料池；
- 没有要求每集同一种开场、同一种打脸、同一种反转；
- 没有把 0xsline 原创流程照搬过来；
- 没有把 V5 当目标复刻；
- 没有继续修 V28 正文。

---

## 4. 静态风险

### R1：S3 的 `追看强度地图` 仍可能被写成表格口号

风险存在，但比旧版小。  
小修版已经写明它不是固定公式，不能变成“每集开场钩子 + 中段打脸 + 结尾反转”。  
真正能否生效，需要样本跑出来看 S4/S5 是否真的吃到了这张地图。

### R2：S4 至少两个候选方向会增加一点工作量

这是有意保留的轻负担。  
原因是过去 agent 太容易默认选安全载体：文件、系统、会议、证据。  
两个候选不是为了复杂化，而是逼它做一次创作选择。

### R3：S5 仍然可能写得太概括

小修版已经把“逐项填空”改成“戏的粗稿”，但是否真能写成粗稿，需要样本验证。  
如果下一轮还是只写剧情结论，问题就不是 v16 没写，而是 S5 skill/prompt 没把写作动作烤进去。

---

## 5. 进入样本验证前的通过判断

静态上，这版可以进入样本验证。

理由：

- 它没有大改产品目标；
- 它没有把短剧写法模板化；
- 它没有让 reviewer 兜底；
- 它把 6 个假执行点合并成 3 个生产约束；
- 它把 V28 的失败点纳入对比标准；
- 它保留了洗稿主线、DNC、连续性、海外语感和防复刻。

但静态通过不等于产物会好。  
下一步必须用同一飞书三星样本跑首批 1-10 集，并横向对比 V5 / V22 / V24 / V25 / V28。
