## 一、主线：爆款有效性迁移的 7 个动作

> 这是整个产品的心脏。一个源本要洗成新本，本质就是走这 7 个动作。
> 每个动作是一个"判断 / 创作"动作，不是"填表 / 套规则"。
> **不同洗稿需求走的深度不同**（见第二部分），但动作的骨架是这一条。

```
源本 + 用户需求
   │
 动作1 诊断有效性 ── 这个源本为什么有效？靠什么让观众追、让观众付费？
   │
 动作2 分层 ── 把源本拆成四类：命脉 / 外壳 / 水分 / 烂点
   │
 动作3 减到 minimum ── 逐层剥离，留下"丢了就不爆"的最小命脉集
   │
 动作4 承载判断 ── 用户要的新壳/新集数，装得下这些命脉吗？
   │
 动作5 迁移或等效替换 ── 装得下就迁移；装不下就换一种方式达到同样效果（不硬搬）
   │
 动作6 生成洗稿方案 ── 从以上判断，得出"这一稿具体怎么洗"
   │
 动作7 验证没洗飞 ── 首批新本，怎么证明有效性真的迁移过去了
   │
新本（首批） + 改动说明
```

### 动作1 · 诊断有效性
- **要回答**：这个源本为什么是爆款？观众为什么会追、会付费？
- **怎么做**：通读源本，找出它的有效性来源——情绪怎么起伏、钩子卡点在哪、爽点怎么兑现、什么让人忍不住看下一集。**先不下"好坏分"，先看懂"它靠什么有效"。**
- **调用工具**：情绪曲线数值化（逐集 -9~+9，画出峰谷）、短剧标准库（钩子5类/爽点矩阵当尺子，看它用了什么、强在哪）。
- **产出**：一份"它为什么有效"的说明（这个本的有效性长什么样）。

### 动作2 · 分层（命脉 / 外壳 / 水分 / 烂点）
- **要回答**：源本里每个东西，属于哪一类？
  ```
  命脉 = 让它有效的内核（拿掉就不爆了）—— 必须迁移
  外壳 = 题材/人物/背景/世界观（可以换）—— 换壳对象
  水分 = 注水、拖沓、无效内容 —— 洗掉
  烂点 = 源本本身的毛病（逻辑漏洞/假动机）—— 洗掉，别继承
  ```
- **怎么做**：基于动作1的有效性诊断，把源本元素归到四类。**关键判断**：分清"它好看是因为这个(命脉)"还是"这个只是恰好长这样(外壳)"。
- **产出**：四类清单。

### 动作3 · 减到 minimum
- **要回答**：命脉里，哪些是"绝对不能丢"的最小集？
- **怎么做**：从动作2的命脉里继续剥——一个个试着拿掉，看"拿掉它，这个本还爆不爆"。爆 → 它不是 minimum；不爆 → 它是 minimum，锁住。
  > 注：v2 里没有现成的"逐层减"方法，这是新设计。核心是"剥离测试"，不是打分。
- **用户参与**：工具给出 minimum 候选 → 用户结合自己判断增删 → 锁定。
- **产出**：minimum 命脉清单（这次洗稿死守的底线）。

### 动作4 · 承载判断
- **要回答**：用户要换的新壳（新题材/新世界观）、新集数，能不能装下这些 minimum 命脉？
- **怎么做**：拿 minimum 命脉，逐条问"新壳里有没有对应的承载方式"。
  ```
  例：命脉='当众羞辱主角，让观众揪心'
     新壳='军营'→ 军营有没有'当众羞辱'的合理方式？有（羞辱军衔）→ 能承载
     新壳=某个没有等级关系的设定 → 没有羞辱的合理场景 → 承载不了 → 进动作5
  集数同理：40集的命脉节奏，铺到80集/压到30集，装得下吗？
  ```
- **产出**：每条命脉"能否被新壳承载"的判断。

### 动作5 · 迁移或等效替换
- **要回答**：能承载的怎么迁移，承载不了的怎么办？
- **怎么做**：
  ```
  能承载 → 迁移：把命脉的"观众体验/情绪作用"在新壳里重演（不是照抄数字标签）
  承载不了 → 等效替换：换一种新壳逻辑支持的方式，达到同样的观众体验/情绪作用
            （不硬搬原形式——硬搬就假）
  集数变了 → 用扩写/缩写方法重排节奏，让命脉在新集数里仍然有效
  ```
- **铁律**：迁移/替换的标准是"观众的体验和情绪走向有没有复现"，**不是"形式数字有没有对齐"**。
- **产出**：每条命脉在新壳里的落地方式。

### 动作6 · 生成洗稿方案
- **要回答**：综合以上，这一稿具体怎么洗？
- **怎么做**：把动作1-5的判断收成一个可执行的方案——新壳设定、命脉怎么落、水分烂点怎么处理、集数怎么编排、首批先洗几集。
- **用户参与**：方案给用户确认后才动手。
- **产出**：洗稿方案（用户确认）。

### 动作7 · 验证没洗飞
- **要回答**：洗出来的首批新本，有效性真的迁移过去了吗？
- **怎么做**：对照源本验证（**标准是源本，不是凭空的绝对标准**）：
  ```
  · minimum 命脉，在新本里还在吗？（画新本情绪曲线 vs 源本，看峰谷的体验作用有没有对应）
  · 新壳自洽吗？人物动机/行为在新背景里站得住、不假吗？
  · 水分烂点洗掉了吗？有没有把源本的毛病也带过来？
  · 有没有为了刺激硬堆钩子爽点，反而破坏了剧本自洽？
  · 有没有 AI 味/换皮痕迹？
  ```
- **产出**：验证结果。没过 → 回前面对应动作修；过 → 交付首批。

## 二、不同洗稿需求，走多深（力度与维度）

> 用户的需求不是单一档，是在一大片维度上做选择。这些选择决定 7 个动作各走多深。

### 这是一片维度空间，不是几个固定模式
用户可以选择改动的，至少包括（**这一块需和用户继续聊全、固定成清单——见待定**）：
```
· 题材/类型（都市/霸总/狼人ABO/重生/真假千金…）
· 风格（男频/女频/lgbt）
· 世界观/背景/身份/场景
· 爽点类型、钩子类型、情绪基调
· 集数（扩写/缩写）、节奏密度
· 目标市场/平台
· 绝对不能动的、特别想要的
```

### 力度是一条谱（不是几个并列模式）
```
轻 ──────────────────────────────────→ 重
润色      换壳（换外壳，命脉骨架不动）      深度重写（连命脉都可能调整）
```
- **集数改编不是独立模式**，是可以叠加在任何力度上的维度（换壳可以同时改集数，深度重写也可以）。
- 力度越轻，动作 2/3（分层、减minimum）和动作 4/5（承载、替换）走得越浅；力度越重，每个动作都走得越深。

### 共同地基（不管什么力度都要做）
动作1（诊断有效性）、动作2（分层）、动作3（减minimum）、动作7（验证）——**任何洗稿都得先知道源本为什么有效、命脉是什么、最后验证没洗飞**。区别只在动作 4/5/6 走多深。

## 三、用户怎么参与（交互逻辑）

> 详见 `项目基础说明.md` 第五部分。核心：先对齐、再动手、边做边确认。

用户必须拍板的节点：
1. **需求对齐**（动作前）：要换什么、改多狠、改不改集数、什么不能丢、特别想要什么。
2. **minimum 命脉确认**（动作3后）：工具给候选，用户增删锁定。
3. **洗稿方案确认**（动作6后）：方案对了才动手洗。
4. **首批交付确认**（动作7后）：认可了才往下续写。
5. **续写**：分批洗（每批几集），用户每批过目。

工具负责：读透源本、找有效性、动手洗、保证命脉迁移+新壳自洽+无AI味、**改动透明可追溯**。
用户负责：定方向、做取舍、最终拍板。

## 八、v6 → v7 改了什么

| 改动 | 原因 |
|---|---|
| 主干换成"7个迁移动作" | Codex 指出核心是迁移动作，不是环节流程/七交付物 |
| 七交付物 → 降级为"动作的产物"（第五部分） | 交付物是产物不是主线 |
| 自检三级表 → 降级，并入动作7"验证没洗飞" | 原表是规则堆砌，淹没了"验证"这个动作 |
| 标准库/工艺库 → 降级为"被动作调用的工具"（第四部分） | 它们是工具不是主干 |
| "三种模式并列" → 改为"力度谱 + 维度空间"，集数改编是叠加维度 | 用户纠正：集数改编非独立模式，力度远不止几档 |
| 砍掉 6维权重百分比、吸睛密度阈值、评分排序残留 | 评分器残留，洗稿对照源本不需要 |
| 补"命脉/外壳/水分/烂点"四分法（动作2） | Codex 提出，v6 只有命脉/壳两分 |
| 地基内容拆出到 `项目基础说明.md` | 之前"产品是什么/怎么交互"一直被几句话带过 |
