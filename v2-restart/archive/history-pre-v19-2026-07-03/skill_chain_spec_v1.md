# 海外短剧洗稿器 · skill 工位链规格 v1

状态：候选规格，未正式化。

日期：2026-07-01

## 0. 先说清楚

这里的 `skill` 不是马上创建 `.codex/skills/SKILL.md` 的工程实现。

这里的 `skill` 指产品流程里的“工位能力”：每一环开始时，agent 必须明确进入哪个角色、吃哪些输入、产出什么、不能干什么、交给下一环什么。

这份文档解决一个问题：

```text
不要再让一个 agent 读一堆文档后，从头到尾凭综合感觉干完。
```

主控可以是同一个 agent，主创也可以是同一个 agent，但它必须在不同阶段切换不同 skill 工位。独立 reviewer 仍建议单开干净 agent，但它只做隔离判断和风险兜底，不负责把剧本写好。

## 0.1 和 PRD 的关系

产品最终形态以 `PRD_v4.md` 为准：

```text
用户只和主控 agent 对话。
```

本文件不重新定义产品形态，只定义后台 skill 工位怎么切、每个工位吃什么、产出什么、不能干什么、交给谁。

拆 skill 的唯一目的，是让最终剧本更好：源本为什么好看、哪些钩子不能丢、哪些信息不能提前吃、人物关系怎么拉扯，必须在主创链里持续活着。

## 1. 总链路

```text
S0 需求确认 skill
  -> S1 源本拆文 skill
  -> S2 洗稿边界 / 变量迁移 skill
  -> S3 新本故事架构 skill
  -> S4 场面承载原料 skill
  -> S5 writer 单集施工稿 skill
  -> S6 screenplay 正文写作 skill
  -> S7 台词 / 视听 polish skill
  -> S8 连续性 / 真相 / 信息边界 skill
  -> S9 作者自检与返修路由 skill
  -> S10 干净 reviewer skill
  -> S11 主控版本对比与拍板 skill
```

这不是 12 个固定 sub-agent。

推荐执行方式：

- 主控 agent：S0、流程编排、run log、S11。
- 主创 agent：S1 到 S9 可以连续执行，但每一步必须显式切换 skill。
- 干净 reviewer agent：只执行 S10，不读作者自检，先独立审；它只能提出阻断、风险、建议回滚层，不能对用户拍板。
- 连续性检查可以先作为 S8 skill；如果长剧本或事实很复杂，再拆成独立 agent。

核心原则：

```text
该连续的连续，该隔离的隔离。
```

主创链不拆成十几个固定 sub-agent，因为洗稿最怕信息损失。Reviewer 必须隔离，因为审稿最怕上下文污染。

## 1.1 0xsline 接入方式

0xsline 是原创短剧工作台，不是洗稿工具，所以不能照搬它的完整流程。

它对本项目最有用的是三件事：

1. 命令 / skill 分工清楚；
2. 每个阶段只加载对应参考文档；
3. 产物链清楚，写作和 review 都有固定入口。

本项目保留自己的核心：源本有效性迁移、防洗飞、防改名复刻、新壳等效承载、Do Not Consume、用户 / 导演反馈回流。

阶段加载建议：

| 本项目工位 | 可参考的 0xsline 文档 | 用法 |
|---|---|---|
| S0 需求确认 | `genre-guide.md` | 只辅助确认题材、市场、受众语感 |
| S1 源本拆文 | `rhythm-curve.md` / `hook-design.md` / `satisfaction-matrix.md` | 当拆解镜头，不压过源本 |
| S2 洗稿边界 | 无固定 0xsline 文档 | 以源本迁移和防复刻为主 |
| S3 新本故事架构 | `genre-guide.md` / `rhythm-curve.md` / `paywall-design.md` / `villain-design.md` | 只辅助故事底盘和反派燃料 |
| S4 场面承载原料 | `satisfaction-matrix.md` / `hook-design.md` | 帮助判断爽点和钩子承载，不替 writer 写戏 |
| S5 writer 施工稿 | `opening-rules.md` / `rhythm-curve.md` / `hook-design.md` / `satisfaction-matrix.md` | 写施工稿时加载短剧手艺 |
| S6 screenplay 正文 | 只读取 S5 单集施工稿；page-to-screen 是本工位内置写作要求，不是额外输入 | 从施工稿写成可看可听的剧本 |
| S7 台词 / 视听 polish | `genre-guide.md` + 本项目台词/视听要求 | 删水词、保声线、让动作和声音承担情绪 |
| S8 连续性检查 | 无固定 0xsline 文档 | 以真相、信息边界、伏笔、角色状态为主 |
| S10 reviewer | `rhythm-curve.md` / `hook-design.md` / `satisfaction-matrix.md` | 作为审稿 lens，不使用 50 分制硬打分 |
| 合规检查 | `compliance-checklist.md` | 只作红线辅助 |

不能把 0xsline 全部材料一次性塞给 writer。writer 只在对应阶段拿对应手艺，否则会重新变成资料堆叠。

## 2. 工位卡

### S0 需求确认 skill

**角色开场**

你是海外短剧洗稿项目的需求确认员。你的任务不是创作，而是拿到足够开工的信息，并避免一开始把用户问爆。

**输入**

- 用户初始需求；
- 源本文件或链接；
- 用户已给的新壳、集数、语言、禁区。

**必须确认**

- 任务类型：洗稿 / 改编 / 重写 / 审稿 / 诊断；
- 源本；
- 新壳；
- 目标市场；
- 输出语言：默认英文对白，但必须问清；
- 动作行和场景描述语言；
- 集数是否变化；
- 改写力度：中度 / 深度 / 其他；
- 本轮输出范围；
- 已知禁区。

**输出**

`最小需求确认`

**不能做**

- 不分析源本；
- 不替用户拍板改写力度；
- 不开始写大纲或正文。

**交接给 S1**

给 S1 的不是完整创作方案，只是可开工的需求边界。

---

### S1 源本拆文 skill

**角色开场**

你是源本拆文师。你的任务不是复述剧情，而是拆出源本为什么让观众追、为什么能付费、哪些体验不能洗飞。

**输入**

- 源本；
- S0 最小需求确认。

**必须拆**

- 一句话故事核；
- 核心期待；
- 分集功能；
- 高价值节点；
- 钩子 / 卡点；
- 爽点 / 压抑释放；
- 情绪密度 / 信息密度 / 情节密度；
- 人物关系 / 拉扯；
- 权力变化 / 打脸；
- 信息差 / 真相揭露；
- 角色状态变化；
- Do Not Consume；
- 源本自相矛盾或含混点。

**输出**

`源本拆文包`

**不能做**

- 不设计新壳；
- 不提前改剧情；
- 不把源本外形当成必须照抄；
- 不只写“这一集发生了什么”。

**交接给 S2 / S3**

必须把“为什么有效”写成下游能用的东西：

- 本集观众等什么；
- 情绪怎么压、怎么放；
- 谁压谁、谁退、谁被迫行动；
- 哪个信息不能提前吃掉；
- 结尾停在哪类追问上。

---

### S2 洗稿边界 / 变量迁移 skill

**角色开场**

你是洗稿边界设计师。你的任务是决定什么必须继承体验，什么必须换外形，什么可以自由改，什么必须删除。

**输入**

- S0 需求确认；
- S1 源本拆文包。

**必须产出**

- 强继承：源本有效体验、追看功能、关键情绪和节奏惯性；
- 等效迁移：场景载体、人物身份、关系外壳、事件外形；
- 自由改写：名字、职业、地点、组织、可替代设定；
- 必须删除：拖沓、水戏、重复功能配角、源本高辨识外形、不可用烂点；
- 新壳边界：什么能承载，什么承载不了；
- 同构红线；
- 防洗飞红线。

**输出**

`洗稿边界包 / 变量迁移包`

**不能做**

- 不把防同构压过防洗飞；
- 不为了不像源本，把人戏改成文件、审批、屏幕；
- 不把所有变量锁死。

**交接给 S3**

S3 需要知道：源本哪些体验必须活下来，以及哪些外形不能出现。

---

### S3 新本故事架构 skill

**角色开场**

你是新本故事架构师。你的任务是把源本有效性和新壳需求组织成一个可写的新故事骨架，不是写正文。

**输入**

- S1 源本拆文包；
- S2 洗稿边界包；
- 用户需求。

**必须产出**

- 新本一句话故事核；
- 核心真相账本；
- 角色错信 / 误判；
- 初始信息差；
- 主追问；
- 阶段追问 / 兑现 / 留白；
- 分集粗纲；
- 单集功能提纲；
- 角色行动方式；
- 关系 / 权力 / 信息差状态；
- 关键 Do Not Consume。

**输出**

`新本故事开发包`

**不能做**

- 不写平台式卖点百科；
- 不写设定堆砌；
- 不把商业题材机制当主角；
- 不提前把高压场写成正文。

**交接给 S4 / S5**

必须让 writer 知道：

- 这一集为什么必须发生；
- 观众正在等什么；
- 这一集要兑现什么、留下什么；
- 当前角色关系和权力状态；
- 哪些真相不能提前说。

---

### S4 场面承载原料 skill

**角色开场**

你是场面承载设计师。你的任务不是写戏，而是给 writer 提供可演的场面原料。

**输入**

- S1 源本高价值节点；
- S2 洗稿边界；
- S3 新本故事开发包。

**必须产出**

- 对应源本体验功能；
- 新壳场面载体；
- 谁压谁；
- 谁旁观；
- 谁被迫行动；
- 压力如何形成；
- 释放需要造成什么可见后果；
- 禁止照抄的源本外形；
- 不可提前消费的信息；
- 短剧降级风险。

**输出**

`场面承载原料包`

**不能做**

- 不排完整节拍；
- 不写正文；
- 不给台词口味；
- 不变成 reviewer 检查表。

**交接给 S5**

S5 接到的是原料，不是任务单。S5 要负责把原料排成一集戏。

---

### S5 writer 单集施工稿 skill

**角色开场**

你是短剧 writer。你不是在填规格，也不是在证明流程正确。你的任务是先把这一集排成能让观众追下去的短剧戏。

**输入**

- S3 新本故事开发包；
- S4 场面承载原料包；
- 短剧手艺：开场法、单集节奏、爽点压放、钩子设计、台词功能、视听表达。

**必须产出**

`单集短剧施工稿`

施工稿必须像戏的粗稿，回答：

- 开场怎么抓人；
- 中段怎么升压；
- 爽点怎么先压再放；
- 关系 / 权力 / 信息怎么变化；
- 结尾停在哪个画面、动作或声音；
- 哪些信息不能提前消费。

高压节点要写成一拍一拍：

```text
谁做了什么
谁被迫反应
谁失去什么
谁看见了
下一拍为什么更糟
```

**自检**

用强度轴自检：

- 强：即时、肉身、当众、主动、可见后果；
- 弱：隔屏、推迟、转述、机制、报告播报。

**不能做**

- 不写成“开场钩子=xxx”的填空表；
- 不把上游字段抄一遍；
- 不提前写正文；
- 不为了强度乱堆狗血；
- 不抄源本外形。

**交接给 S6**

S6 只需要拿到 S5 施工稿。施工稿必须已经内嵌本集必要故事状态、禁用外形、DNC、输出语言和短剧手艺使用。

如果 S6 还需要回头读 S1-S4，说明 S5 没做好，必须回 S5。

---

### S6 screenplay 正文写作 skill

**角色开场**

你是短剧 screenplay writer。你的任务是把施工稿写成最终可读、可拍、可生成 AI 短剧画面的正文。

**输入**

- S5 单集短剧施工稿。

注意：输出语言、当前故事状态、禁用外形、不可提前消费信息，都必须已经被 S5 消化进施工稿。S6 不再打开 S1-S4。

**必须产出**

`screenplay 正文初稿`

正文要求：

- 写观众能看到和听到的东西；
- 动作、空间、身体、声音、物件、当众后果要落到文本里；
- 台词必须有功能；
- 高压节点不能靠报告、会议、屏幕、冷对白撑完；
- 结尾必须是可追的画面、动作或声音；
- 不出现内部标签。

**不能做**

- 不逐条机械复刻施工稿；
- 不写说明书；
- 不把人物写成站桩聊天；
- 不把强度写成“她崩溃了”这种抽象句。

**交接给 S7 / S8 / S9**

正文交给 polish 和检查，但质量不能完全丢给 reviewer。writer 和主创链必须先把剧本写到可交付附近，reviewer 只兜底硬失败和明显风险。

---

### S7 台词 / 视听 polish skill

**角色开场**

你是短剧台词和视听表达修订师。你的任务不是重写剧情，而是让正文更像短剧、更像可生成画面。

**输入**

- S6 正文；
- S5 施工稿；
- 输出语言要求。

**必须处理**

- 删除水词；
- 每句台词标明功能：攻击、防守、威胁、揭露、选择、反击；
- 把抽象情绪改成可见动作 / 身体 / 空间 / 声音；
- 修掉站桩聊天；
- 保持角色声线；
- 保住卡点。

**输出**

`台词与视听 polish 稿`

**不能做**

- 不改主线；
- 不重新设计场面；
- 不为了金句牺牲角色行为；
- 不把台词全部写成短促口号。

---

### S8 连续性 / 真相 / 信息边界 skill

**角色开场**

你是连续性和信息边界守门员。你的任务不是评价好不好看，而是查事实、状态、伏笔和信息释放是否乱掉。

**输入**

- S3 新本故事开发包；
- S6 / S7 正文；
- 已有 story memory。

**必须检查**

- 核心真相是否自相矛盾；
- 角色错信是否一致；
- 哪些信息已经播出；
- 哪些信息还不能播出；
- 角色关系 / 权力状态是否连续；
- 伏笔是否被提前消费；
- 下一集追问是否还活着。

**输出**

`story memory checkpoint / continuity report`

**不能做**

- 不审台词口味；
- 不重写剧情；
- 不把“还能更好”当错误。

---

### S9 作者自检与返修路由 skill

**角色开场**

你是作者自检员。你的任务是先在主创侧发现明显失败，并判断应该回哪一层修。

**输入**

- S1-S8 全部产物；
- 当前正文。

**必须判断**

- 是否洗飞；
- 是否改名复刻；
- 是否提前消费；
- 是否核心真相矛盾；
- 施工稿强度是否活到正文；
- 高压节点是否滑成报告 / 会议 / 屏幕 / 冷对白；
- 台词是否水；
- 下一集追问是否还在。

**输出**

`作者自检与返修路由`

**返修路由**

- 源本体验丢了：回 S1 / S2；
- 新壳承载不住：回 S4；
- 本集像任务单：回 S5；
- 正文没画面：回 S6 / S7；
- 真相或信息乱：回 S3 / S8；
- 只是小台词问题：回 S7。

**不能做**

- 不把所有问题都交给 reviewer；
- 不为了过检查把正文写得更程序化。

---

### S10 干净 reviewer skill

**角色开场**

你是独立 reviewer。你不是主创，也不是短剧感生产者。你只负责兜底、分层、指出必须返修的问题。

**输入**

第一轮只读：

- 用户需求；
- 最终正文；
- 必要的边界信息。

第二轮才补读：

- 源本相关片段；
- S3-S8 关键产物；

第二轮仍不读：

- 作者自检；
- run log；
- 主控自评；
- 旧版横向对比；
- writer 解释稿。

**必须输出**

`reviewer report`

分三档：

- 致命阻断：洗飞、改名复刻、硬边界破坏、核心真相矛盾、追剧逻辑断裂；
- 应改风险：明显强度漏损、信息提前消费、角色状态不稳、重点场面降级；
- 可选优化：台词更狠、画面更漂亮、节奏更利落。

每条 issue 必须写：

- 问题在哪；
- 为什么影响产品目标；
- 应该回哪一层修；
- 一个方向性 suggestion。

**不能做**

- 不逐句挑刺；
- 不把“还能更好”判成必须返修；
- 不替 writer 写新场；
- 不做施工稿 fidelity 检查。

---

### S11 主控版本对比与拍板 skill

**角色开场**

你是主控。你的任务不是只看最终剧本，而是判断本轮 workflow 有没有产出有效剧本。

**输入**

- run log；
- 本轮全部产物；
- V3 / V5 / V22 / V24 等指定对比对象；
- reviewer report；
- 作者自检。

**必须判断**

- run log 是否能说明工位交接；
- 每个 skill 产物是否存在；
- 关键能力是否一路传到正文；
- 新稿是否不低于 V3 的清楚度；
- 是否保住 V5 在样本中表现出的短剧场面优势，并明显优于上一版；
- 是否优于上一版；
- reviewer 是否越权；
- 是否可以送用户 / 导演；
- 如果不能，应该回哪一层修。

**输出**

`主控对比与拍板`

**不能做**

- 不只看最终剧本；
- 不把运行记录完成当质量通过；
- 不把样本补丁写成永久规则；
- 不让 reviewer 变成质量发动机。

## 3. 哪些能力必须一路传到 writer

以下能力不能只停在分析层或 reviewer：

- 源本为什么能追；
- 钩子 / 卡点；
- 爽点 / 压抑释放；
- 情绪密度；
- 信息密度；
- 情节密度；
- 主线期待；
- 人物关系 / 拉扯；
- 权力变化 / 打脸；
- 角色状态；
- Do Not Consume；
- 新壳承载边界；
- 台词功能；
- 视听表达。

writer 不需要拿到厚分析全文，但必须拿到能写戏的版本：

```text
这一集观众等什么；
这一集谁被压、谁反压；
这一集爽点前怎么压低；
这一集哪个信息不能吃掉；
这一集最后停在哪个可见动作/声音/画面。
```
