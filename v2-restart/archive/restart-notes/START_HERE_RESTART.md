## 先说结论

这个目录现在不要当成“已有产品方案”来接着做。

它更适合作为一包材料：

- 自有剧本样本

## 当前判断

前一轮工作最大的问题不是“某个模块没写好”，而是产品定义被讨论带散了：

- 一会儿像剧本分析器
- 一会儿像评分器
- 一会儿像多 agent 架构
- 一会儿像洗稿/重构系统
- 一会儿像完整短剧生产链

这些方向都可能相关，但不能混在一个未定型方案里。

重启时第一步不应该继承旧 spec，而应该重新回答产品问题：

> 这个工具最终服务谁？用户输入什么？系统每一步产出什么？哪些产出给人看，哪些给后续 AI agent 用？第一阶段只验证哪一个最小闭环？

## 重启建议

下一轮从零做时，建议只做一件事：

> 先定义产品工作流，不定义 agent 架构，不定义评分权重。

最小问题：

1. 用户拿一部旧剧本进来，想得到什么？
2. 甲方、导演、监制、编剧、AI 视频 agent 分别需要看什么？
3. 系统要先拆出哪些事实，才能做判断或重构？
4. 哪些判断必须由人确认，哪些可以自动做？
5. 第一阶段用哪 2-3 个剧本验证这个工作流真的有用？

这个问题没回答清楚前，不要继续写完整 spec、评分表、多 agent 架构或生成器。
