> 上游产品定义：`PRD_v4.md`
> 直接来源：`workflow_spec_v7_创作优先精简候选版.md`、`workflow_spec_v2.md`、V5 样本写作前工作稿。

---

## 0. v8 解决什么

v8 不改产品定义。

产品仍然是：

```text
把一个已被市场验证或值得参考的短剧源本，
洗成一个新壳下仍然好看、能追、能付费的新剧本。
```

v7 的减法方向是对的：不能让 workflow 变成证明题。

但 V17 暴露了一个新问题：

```text
流程变轻后，写作前输入太薄。
主创知道每集功能，但缺少把功能变成场面的中间抓手。
于是正文容易变成剧情功能复述、动作占位、台词金句化。
```

v8 的改法不是回到 v2 的重流程，而是恢复 V5 最有效的一层：

```text
关键场面迁移卡。
```

它只服务高压节点，不做全剧大表。它的作用是把：

```text
源本追看功能
-> 新壳场面
-> 动作链
-> 权力/关系/情绪变化
-> 可见后果
-> 正文剧本
```

接起来。

- 或只写成备注；

## 3. v8 主流程

```text
0. 最小需求卡
1. 主创读本笔记
2. 二次需求确认
3. 洗稿方案
4. 关键场面迁移卡
5. 首批写作简报
6. 首批正文
7. 作者问题清单
8. 干净 reviewer
9. 主控判定与版本对比
```

相对 v7，只新增一个强制主步骤：

```text
4. 关键场面迁移卡
```

这个步骤不是规则扩容，而是把 V17 缺掉的写作抓手补回来。

## 4. 改写力度

默认不做轻度换名。

### 贴源换壳

只有用户明确要求“尽量贴源本，只换身份/背景/名字”时使用。

即便贴源换壳，也不能整串复用源本高辨识桥段。

### 中度重构

默认基准。

保留：

- 源本核心期待；
- 付费/留存阶段；
- 关键钩子惯性；
- 情绪走势；
- 爽点周期；
- 关系拉扯；
- 信息差和权力变化。

重建：

- 高辨识事件外形；
- 场景；
- 道具；
- 证据形式；
- 角色身份和职业；
- 具体对白。

中度重构不是按集翻译源本剧情。

### 深度重构

允许更大幅度重排事件、集数和阶段比例。

但仍然不能丢掉源本让观众继续看、继续付费的发动机。

## 5. 0. 最小需求卡

目标：拿到足够开工的信息，不把用户问爆。

必须确认：

```text
源本：
新壳：
目标市场：
集数是否变化：
改写力度 / 变量边界：
本轮输出范围：
输出语言 / 审稿语言：
正文用途：人审稿 / AI 视频生成用剧本 / 中英双语 / 字幕台词版
已知禁区：
```

默认：

```text
台词英文；
动作和场景描述中文；
内部分析中文；
产品目标是 AI 出海短剧剧本文字。
```

---

## 6. 1. 主创读本笔记

目标：让主创真正知道源本为什么能追，不是复述剧情。

必须回答：

```text
1. 一句话故事核
2. 观众追看的核心期待
3. 首批每集凭什么让人继续看
4. 高价值节点：具体发生什么，强度靠什么成立
5. 钩子与卡点：停在哪一拍，为什么想看下一集
6. 爽点 / 压抑释放：哪里压低，哪里兑现
7. 情绪走势和信息走势
8. 人物关系拉扯：谁靠近、谁退开、谁误判、谁压谁
9. 角色状态：谁在忍、谁快崩、谁掌握主动权
10. 源本水分、烂点、禁止复用外形
```

通过标准：

```text
能说清源本“让人心痒继续看”的东西；
能说清哪些是必须迁移的体验，哪些只是源本外形；
能为关键场面迁移卡提供原料。
```

失败信号：

```text
只剩剧情摘要；
只剩节点编号；
只剩“有钩子、有爽点、有反转”这种标签；
说不出源本强度靠什么成立。
```

---

## 7. 2. 二次需求确认

目标：读完源本后，主创先判断新壳能不能承载源本，再让用户确认关键取舍。

主创先给建议，不把问题全甩给用户。

必须确认：

```text
新壳是否能承载源本最强追看机制；
改写力度用贴源换壳 / 中度重构 / 深度重构中的哪一种；
哪些变量贴近源本；
哪些变量等效迁移；
哪些变量自由改；
哪些源本外形必须避开；
哪些源本烂点必须删；
集数变化时，钩子和留存阶段如何重排；
用户特别想保留或特别不想要什么。
```

如果用户需求会明显削弱源本有效性，必须直接提示风险。

---

## 8. 3. 洗稿方案

目标：在写正文前，把“这次怎么洗”讲清楚。

这是给用户和主控看的，不是给 writer 的厚审计表。

结构：

```text
1. 本次改写一句话方案
2. 保留什么：核心期待、钩子惯性、爽点周期、情绪走势、关系拉扯、信息差、权力变化
3. 新壳故事发动机：新壳里靠什么自然产生冲突、压迫、反击和权力变化
4. 换掉什么：身份、职业、场景、世界观、源本高辨识事件外形
5. 删除什么：拖沓、水分、烂点、重复功能角色
6. 首批阶段骨架：每集推进什么，卡在哪里
7. 高压节点处理：哪些节点必须进入关键场面迁移卡
8. 主要风险：洗飞、改名复刻、短剧降级、长剧化、状态漂移
```

“新壳故事发动机”不是工程词。

它只回答一句话：

```text
这个新版本里，剧情为什么会自然发生，而不是源本剧情换场景？
```

---

## 9. 4. 关键场面迁移卡

目标：把源本功能变成新壳下可写、可看、能推进的短剧场面。

这是 v8 的核心补回项。

### 适用范围

只做高风险节点：

- 前 1-3 集；
- 每集结尾卡点；
- 强危机、强羞辱、强误会、强打脸；
- 身份揭露；
- 关系质变；
- 火葬场；
- 首批结尾；
- 同构风险高的源本桥段；
- 上一版被导演/reviewer 指出的失败点。

普通过渡场不做迁移卡。

### 输出结构

```text
关键场面迁移卡

源本节点：
源本体验功能：
源本强度来源：
源本禁止外形：

新场景目的：
新壳故事发动机如何支撑这一场：

新冲突动作链：
  1. 起手动作：
  2. 压迫升级：
  3. 反击 / 反转 / 选择：
  4. 不可逆后果：

权力变化：
关系变化：
信息差变化：
情绪变化：

观众直接看到 / 听到什么：
证据、文件、录音、屏幕信息如果出现，会触发什么人物行动：
台词功能上限：攻击 / 防守 / 逼迫 / 拒绝 / 试探 / 威胁 / 揭露 / 反击 / 选择
卡点停在什么动作、反应、选择或后果前：
禁止提前消费：

结论：可进入写作简报 / 需重建 / 放弃
```

### 通过标准

不通过的情况：

```text
只是“放证据 -> 众人震惊 -> 反派否认 -> 再放证据”；
人物没有被迫行动；
权力、关系、信息或情绪没有变化；
只能靠对白讲出情绪；
卡点已经把下一集核心爽点说完；
新场面只是在源本事件上换名词。
```

通过的情况：

```text
这一场有具体动作链；
每个重要信息都会造成现场后果；
观众能看见谁失去什么、谁压过谁、谁被迫选择；
writer 能直接拿它写正文。
```

## 10. 5. 首批写作简报

目标：把读本笔记、洗稿方案和关键场面迁移卡压成 writer-facing 输入。

简报要薄，但不能空。

每集写：

```text
上一集停在哪里：
本集观众在等什么：
本集开头怎么接住：
本集主要冲突：
本集最大刺激点：
本集关系 / 局势 / 信息 / 情绪变化：
本集钩子停在哪里：
本集不能提前说破什么：
本集源本追看功能迁移什么：
本集禁止复用什么源本外形：
本集使用哪张关键场面迁移卡：可空
```

高压场追加：

```text
谁压谁：
谁被迫行动：
谁失控或做选择：
观众能直接看到 / 听到什么：
这场结束后有什么不可逆后果：
```

写作简报不能替代关键场面迁移卡。它只负责把卡片分配到每集正文。

---

## 11. 6. 首批正文

目标：交付真正的剧本文字。

正文默认格式：

```text
EPISODE N

INT./EXT. 场景 - 时间

动作行：只写观众能看见、听见、感受到的外部行为、空间变化、物件、声音、身体反应。

角色名
English dialogue.
```

要求：

```text
1. 正文不能出现 Purpose / Function / Source Ref / Gate / 这一场的作用是。
2. 每场必须有当前行动，不写“她很痛苦”这种空描述。
3. 高压场不能靠多人低信息对白拖时间。
4. 台词要有功能：推进、压迫、遮掩、误判、试探、反击、卡点。
5. 能用动作、空间、物件、身体反应、当众后果表达的，不优先用解释性对白。
6. 结尾钩子停在动作、反应、选择或后果前一拍，不把下一集爽点说完。
7. 不写孤立占位动作，例如只写“她抬眼 / 他僵住 / 全场安静”，除非前后有具体动作和状态支撑。
8. 高压场必须消化对应迁移卡，不能只借一句口号。
```

这里不是要求每场都爆。

低压过渡可以克制，但不能空转，不能用无效聊天冒充推进。

---

## 12. 7. 作者问题清单

目标：作者先诚实指出自己这版最可能哪里不行。

只写：

```text
1. 我最担心洗飞的地方；
2. 我最担心像改名复刻的地方；
3. 我最担心短剧降级或长剧化的地方；
4. 我认为旧版或源本局部更强的地方；
5. 哪些关键场面迁移卡在正文里可能没执行到位；
6. 我需要 reviewer 特别看的地方。
```

作者问题清单不能替代 reviewer。

## 13. 8. 干净 reviewer

Reviewer 每次单开临时干净上下文。

### 第一轮

第一轮只读：

```text
最小需求卡；
首批正文。
```

不读作者问题清单，不读主创解释，不读关键场面迁移卡。

第一轮判断：

```text
这是不是一个能看的短剧首批；
有没有明显长剧化、程序化、说明书化；
有没有台词水、人物冷、视听弱、卡点说完；
是否不看源本也站得住。
```

### 第二轮

第二轮再读：

```text
主创读本笔记；
洗稿方案；
关键场面迁移卡；
首批写作简报；
必要源本片段或源本节点。
```

第二轮判断：

```text
有没有洗飞源本有效性；
有没有改名复刻；
有没有把源本强戏洗成弱场面；
有没有把钩子、卡点、爽点周期、关系拉扯、节奏惯性洗丢；
关键场面迁移卡有没有真的进入正文；
失败如果存在，应该回到哪一步。
```

Reviewer 输出只允许：

```text
pass / revise / block
```

如果是 revise，必须说明失败层：

```text
需求没对齐；
源本没读透；
洗稿方案错；
关键场面迁移卡缺失或选错；
写作简报传递断；
正文执行弱；
局部台词/场面可修；
状态连续性断。
```

## 14. 9. 主控判定与版本对比

主控必须看：

```text
manifest；
正文；
作者问题清单；
reviewer 第一轮；
reviewer 第二轮；
和 V3 / V5 / 上一版本的对比。
```

主控只能给四种结论：

```text
可给用户/导演看；
局部 revise；
回上游重做方案；
暂停，不再修这个样本。
```

如果问题只在 1-2 个局部场面，才做局部 revise。

如果问题是整体不像剧本、整体程序化、整体丢源本追看冲动、整体又像源本剧情链，不允许继续在正文上打补丁，必须回到洗稿方案或关键场面迁移卡。

只要声称“按 v8 完整跑”，必须先建立 `workflow_run_manifest`。

manifest 只证明执行完整，不证明文本合格。

结构：

```text

输出范围：
```

## 16. 当前样本验证要求

下一次用本 workflow 跑飞书三星样本时：

```text
源本：[修改版] 黑手党少主痛哭求原谅_前妻她杀疯了
新壳：海外豪门商战 / 家族企业权力斗争
目标市场：海外短剧
集数：不变
输出范围：首批 1-10 集
改写力度：中度偏重
正文默认：中文动作/场景描述 + 英文对白
```

跑完后必须对比：

```text
V3：基础底线，主线清楚但同构高；
V5：当前最值得参考的场面迁移基准；
V17：v7 轻流程跑通但 delivery revise 的上一版；
V18：本次 v8 新产物。
```

对比不能只看旧问题。必须整体看：

- 源本有效性；
- 同构风险；
- 新壳是否自然长剧情；
- 关键场面是否有动作链；
- 关系拉扯；
- 钩子和卡点；
- 情绪和爽点递进；
- 台词功能；
- page-to-screen 剧本感；
- 是否过度程序化；
- 是否比 V5 更值得继续。
