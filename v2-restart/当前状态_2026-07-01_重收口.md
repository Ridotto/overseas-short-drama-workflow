## 3. 当前可信事实

### 3.1 V3 为什么还有价值

V3 不是最终答案，但它有价值：

- 主线清楚；
- 新壳基本站得住；
- 源本追看功能没有明显洗飞；
- 写作前稿里有源本有效性、留存骨架、情绪爽点、主线期待、人物压力、节点拆解、变量迁移、同构红线、新壳承载判断、阶段骨架、分集功能表、关键场景草案。

V3 的问题：

- 还不够短剧强；
- 商战机制有点解释化；
- 第 7-8 集仍有公开证据揭露感；
- 不是稳定流程证明。

### 3.2 V5 为什么是最重要正向证据

V5 是按 `workflow_spec_v2.md` 跑出来的。

V5 的价值不是“它细节更刺激”，而是它证明了一条链有效：

```text
源本拆文包
-> 洗稿边界包
-> beat sheet / outline
-> 新壳场面迁移卡
-> Episode function map
-> Scene packet
-> Screenplay draft
-> 作者自检与返修路由
-> Story memory
```

V5 最关键进步：

```text
writer 输入从“信息点 / 证据点 / 功能点”
变成了“场景目的 / 动作链 / 权力变化 / 可见后果 / 禁止提前消费”。
```

这就是第 7-8 集为什么从“听证会”变成了“继承墙、医疗授权、基金冻结、认证失败、胸针被摘”的原因。

### 3.3 V18 为什么不能继续当方向

V18 证明 v8 有一半方向对：

```text
关键场面迁移卡不能删。
```

但 V18 同时证明 v8 不够：

```text
只恢复迁移卡的形式不够。
迁移卡如果选错场面，正文会稳定执行错误。
```

V18 的典型失败：

- E1 写回厨房；
- E1 写回孕检单 / 吞纸；
- E2 写回车祸；
- E2 写回火光弃救；
- E7-E8 仍偏合规听证。

所以不能再从 v8 小修。
