# PRD v10 vs V3 / V8 / V9 版本对比

比较对象：

1. 用户指定基线：`PRD_v3_豪门商战版_首批1-10集.md`
2. 重要上一合格参照：`PRD_v8_豪门商战版_首批1-10集.md`
3. 直接上一轮：`PRD_v9_豪门商战版_首批1-10集.md`
4. 本版：`PRD_v10_豪门商战版_首批1-10集.md`

结论：V10 作者侧优于 V9 的局部症状，尤其 Step 8 高压 scene packet、第 7-8 集人物反扑、第 10 集追问；但 V10 也比 V9 更复杂，必须经独立 reviewer / challenge 判断是否“更好看”而不只是“更完整”。

## 1. 本版 vs 用户指定基线 V3

### 源本有效性

V3 保留了源本情绪链，但桥段外形仍接近源本：假孕/继承人、事故二选一、亲子/信托、旧物崩溃、公开跪求。  
V10 保留同一体验功能：

- 第 2 集仍有男主亲手拒救导致失子；
- 第 4-5 集仍有女主高位回归和女配失权；
- 第 7-8 集仍有当众拆女配、男主护错、男主同罪；
- 第 9-10 集仍有旧物火葬场和公开认罪但不原谅。

判断：V10 没有比 V3 洗飞。

### 同构风险

V10 继续扣除 V3 / 源本高辨识外形：

- 不用车祸、火海、事故二选一；
- 不用假孕、亲子鉴定、当场验血；
- 不用仓库保险箱、小毛衣、雨夜下跪；
- 不用枪口、自杀测试。

第 7-10 集新增的 `medical grants / emergency trust access / federal review` 是豪门商战壳下的新反扑载体，不是源本桥段复刻。

判断：V10 同构风险明显低于 V3。

## 2. 本版 vs V8

### V10 进步

| 维度 | V8 | V10 |
|---|---|---|
| 第 7 集 | Chloe 扫信托台失败，Adrian 担保 | Chloe 先失败再用医疗 grants 做人质，逼 Serena 流血救场，反扑更主动 |
| 第 8 集 | CEO key 和旧语音成立 | 增加 Chloe 反咬 Serena、安保抓腕、Adrian 亲手拦 Chloe，人物动作更强 |
| 第 10 集 | 直播认罪 + 不原谅，闭环较完整 | 保留认罪和不原谅，但调查员转向 Serena，追看问题更尖 |
| Step 8 | V8 作者侧通过，但 packet 仍偏薄 | V10 对 E7-E10 明确补高压 scene packet 和 Script Blocks |

### V10 退步 / 风险

- V10 比 V8 机制更密，尤其 `emergency signature / trust access / federal review`。
- V8 的第 10 集更干净，V10 的第 10 集更悬，但也更可能稀释火葬场爽点。

判断：V10 在“Chloe 反扑”和“第 11 集追问”上优于 V8；读感是否优于 V8 需 reviewer 判断。

| 风险 | 说明 | 是否阻断作者侧 |

判断：V10 是对 V9 共同阻塞点的实质局部修稿，不只是台词润色。

## 4. 本版进步

- Step 8 不再只靠最终正文倒推，已有 E7-E10 写作前高压包。
- 第 7 集从“Chloe 扫描失败”升级为“Chloe 拿医疗款做盾，Serena 被迫流血救场”。
- 第 8 集从“系统播放旧语音”升级为“Chloe 反咬 + 安保抓腕 + Adrian 主动插 key”。
- Chloe 后半段有有效反扑，不再一崩到底。
- 第 10 集留下更尖锐第 11 集追问：Serena 是否会因为救人签名被第二次送进 referral。

## 5. 本版退步

- 商战法律机制比 V9 更重。
- 第 10 集没有 V9 那么干净完整，满足感可能降低。
- V10 仍沿用 V8/V9 大骨架，不是完全新题材重开。

## 6. 是否允许进入用户 review

作者侧判断：暂不建议直接进入用户/导演 review。

允许进入：独立 reviewer / 6-agent challenge。

理由：

- V10 已完成主创侧修稿和作者自检；
- 但 `workflow_spec_v2.md` Step 14 仍未执行；
- V10 的关键风险正是外部读者能否顺畅理解第 7-10 集新增机制，必须由干净 reviewer 判断。

## 7. 下一步

1. 用 V10 正文先开干净 reviewer 第一轮，不读作者质量门和版本对比。
2. 第一轮后再给写作前工作稿，核查 Step 8 carry-through。
3. 如果 reviewer 仍为 `revise`，优先判断问题是“机制复杂度”还是“人物反扑可信度”，不要先小修台词。
4. 如果 6-agent challenge 无 block 且 revise 非阻塞，再由主控决定是否进入用户/导演 review。
