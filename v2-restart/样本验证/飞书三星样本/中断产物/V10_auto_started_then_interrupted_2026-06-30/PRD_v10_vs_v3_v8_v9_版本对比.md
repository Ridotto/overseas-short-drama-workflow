# PRD v9 vs V3 / V8 版本对比

比较对象：

1. 用户指定基线：`PRD_v3_豪门商战版_首批1-10集.md`
2. 上一轮交付版：`PRD_v8_豪门商战版_首批1-10集.md`
3. 本版：`PRD_v9_豪门商战版_首批1-10集.md`

结论：V9 作者侧允许进入独立 reviewer；不建议直接给导演。V9 相比 V3 明显降低改名复刻风险；相比 V8 是“同骨架的清洁重跑 + 局部增强”，不是完全重开的一版。

## 1. 本版 vs 用户指定基线 V3

### 源本有效性

V3 保留了源本主要情绪链：被替代、事故失去孩子、五年后归来、股东大会打脸、旧物火葬场、公开认罪。  
V9 同样保留这些体验功能，并进一步把“源本强度”从分析带到了正文：

- 第 2 集不是只说签 referral，而是写出流血、医疗例外按钮、拒医、拘押车和监护仪长音；
- 第 7 集不是单纯展示证据，而是逼 Chloe 亲手扫信托台失败，逼 Adrian 亲手按担保键；
- 第 8 集不是靠女配喊出男主有罪，而是 Adrian 自己插 CEO key，听见自己的旧语音；
- 第 9 集火葬场有胎心音、腕带、超声和空发布会大厅。

判断：V9 没有比 V3 洗飞。

### 同构风险

V3 仍接近源本外形：

- Serena/白月光怀孕或继承人骗局接近源本假孕；
- Hawthorne Tower 安全档案事故接近源本“危机二选一”；
- 亲子/信托父系确认接近亲子鉴定；
- 第 9 集封存档案层旧物 + 暴雨崩溃接近仓库保险箱 + 暴雨跪地。

V9 扣除这些外形：

- 不用假孕、亲子、当场验血；
- 不用车祸/爆炸/事故救人二选一；
- 不用仓库保险箱、小毛衣、尾戒；
- 不用枪口自杀测试；
- 第 9 集取消雨夜跪地，改为空发布会大厅和临时铭牌。

判断：V9 同构风险显著低于 V3。

### 短剧场面

V3 第 7-8 集偏“证据出示 -> 众人震惊 -> 对峙”，场面动作链不足。  
V9 第 7-8 集把证据变成操作：

- Chloe 的手必须扫过信托台；
- 胸针被机械臂吸走；
- Adrian 的手必须按红色担保键；
- Adrian 的 CEO key 必须插入主机；
- 权限卡当场变灰。

判断：V9 的动作链和当众后果强于 V3。

## 2. 本版 vs 上一轮交付版 V8

### 共同点

V9 和 V8 共享核心骨架：

- 女主名 Serena / 新身份 Aria North；
- Blackwell、Atlas、federal referral、no medical exception；
- 救市 token / CEO key / 联邦托管箱 / 公开认罪；
- 第 10 集“听见了，但不原谅”。

因此 V9 不是“全新题材重构”，而是按同一方向进行 clean workflow 重跑。

### V9 进步

| 维度 | V8 | V9 |
|---|---|---|
| 第 1 集危险入口 | referral 发送存在，但更像后台文件动作 | 明确把 “Chloe 的 SEND” 作为集尾钩子，和徽章撕裂、超声被踩连在一起 |
| 第 2 集身体冲击 | 已有拒医、手铐、拘押车 | 更强调血滴、调查员当场询问医疗例外、Adrian 手指按 `DENY`，源本“转身不救”的动作等效更清楚 |
| 第 7 集动作后果 | 信托台、胸针摘除、账户冻结 | 加强为信托台机械回收胸针、授权章被摘、红色责任书打印、席位灯锁黄，后果更集中 |
| 第 8 集男主同罪 | CEO key 和旧语音已成立 | 补强“Adrian 第一次扣住 Chloe 手腕”和“这个键是你按的”，让他从护错转向自证更可见 |
| 第 9 集火葬场 | 托管箱 + 暴雨纪念墙 | 改为空发布会大厅，避免源本雨夜外形；让他把 CEO 徽章扔进托管箱，和第 1/2 集发布会闭环 |
| carry-through 证据 | V8 有 manifest 和质量门 | V9 在写作前工作稿、质量门里显式列出源本节点 -> episode map -> scene packet -> 正文锚点 |

判断：V9 在“源本强度进入正文前 packet 和正文”上比 V8 更显式；第 2、7、8、9 集动作承载略强。

### V9 退步 / 风险

| 风险 | 说明 | 是否阻断 |
|---|---|---|
| 与 V8 共享骨架 | 对读过 V8 的人来说，V9 不是新故事，而是同方案的打磨。 | 不阻断本轮验证，但必须如实标注。 |
| 商战机制仍密 | referral、rescue token、trust hall、CEO key 仍有理解成本。 | 不阻断，需 reviewer 看是否仍冷。 |
| 第 3 集成长仍压缩 | Aria North 的五年成长仍只用几个场面交代。 | 不阻断首批。 |
| Marcus 仍偏工具人 | 主要承担救援和见证，利益动机还没真正动。 | 不阻断首批，后续必须补。 |
| V9 文本密度比 V8 更直给 | V9 为了验证动作链，部分场面更硬、更直接；可能牺牲一点 V8 的整洁感。 | 不阻断，需 reviewer 判断读感。 |

## 3. 本版进步

- 相比 V3，去掉了假孕、亲子、车祸、保险箱、雨夜下跪、枪口测试等高辨识源本外形。
- 第 2 集把“男主选择”落成可见动作：按 `DENY`。
- 第 7-8 集把证据转成操作：扫台、担保、插 key、权限卡变灰。
- 第 9 集把火葬场和发布会空间闭环，降低源本雨夜复刻。
- 第 10 集保留首批小闭环：男主公开认罪，女主听见但不原谅。
- 质量门明确列出 carry-through 证据，便于主控和 reviewer 查漏。

## 4. 本版退步

- 不是全新重构，对“继续沿 V8 方向能否稳定变好”的验证价值高，对“换一个完全不同豪门商战事件载体”的验证价值有限。
- 仍有商战机制成本，特别是第 7-8 集。
- 由于没有独立 reviewer，本文件只能代表作者侧对比，不代表最终验收。

## 5. 是否允许进入用户 review

作者侧判断：暂不建议直接进入用户/导演 review。

允许进入：独立 reviewer 分层审稿。

理由：

- V9 完整产出了工作稿、正文、rewrite/dialogue polish、作者质量门、story memory 和版本对比；
- 但当前本 agent 是主创测试 agent，没有独立 reviewer；
- `workflow_spec_v2.md` 要求独立 reviewer 先读正文，再读定位材料，判断是否还有洗飞、改名复刻、短剧降级、会议戏和 carry-through 断裂。

## 6. 下一步

1. 给干净 reviewer 的第一段输入只包含：用户需求、PRD v4 验收标准、V9 首批正文。
2. Reviewer 第一轮不要读作者质量门、rewrite report、版本对比和旧版正文。
3. 第一轮后再给写作前工作稿，让 reviewer 判断源本强度是否真的进入正文。
4. 若 reviewer 结论为 `revise`，优先回修第 2、7、8、9 集的 scene 层，不先做对白小修。
