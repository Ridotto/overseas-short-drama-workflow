### Root Symptoms

1. 如果商战元素只作为术语出现，观众会看到“系统在宣布结果”，不是人物在撕咬。
2. Chloe 必须有反扑，否则 Serena 赢太顺；但反扑不能改变主线刀口，Adrian 的选择必须是火葬场核心。
3. 第 10 集需要留追问，但追问必须有前文轻锚点，否则像硬塞新悬念。

### Prioritized Actions

1. 先回 Step 8：修锚点证据、E9 完整 scene packet、继承字段。
2. 再改 E7/E8 正文：删机制解释，把每个机制结果绑到身体/空间/当众后果。
3. 再改 E2/E3/E4/E10：埋 Marcus 第一笔钱轻伏笔。
4. 最后做台词压缩：保留攻击、逼迫、选择、判决功能，删口号句和解释句。

### Classification

- must-restructure：Step 8 scene packet 证据链。
- must-restructure：E7/E8 机制说明转动作后果。
- must-restructure：Marcus 第一笔钱伏笔提前。
- suggested：前 1-6 集保留骨架，只轻修台词和伏笔。
- observable-risk：V11 仍使用 referral、CEO key、trusteeship 等商战元素，需 reviewer 判断是否足够直观。

## Dialogue Polish

### Target Lines

| 位置 | V10 问题 | V11 处理 |
|---|---|---|
| E7 Chloe 扫描失败 | “rescue token / holder not found” 容易像系统说明 | 正文只保留 `HOLDER NOT FOUND` 屏幕短字，冲突落在胸针被吸走和 Chloe 反咬 |
| E7 Serena 逼 Adrian | V10 “开旧 referral 还我清白 / 替她担保”仍偏流程 | 改为“开旧案，或者替她按”，更短，压力落到红键 |
| E8 验证真相 | V10 使用 root-key / CEO key 仍偏机制 | 保留必要名词，但让水杯碎、手流血、Adrian 亲手插 key 承担冲突 |
| E9 火葬场 | V10 已有效，V11 加强空大厅准备认罪 | 新增“明天，我把我的名字也放进去”，把崩溃转为行动 |
| E10 Marcus 钩子 | V10 “第一笔钱是什么”偏突然 | E2/E3/E4 已埋 Arden bridge，E10 改为“第一笔钱，谁签的？”更尖、更具体 |

### Subtext Benchmarks

- Serena：表层冷静，潜台词是“不允许你们再用公司、程序或补偿挡住孩子的死”。
- Adrian：表层认罪，潜台词是“我不再求她帮我减轻后果”。
- Chloe：表层反咬，潜台词是“我可以输，但不能让 Serena 干净地赢”。
- Marcus：表层保护 Serena，潜台词是“我希望她在第一笔钱被问到前收手”。

### Voice Adjustments

- Serena：句子短，像裁决，不解释机制。
- Adrian：破碎但具体，不再重复“原谅我”，只承认事实。
- Chloe：少哭诉，多攻击和切换战场。
- Marcus：少给旁白，更多用停顿和打断暴露遮掩。

### Rhythm Notes

第 7 集：转信托 -> Chloe 手按不上去 -> 胸针被吸走 -> 红封反咬 -> Serena 被压腕 -> Adrian 按红键。  
第 8 集：root-key 白灯 -> Chloe 撞碎水杯 -> Serena 手流血 -> Adrian 插 key -> 旧语音 -> 胸牌变灰。  
第 9 集：授权等待 -> 开箱手抖 -> 旧物 -> 胎心 -> 长音 -> 空大厅铭牌。  
第 10 集：Marcus 催收手 -> Adrian 下跪 -> 直播认罪 -> 手铐 -> Chloe 后手 -> Arden bridge 钩子。
