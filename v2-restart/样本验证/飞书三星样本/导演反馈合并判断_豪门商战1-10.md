# 导演反馈合并判断：豪门商战版 1-10 集

> 日期：2026-06-29
> 反馈来源：导演
> 原话：现在就是改个名字，其他东西都是一模一样的嘛，这个就太明显了。

## 1. 结论

导演反馈成立。

这版不是“洗飞”，而是另一个问题：

**源本有效性迁移到了，但事件载体、场面顺序和关键桥段太同构，导致成品看起来像改名版。**

这说明当前稿不能直接算通过。

## 2. 和 reviewer 意见的关系

独立 reviewer 已经提前指出了同类问题：

| reviewer 问题 | 导演反馈对应 |
|---|---|
| 机械复刻感还在 | 导演直接感知为“改个名字” |
| 商战壳像商业词替换 | 导演感知为“其他东西一模一样” |
| 第 10 集跳楼测试太狠 | 属于后续具体修点，但不是导演这次主反馈 |
| 7-8 集证据链太顺 | 属于同构桥段之一 |

主创自检当时也提过“车祸、假孕、亲子鉴定、雨中崩溃高度贴源本”，但严重度判断过低。

## 3. 问题不在什么地方

问题不在：

- 人名没换够；
- 场景名不够商业；
- 台词需要润色；
- 需要多加几个商战词；
- 需要把车祸改成另一种车祸。

如果只做这些，仍然会像源本。

## 4. 真正的问题

当前稿复刻的是整条事件链：

```text
假孕白月光登堂入室
-> 男主逼离婚
-> 女主吞孕检单
-> 车祸二选一
-> 女主流产
-> 五年后强势归来
-> 公开打脸
-> 亲子鉴定
-> 男主翻旧物崩溃
-> 暴雨下跪求原谅
```

这些节点各自有功能，但不能整串原样保留。

应该保留的是功能链：

```text
女主被替代/误判
-> 男主做出错误选择
-> 女主承受不可逆损失
-> 女主以隐藏能力重回权力场
-> 白月光谎言被拆
-> 男主知道自己亲手造成不可挽回后果
-> 男主权力坍塌并求赎罪
```

## 5. 修订方向

下一版不应该在原稿上局部改名，而应该从变量迁移层重做 1-10 集。

建议保留：

- 女主被误判和替代；
- 男主错误选择；
- 不可逆损失；
- 女主隐藏能力；
- 五年后权力反转；
- 白月光骗局拆穿；
- 男主崩溃赎罪。

必须换掉或大幅改造：

| 当前桥段 | 问题 | 改法方向 |
|---|---|---|
| 假孕白月光登堂入室 | 太像源本启动器 | 可换成白月光带着“继承人信托/项目救命合同/伪造救命恩人身份”回归 |
| 吞孕检单 | 标志性桥段太像 | 可换成女主销毁/吞下/藏起的不是孕检单，而是能救集团或证明身份的核心证据 |
| 车祸二选一 | 源本核心事件几乎照搬 | 可换成董事会危机、工厂事故、医疗抢救名额、背锅入狱、核心专利事故 |
| 流产双胎 | 情绪强但高度同构 | 可保留怀孕，也可换成父亲死亡、女主失去生育能力、被迫背锅入狱、核心事业被毁 |
| 公开亲子鉴定 | 源本打脸形式照搬 | 可换成审计链、信托受益人、伪造股权、项目事故真凶、救命恩人身份反转 |
| 旧仓库孕检单小毛衣 | 太像源本旧物崩溃 | 可换成董事会档案、旧录音、医院抢救记录、女主替他背锅的签字文件 |
| 暴雨下跪 | 追妻火葬场常规像素 | 可换成公开发布会认罪、辞任 CEO、交出投票权、跪在基金会纪念墙前 |
| 跳楼测试 | 女主正当性风险 | 改成商战化人格摧毁测试，不诱导自杀 |

## 6. 对 workflow 的反推

这次验证说明：

1. “没洗飞源本有效性”不等于“新本合格”。
2. 必须增加一个独立判断：**相似度 / 同构度是否过高**。
3. 如果外部读者能一眼看出“只是改名”，应视为硬失败或至少重警告，不应继续写后段。
4. 中等换壳也不能整串继承源本标志性桥段；应该继承功能链，重建事件载体。

## 7. 下一步建议

不要继续 11-37 集。

下一步应该是重做 1-10 集修订方案：

1. 先不写正文；
2. 先列“保留功能链 vs 必须替换的桥段链”；
3. 重新设计豪门商战版 1-10 集事件表；
4. 通过后再写正文。

这一步可以由主创 agent 完成，然后再让干净 reviewer 审事件表，别等整篇正文写完才发现太像。
