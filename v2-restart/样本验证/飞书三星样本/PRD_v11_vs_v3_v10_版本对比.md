# PRD v11 vs V3 / V10 版本对比

比较对象：

1. 用户指定基线：`PRD_v3_豪门商战版_首批1-10集.md`
2. protocol-fail 内容参考：`PRD_v10_豪门商战版_首批1-10集.md`
3. 本版：`PRD_v11_豪门商战版_首批1-10集.md`

结论：V11 不低于 V3，且作者侧判断优于 V10；允许进入 6-agent challenge，不建议跳过 reviewer 直接交付用户 / 导演 review。

## 1. 本版 vs 用户指定基线 V3

### 源本有效性

V3 已经保留了被替代、事故失子、五年后归来、股东大会拆女配、旧物火葬场、公开认罪等主要情绪链。  
V11 继续保留这些体验，并把源本高辨识外形扣得更清楚：

- 源本 E2 车祸 / 火海 -> V11 E2 referral + 医疗例外 `DENY` + 拘押车流血；
- 源本 E7 黑医 / 床照 -> V11 E7 信托台手掌扫描失败 + 胸针被吸走 + 红色担保键；
- 源本 E8 亲子鉴定 / 当场验血 -> V11 E8 root-key 被破坏 + Adrian 自己插 CEO key + 旧语音；
- 源本 E9 仓库保险箱 / 小毛衣 / 雨夜 -> V11 E9 联邦托管箱 + 胎心 + 空发布会大厅；
- 源本 E10 枪口测试 -> V11 E10 直播刑责认罪 + 手铐 + trusteeship 后手。

判断：V11 没有比 V3 洗飞，且同构风险低于 V3。

### 短剧场面

V3 的第 7-8 集更接近股东大会证据听证，Serena / Claire 的赢法仍偏文件压制。  
V11 把关键结果绑定到可拍动作：

- Chloe 的胸针被机械臂吸走；
- Serena 的旧拘束疤被安保按在见证栏上；
- Adrian 亲手按红色担保键；
- root-key 掉进水里，Serena 手掌被玻璃划破；
- Adrian 自己插 CEO key；
- CEO 胸牌变灰；
- 托管箱铁皮割开 Adrian 掌心；
- 手铐在直播中扣上。

判断：V11 的短剧场面强于 V3。

### 台词与读感

V3 台词更接近传统豪门剧冲突，情绪强，但部分桥段解释性仍多。  
V11 台词更短，关键句承担选择压力：

- `开旧案，或者替她按。`
- `你的 key。她的谎。选一个。`
- `别拿公司挡在孩子前面。`
- `听见了。但我没说原谅。`

判断：V11 不低于 V3，且更符合当前 workflow 对短剧化表达的要求。

## 2. 本版 vs V10

### V10 可用基础

V10 的方向和内容链条可参考：

- Serena / Aria North；
- Blackwell、Atlas、referral、CEO key、托管箱；
- E7/E8 的 Chloe 反扑；
- E10 的公开认罪、trusteeship、Marcus 第一笔钱钩子。

但 V10 是 protocol-fail，不能作为合格执行样本；内容上也有共同 revise 点：

1. Step 8 pass 范围过满，E1-E6 / E9 证据不足；
2. 第 7-8 集机制味仍重；
3. Chloe 反扑要有效但不能抢 Adrian 火葬场；
4. Marcus 第一笔钱偏突然；
5. Serena 赢要更难、更有代价。

### V11 进步

| 维度 | V10 | V11 |
|---|---|---|
| 运行协议 | manifest 时间晚于主要产物，protocol-fail | manifest-first，且 Step 0-13 持续回写 |
| Step 8 | E7/E8/E10 较强，但 E1-E6 复用和 E9 完整度被 challenge 指出不足 | E1-E6 明列 V9/V10 锚点；E9 完整 Scene 1-3；E7/E8/E10 第二场标注继承 |
| 第 7 集 | 有反咬和压腕，但仍使用较多机制词 | 更集中在“手按不上去、胸针被吸走、旧疤被压、红键被按” |
| 第 8 集 | 有 root-key 破坏和 CEO key，但机制说明仍偏多 | 冲突更物理化：水杯碎、手流血、Adrian 被迫插 key、胸牌变灰 |
| 第 9 集 | 正文有效，但工作稿 scene packet 偏简略 | 工作稿和正文都强化托管箱、胎心、空大厅铭牌 |
| Marcus 钩子 | E10 才明显出现 `Arden Holdings` | E2/E3 出现 Arden bridge，E4 banker 提到 bridge 被 Marcus 打断，E10 再爆 |
| Chloe 反扑 | E7/E8/E10 都有，可能分散注意力 | 反扑保留，但刀口回到 Adrian：按红键、插 key、被手铐 |

判断：V11 作者侧优于 V10。

## 3. 本版进步

- 修复 V10 最大运行问题：manifest-first 被证明。
- 修复 Step 8 证据问题：E1-E6 复用有锚点，E9 完整 scene packet，继承字段明确。
- 第 7-8 集减少机制说明，更多依靠动作、身体、空间和当众后果。
- Serena 赢得更难：E7 被按回嫌疑人位，E10 拿到真相但失去即时接管。
- Chloe 有反扑但不替 Adrian 抹罪。
- Marcus 第一笔钱提前埋伏笔，第 10 集钩子更可追。

## 4. 本版退步 / 新风险

- V11 删弱了一些 V10 的商战机制解释，可能让部分商业规则更依赖观众从动作中理解。
- Marcus 伏笔比 V10 早，但仍很轻，reviewer 可能要求再加一个无解释锚点。
- Chloe 的 E10 后手仍可能被认为反派便利，需要第 11 集解释她如何触发 trusteeship。
- Serena 的冷处理是有意选择，但可能被部分 reviewer 认为情绪外化仍不够。

## 5. 是否允许进入用户 review

不建议直接进入用户 / 导演 review。

允许进入：6-agent challenge / 独立 reviewer。

理由：

- V11 主创侧完成写作前工作稿、正文、rewrite/dialogue polish、作者质量门、story memory 和版本对比；
- manifest-first 和持续回写已修；
- 但 Step 14 独立 reviewer 尚未执行；
- 剩余问题属于外部干净审稿应判断的质量风险。

## 6. 下一步

1. 主控另开干净 reviewer / 6-agent challenge。
2. 第一轮 reviewer 只读：用户需求、PRD v4 验收标准、V11 首批正文。
3. 第一轮后再给 V11 写作前工作稿，检查源本强度是否进入 scene packet 和正文。
4. 若仍为 `revise`，优先回修第 7-10 集 scene 层，不扩 PRD 或 workflow。
