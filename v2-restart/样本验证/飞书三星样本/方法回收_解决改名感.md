# 方法回收：解决“只是改名”的问题

> 日期：2026-06-29
> 背景：豪门商战版 1-10 集被导演反馈“现在就是改个名字，其他东西都是一模一样的”。

## 结论

这次问题不在对白润色，也不在短剧标准库规则不够，而在正文前缺了一步：

**先证明新本的具体事件载体已经重建，再写剧本。**

源本有效性要保留，但源本的桥段外形、事件顺序、道具证据、人物进出场方式、揭露方式，不能整串搬过去换名。

## 从旧项目回收的可用方法

| 来源 | 可用点 | 当前怎么用 |
|---|---|---|
| `设计提纲_v2.md` / `设计提纲_v3.md` | 分层诊断、改稿单、禁止项清单 | 把“改名感”定位为场景/事件载体层问题，不先修句子 |
| `设计提纲_v4.md` / `设计提纲_v5.md` | beat 卡片、minimum 保留、改动追溯 | 保留功能链，扣除原事件外形，建立原功能到新事件的追溯 |
| `1-short-drama_短剧标准库` | 钩子、爽点、节奏、付费节点的功能指标 | 只作为功能检查：每集有新信息，钩子有兑现，爽点有压抑释放 |
| `2-oh-story_追踪与去AI味` | 情节节点编号、上下文/伏笔/角色状态追踪 | 用节点编号追踪源本功能和新本事件，不靠感觉改写 |
| `3-how-to-make-script_方法论` | 源本 DNA、四层失败诊断、镜头化/可演性检查 | 先判断失败层，再检查每场有没有可见动作改变局面 |
| `4-dramatron_分层生成` | 先人物/情节点/地点/对白分层生成 | 借“先重建场景和 beat，再写对白”，不借公式化生成 |
| `5-shortdrama-pipeline_质检` | 人工审核门槛、shot 不过载、对白不过载 | 只放到后置质检，不用它解决事件同构 |

## 不能继承的旧方向

- 不把短剧标准库的比例、题材模板、开场模板当硬规则。
- 不把旧项目的“贴近对标”照搬过来；贴太近正是这次问题。
- 不把工程化 pipeline 当成当前解法；它解决生产状态，不解决创作同构。
- 不继续扩设计提纲，不再把每个问题都变成新规则。

## 新增前置产物：事件载体迁移表

写正文前先交这张表。

| 字段 | 要回答什么 |
|---|---|
| 集数/节点ID | 对应源本哪一集、哪一拍 |
| 源本叙事功能 | 这一拍让观众继续看的原因是什么 |
| 源本事件载体 | 原本用了什么具体事件、场景、道具、证据、行动 |
| 必须扣除的原形式 | 哪些东西一保留就会像改名 |
| 新壳承载逻辑 | 豪门商战里真正能制造同等压力的机制是什么 |
| 新事件载体 | 新本具体发生什么事 |
| 信息/关系/局势变化 | 这一拍结束后有什么真实变化 |
| 同构风险 | 高/中/低，以及为什么 |

硬标准：

如果“新事件载体”只是把源本名词换成商业名词，这张表不通过，不进入正文写作。

## 下一步用于当前样本

1. 不继续写 11-37 集。
2. 先为豪门商战版 1-10 集补做事件载体迁移表。
3. 表通过后再重写 1-10 集正文。
4. 主创自检先查同构风险。
5. 干净 reviewer 再独立审新稿，不先看作者自检。
