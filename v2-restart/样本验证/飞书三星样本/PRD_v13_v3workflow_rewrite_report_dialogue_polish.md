# PRD v13-v3workflow Rewrite Report + Dialogue Polish

> 对应正文：`PRD_v13_v3workflow_豪门商战版_首批1-10集.md`

## 1. 作者侧结论

状态：ready for reviewer，但不能作者自判 pass。

这版主要验证 v3 workflow 的核心假设：

```text
writer 不读厚审计证明链，只读薄 writer brief；
高压节点先做新场面粗验；
正文只写观众能看到 / 听到的戏；
文件、屏幕、合同降级为辅助道具。
```

## 2. 本版做对的地方

### 2.1 第二集从“机构拒医键”改成“人对人放弃”

V12 的第二集高压本体是：

```text
Medical exception request -> approve / deny -> CEO 按键
```

V13 改成：

```text
Serena 戴铐卡在玻璃通道里流血
Adrian 看到她
安保问先剪谁
Adrian 亲手剪开 Chloe，留下 Serena
火门落下
```

这更接近源本“男主看见女主求救但选择另一个人”的体验，不再把伤害交给系统。

### 2.2 第 7-8 集从证据听证会改成身体危机触发

V12 的第 7-8 集主要风险是 key / broadcast / read-back / property box 这种机制推进。

V13 改成：

```text
儿童医院剪彩
孩子受伤
Adrian 要捐血
Chloe 阻止
孩子当众喊 Daddy Ben
Bennett 失态
Chloe 跪抱求饶
Chloe 反咬 Adrian
```

真相不是报告读出来，而是人物在压力下崩出来。

### 2.3 第十集保留“直播按钮”钩子，但不让按钮成为发动机

第十集的发动机不是按钮本身，而是：

- Adrian 跪在 Serena 面前；
- 他说出“我选择了 Chloe”；
- 他交出戒指 / CEO / 投票权；
- Serena 仍不原谅；
- 直播按钮只作为最后公开惩罚的卡点。

## 3. 作者侧发现的风险

### 3.1 商战辅助道具仍然偏多

正文里仍然出现：

- `ON SCREEN`
- `ON DOCUMENT`
- red federal referral folder
- rescue contract box
- stock ticker
- resignation / voting trust transfer

这些不是本版的场面发动机，但仍可能让读者感觉“商业机制词偏多”。后续如果 reviewer 判机制感仍重，优先删减这些说明性道具，而不是再加身体反应。

### 3.2 第 4-5 集女主回归后的商业压迫还可以更狠

第 4 集的“买下房间”和第 5 集的“压 Chloe 到玻璃前”比 V12 更像戏，但商业后果仍有点概括。可选返修方向：

- 让董事当场离席站到 Serena 身后；
- 让 Chloe 原本准备的 donor photo 被 Serena 当众撤掉；
- 让 Adrian 的座位 / 主位被 Serena 直接夺走。

### 3.3 Marcus 的危险性还不够

Marcus 目前更多是保护者和旁观者。源本 Victor 既救女主，也有占有和交易意味。V13 首批里 Marcus 的阴影较弱，后续可补：

- Marcus 当年救她不是纯善意；
- 他想借 Serena 吞 Blackwell；
- 他会刺激 Adrian 的嫉妒，但不能抢走女主复仇主线。

### 3.4 英文台词整体比 V12 短，但仍有少量解释句

需要重点复查：

- Episode 1 Serena 解释 author seal；
- Episode 3 Marcus 解释新身份；
- Episode 4 Serena 的“一亿”后商业身份揭示；
- Episode 10 Adrian 的完整认罪。

这些句子目前还有功能，但 reviewer 可判断是否过说明。

如果 reviewer 判不通过，建议按以下层级回滚：

| 问题 | 回滚层 |
