# PRD V18 v8workflow vs V3 / V5 / V17 整体对比

## 1. 总结论

V18 不是 pass。

它证明了 v8 的一个判断是对的：恢复 `关键场面迁移卡` 后，正文确实比 V17 更有动作链，前置材料也更能把分析传到正文。

但 V18 同时暴露了一个更关键的问题：

```text
关键场面迁移卡如果选错事件载体，
workflow 会更稳定地把错误写进正文。
```

V18 的最大问题不是“没执行迁移卡”，而是迁移卡本身在 E1/E2 把源本高辨识外形带回来了：

- E1 回到厨房；
- E1 回到孕检单；
- E1 回到吞孕检单；
- E2 回到车祸 / 两车遇险 / 男主先救女配；
- E2 回到火光吞车。

这比 V17 的“换成玻璃中庭事故”更接近源本，也更接近导演最早骂过的“改个名字”风险。

所以 V18 的状态是：

```text
workflow 连接层有进步；
产物质量不如 V5；
同构风险局部高于 V17；
整体只能 delivery_revise。
```

---

## 2. V18 相比 V3

### V18 更好的地方

V18 的剧本文字比 V3 更像 page-to-screen 正文。

V3 很多段落还是偏中文短剧小说式剧本，场面能看，但动作与对白的转换没那么统一。V18 至少大部分段落写成了：

```text
场景 -> 动作 -> 英文对白 -> 动作后果
```

V18 的 E4-E5 也比 V3 更有即时场面动作：

- Celeste 胸针被摘；
- 保镖换边；
- 黑卡被剪；
- 酒店安保和合规审计员现场进入；
- 套房门禁失效。

这些不是纯文件播报。

### V18 更差的地方

V3 的问题是“整体像源本功能链换商战词”，但 V3 在 E1/E2 至少没有把源本最标志性的外形全部拿回来：

- V3 E1 是玻璃温室，不是厨房；
- V3 是超声影像被碎纸机吞，不是孕检单吞咽；
- V3 E2 是低氧档案库，不是车祸。

V18 反而在 E1/E2 回到了更接近源本的外形：

- `INT. WHITMORE ESTATE - KITCHEN`
- `Six weeks pregnant` 孕检单；
- Ava 把孕检单放进嘴里咽下；
- 冷链货车撞轿车；
- 两边同时遇险；
- Adrian 抱走 Celeste，Ava 留在火光里。

这会让导演很容易说：这不是又回到原版了吗？

### 对 V3 的判断

V18 在剧本文字形态上强于 V3。

但在最敏感的 E1/E2 同构问题上，V18 不一定优于 V3，甚至更危险。

## 3. V18 相比 V5

V5 仍然是当前更值得参考的版本。

### V5 强在哪里

V5 的最大优点是：它没有只把“证据”变成材料，而是让每个信息点带来现场后果。

典型是 E7-E8：

```text
继承墙刻名
-> 医疗授权拒签
-> 基金冻结
-> 继承认证失败
-> 孩子名字被红色封条盖住
-> 家族胸针被摘
-> 旧切电授权回收
```

这条链条每一步都让角色失去东西。

### V18 不如 V5 的地方

V18 的 E7-E8 虽然也有：

- 直播关不掉；
- 投票牌变黄；
- 基金会付款记录；
- 信托血样复核；
- 董事会要求 recess；

但整体仍然是：

```text
证人说
屏幕显示
信托律师开箱
设备跑结果
Celeste 反咬
```

它有程序后果，但人物对抗不足。

V5 的“Chloe 正要把孩子写进继承墙，结果被现场封掉”更短剧、更可见、更像一场戏。V18 的股东会更像一套证据流程。

### V18 比 V5 好的地方

V18 的 E10 比 V5 更贴豪门商战。

V5 的枪响黑屏很刺激，但类型边界风险大，容易滑回黑帮/犯罪。

V18 用：

- CEO 徽章断裂；
- 投票信托；
- 辞呈；
- 事故证词；
- GO LIVE；
- 男主直播认罪卡在第一个字。

这个方向更干净，也更适合后续继续写追妻火葬场。

### 对 V5 的判断

V18 没有超过 V5。

V18 的最佳价值是证明：v8 的连接层方向有用，但 V5 的场面迁移质量更高。

如果要继续优化 workflow，应该吸收 V5 的“场面迁移卡质量标准”，而不是只吸收“迁移卡这个形式”。

## 4. V18 相比 V17

### V18 更好的地方

V18 比 V17 更明确地把前置分析接到了正文：

- V18 有关键场面迁移卡；
- 正文多数高压场确实按卡执行；
- E4-E5 比 V17 更有具体权力动作；
- E10 比 V17 更集中，直播认罪卡点更清楚。

V17 的问题是写作简报太薄，正文容易变成“功能摘要”。V18 至少把很多功能落成了动作。

### V18 更差的地方

V17 的 E1/E2 虽然有问题，但它至少没有直接用厨房、吞孕检单、车祸。

V17 用的是：

- 发布会后台；
- 超声影像被踩；
- 玻璃中庭坍塌；
- 灭火雾；
- 发布会事故。

这些同构风险低于 V18 的厨房/吞孕检单/车祸。

所以 V18 在“场面更实”上赢 V17，但在“去同构”上局部输 V17。

### 对 V17 的判断

V17 证明轻流程能跑出可看文本，但写作抓手不足。

V18 证明补抓手有用，但抓手本身必须管住事件载体选择。

两者合起来说明：

```text
v7 太薄；
v8 只补形式还不够；
下一步要补的是“迁移卡如何判定载体是否合格”，不是继续加一堆新表。
```

## 5. 对 v8 workflow 的反推

### 成立的部分

v8 的核心方向成立：

```text
洗稿方案 -> 关键场面迁移卡 -> 写作简报 -> 正文
```

这条链比 v7 更能把分析传到正文。

证据：

- V18 正文多数高压场能回到迁移卡；
- reviewer 也确认 KC1、KC2、KC3、KC4、KC5、KC7、KC8 大体进入正文；
- E10 卡点比 V17 更集中。

### 不成立或不足的部分

v8 目前没有足够约束“迁移卡自身的选场能力”。

如果迁移卡写出：

```text
厨房
孕检单
吞孕检单
车祸
两边同时遇险
火光吞车
```

那么正文执行得越好，越像源本。

这不是正文执行弱，而是前置场面迁移卡的 gate 过弱。

### 需要修的不是新流程，而是一个 gate

不要再新增主流程。

只需要在 `关键场面迁移卡` 的通过标准里加一条硬判断：

```text
迁移卡不能只避开源本名词，
也要避开源本高辨识动作形状。
```

如果源本是：

```text
厨房逼离婚 / 吞孕检单 / 车祸二选一 / 火光弃救
```

新迁移卡就不能再写：

```text
厨房 / 孕检单 / 吞下 / 两车事故 / 火光弃救
```

哪怕人物、地点、公司名都换了，也不通过。

## 6. 最终判断

V18 状态：

```text
execution_status: complete
delivery_status: revise
```

V18 不建议给导演。

原因：

1. E1/E2 同构风险太明显；
2. E7-E8 仍偏证据程序，不够失控；
3. 英文对白有少量硬伤；
4. V18 没有超过 V5。

但 V18 有验证价值：

1. v8 证明“关键场面迁移卡”这层不能删；
2. v8 也证明“有迁移卡”不等于“迁移卡选得对”；
3. 下一步如果修 workflow，只修迁移卡的载体合格标准，不要重新扩大主流程。
