# PRD v17-v7workflow vs V3 / V15 / V16 整体对比

## 1. 总结

V17 是一次有效的 v7 跑通样本，但不是可交付 pass。

它相对旧版的主要价值：

```text
更轻；
更像剧本；
没有回到改名复刻；
没有回到 V12 式程序审判机器；
前 3 集和第 9-10 集成稿感较强。
```

它的主要问题：

```text
中段 E4-E8 仍有公开证据处刑的整齐感；
部分英文台词金句化；
E3-E4 认知状态略断；
E10 钩子有效但爆炸性弱于源本。
```

## 2. 对 V3

V3 的问题是导演已经明确指出的“改个名字，其他东西太像”。

V17 相比 V3：

- 事件载体换得更彻底：车祸、黑帮拍卖、废弃仓库、旋转餐厅被换成发布会后台、玻璃中庭、债权拍卖、儿童翼剪彩、旧事故现场、直播发布会；
- 不再整串复刻源本事件链；
- E2 没有用档案室替代强危机，而是用了玻璃中庭坍塌；
- E10 的公开直播交权比 V3 更贴合豪门商战壳。

V17 仍需注意：

- E7-E8 仍然接近“公开放证据打脸”，只是外形不像源本；
- 如果再修，不应回到“多加商业词”，而应让人物失控和当众后果承担更多信息。

## 3. 对 V15

V15 是 v4 草案的 candidate checkpoint，reviewer 给过 pass。

V17 相比 V15：

- 流程明显更轻，强制中间产物少；
- 正文没有那么像执行结果报告；
- E1-E3 的情绪更集中，Claire 的失子和复仇选择更直观；
- E9 旧事故现场比 V15 的部分证据链更有身体感。

V15 相比 V17 的优势：

- V15 的整体链路证据更完整；
- V15 在 reviewer 侧曾经拿到 pass；
- V15 部分场面压力链更稳，V17 中段还会变整齐。

判断：

```text
V17 不证明 v7 已优于 v4；
但 V17 证明 v7 可以用更轻流程跑出不低于基本可看线的文本。
```

## 4. 对 V16

V16 是 v6 草案完整受控验证，结论 `delivery_revise`。

V17 相比 V16：

- 少了大量中间产物，主创注意力更集中在正文；
- 没有明显 key / 权限 / 系统 / 托管箱类机制戏；
- E2 强危机比 V16 更接近源本冲击；
- E3 失子到复仇选择比 V16 更明确；
- E10 仍用 live，但不是纯按钮机制，加入了 “say their names” 的人物羞辱和公开认罪。

V16 相比 V17 的优势：

- V16 的 Source Bible / Anchor / Brief 证据更完整；
- V16 局部场面如 E8/E10 有些设计更精确；
- V17 E4-E8 的打脸结构还有模板感。

判断：

```text
V17 比 V16 更轻、更像剧本；
但 V17 没有完全解决正文执行弱的问题。
```

## 5. 对 v7 workflow 的反推

v7 的方向成立一半：

- 减少中间产物没有导致源本洗飞；
- 主创读本笔记 + 洗稿方案 + 写作简报足以支撑首批正文；
- 干净 reviewer 能正常定位失败层；
- manifest 没有再反向绑回 v2 旧表。

v7 还没证明：

- 它能稳定产出 pass 级稿；
- 它能稳定避免中段证据打脸模板化；
- 它能让正文每段都足够像短剧而不是“好看的剧情概要”。

## 6. 当前结论

```text
V17 状态：execution complete / delivery revise
v7 状态：候选 workflow 跑通，但不能正式化
下一步：不继续无限修；如要继续，只能做一次针对 reviewer 三点的局部 revise，或交给用户/导演判断 v7 方向是否比 V15/V16 更值得保留
```
