# PRD v13-v3workflow vs V3 / V8 / V12 整体对比

> 对比对象：  
> - V3：`PRD_v3_豪门商战版_首批1-10集.md`  
> - V8：`PRD_v8_豪门商战版_首批1-10集.md`  
> - V12：`PRD_v12_豪门商战版_首批1-10集.md`  
> - V13：`PRD_v13_v3workflow_豪门商战版_首批1-10集.md`  
>
> 本报告按整体剧本能力对比，不只盯视听语言、机制戏或前几轮暴露的问题。

## 1. 总结论

V13 是目前这组豪门商战样本里，整体最接近“可用海外短剧首批”的版本。

原因不是它没有问题，而是它同时做到了三件事：

1. 源本 1-10 的有效体验链还在；
2. 不再像 V3 那样容易被导演看成“改名版”；
3. 不再像 V8 / V12 那样被 key、系统、信托台、授权、屏幕、文件带成机制戏。

V13 仍不该直接当最终导演送审版。它还有局部修点：

- 第 7 集亲子爆雷触发偏硬；
- 第 6 / 9 集旧物出现太顺；
- 第 1 集商业术语略密；
- 第 5 集 witness 抓手还可以更实；
- 第 10 集纸面投降可以再少一点，更多压到直播公开自毁。

但这些是局部修稿问题，不是整条 workflow 又跑歪。

## 2. 四版整体定位

| 版本 | 整体定位 | 最大优点 | 最大问题 |
|---|---|---|---|
| V3 | 第一版豪门商战可读稿 | 情绪链完整，容易读懂，源本有效性没丢 | 太像“把黑手党源本换成豪门词”，第二集档案库冲击弱，导演认为像改名 |
| V8 | v2 workflow 跑完整后的相对较好版 | 流程完整，场面比 V3 更有动作，9-10 集有效 | 第 7-8 集信托台 / CEO key / 屏幕 / 权限太重，像公司系统戏 |
| V12 | 局部修补后的机制强化版 | 试图让 Adrian 亲手读 / 亲手按，责任更明确 | 修法仍在机制骨架上加血肉，越修越像“审判机器” |
| V13 | v3 workflow 草案验证版 | 用人对人场面、身体、空间、当众后果承载强戏，整体短剧感最好 | 个别触发点方便，少量商业术语和旧物获取仍需打磨 |

## 3. 按产品目标对比

### 3.1 是否保住源本有效性

| 版本 | 判断 |
|---|---|
| V3 | 保住。被替代、失子、回归、拆谎、男主火葬场都在。 |
| V8 | 保住。并且用新壳补了 referral、拒医、信托等对应物。 |
| V12 | 保住。甚至把“男主亲手造成后果”钉得更硬。 |
| V13 | 保住。体验链完整，且不靠完整审计链解释。 |

结论：四版都不是“洗飞”。真正差异不在是否保留源本功能，而在这些功能被什么样的场面承载。

### 3.2 是否避免改名复刻

| 版本 | 判断 |
|---|---|
| V3 | 最弱。档案库、医疗记录、股东大会、旧物、直播认罪，功能和形状都贴源本太近。 |
| V8 | 明显优于 V3。黑手党外形没了，但换成了较重的企业系统外形。 |
| V12 | 同构更低，但机制外形更重。不是改名复刻，却像机制复刻。 |
| V13 | 最好。没有车祸、亲子鉴定、仓库小毛衣、雨中下跪，也没有 key / 信托台成为主干。 |

结论：V13 在“不是改名版”上当前最好。

### 3.3 新本是否作为短剧自己站得住

| 版本 | 判断 |
|---|---|
| V3 | 情绪站得住，但短剧冲击和新壳新鲜度不够。 |
| V8 | 大段可以站住，7-8 集偏冷偏机制。 |
| V12 | 逻辑很硬，但读感偏程序、偏流程。 |
| V13 | 最像一部能拍的海外短剧首批：羞辱、火场玻璃门、黑衣回归、孩子喊错父亲、直播按钮都能形成可见爆点。 |

## 4. 分段对比

### 4.1 第 1-2 集：开局与不可逆损失

V3：

- 优点：开局清楚，离婚、怀孕、CO2 档案库危机都能理解。
- 问题：第 2 集“安全档案库”对比源本车祸，视觉和情感冲击明显弱；导演已经点过。

V8 / V12：

- 优点：把男主拒医动作前置，责任更直接。
- 问题：`root-key / referral / medical exception / CEO key / DENY` 这些东西容易把情绪压成流程。

V13：

- 优点：第 2 集核心变成“玻璃门内出血的 Serena”和“Adrian 亲手剪开 Chloe、留下 Serena”。这是人对人的选择，不是系统选择。
- 风险：第 1 集商业术语略密，但还没有压过超声被踩、徽章被撕、红 folder 这些可见动作。

判断：V13 最优。

### 4.2 第 3-4 集：失子、重建、回归

V3：

- 私人医院、Marcus、五年后拍卖会都清楚。
- 但回归桥段仍和源本“拍卖会高价压人”较近。

V8 / V12：

- 回归更偏企业权力和资本控制，但机制名词较多。

V13：

- 失子段更干净：醒来摸小腹、电视里 Adrian 护 Chloe、Marcus 给新身份。
- 回归卡点执行得好：黑衣女人入场、不摘墨镜、不急着解释。

判断：V13 和 V3 都顺，但 V13 的“先不解释”更短剧。

### 4.3 第 5-6 集：女主强度与男主真相入口

V3：

- 主要靠查资料、办公室对话、档案线推进。

V8 / V12：

- 用 root-key、CEO office、档案室、托管物承接真相，连续性强但机制重。

V13：

- 第 5 集 Chloe 主动闯入被 Serena 反杀，有动作、有空间、有羞辱。
- 第 6 集男主触碰小腹引发 Serena 身体闪回，再通过伤疤和遗物进入真相，比直接读报告更像戏。
- 风险：遗物盒出现太顺，男主主动追查成本不够。

判断：V13 方向最好，但第 6 集需要小修。

### 4.4 第 7-8 集：骗局崩塌与男主同罪

V3：

- 股东大会 / 证据揭露较清楚，但还是听证会。

V8：

- 第二轮 reviewer 后可 pass，但靠信托台、担保键、CEO key、legal hold、权限卡变灰。
- 能过，但观感仍偏“商业系统执行场”。

V12：

- 进一步把 Adrian 拉进 read-back / confirm，责任更硬。
- 但也把“人戏”推向“人操作机器”。

V13：

- 第 7 集用孩子受伤、喊 Daddy Ben、Bennett 失态引爆，场面更直观。
- 第 8 集 Chloe 跪抱、被踹开、尿失禁、反咬 Adrian，短剧强度更高。
- 风险：Noah 轻伤后立刻输血这个触发点硬，需要改。

判断：V13 整体最好。它的问题是触发逻辑硬，不是场面方向错。

### 4.5 第 9-10 集：火葬场与首批终钩

V3：

- 男主找到旧物、直播认罪，情绪强，但仍贴源本旧物火葬场。

V8：

- 托管箱、胎心音、公开认罪、“听见了但不原谅”有效，是 V8 最稳定部分。

V12：

- 保留直播认罪和不原谅，但仍有 CEO badge、托管箱、文件转让等机制感。

V13：

- 旧发布会现场血痕、医院手环、男主自扇、跪到 press room，情绪比较直观。
- 第 10 集 `Paper is cheap` 很好，直播按钮钩子清楚。
- 风险：resignation / voting trust 纸面投降可以再压缩；旧物盒太完整。

判断：V8 / V13 都有效；V13 更像人戏，V8 更像机制闭环。作为短剧，V13 更好。

## 6. v3 workflow 是否起作用

本轮不是只证明 V13 文本更好，而是能反推 v3 workflow 的一个关键假设成立：

```text
把分析底稿、writer brief、审计底稿拆开后，
writer 不再稳定滑向“证明源本迁移”的机制戏。
```

证据：

- V13 正文里没有 source ref、carry-through、quality gate 语言；
- 高压场面没有用完整证明链推进；
- reviewer 第一轮只读正文给 `creative_pass`，说明正文自己能读；
- reviewer 第二轮读源本审计后给 `source_audit_pass`，说明没有因为去掉证明链而洗飞。

这说明 v3 的方向比继续在 v2 上补规则更健康。

但不能说 v3 已经彻底稳定。现在只能说：

```text
v3 workflow 在这个样本上跑出了一版明显优于 V3 / V8 / V12 的首批稿；
它值得进入下一步验证或局部返修；
还不能证明换别的源本也稳定。
```

## 7. 当前建议

不要推倒重写 V13，也不要继续扩 PRD。

建议下一步只做 V13 局部修稿：

1. 第 7 集：换掉“轻伤输血”触发，让 Noah 喊 Daddy Ben 由惊恐 / 监护人登记 / Chloe 下意识阻止父亲签字触发；
2. 第 6 / 9 集：让 Adrian 追查旧物更费力，不要护士直接给完整白盒；
3. 第 1 集：压低 `author seal / sovereign fund` 理解成本，把商业机制变成观众一眼懂的现场动作；
4. 第 5 集：给 “Next, I buy witnesses” 一个更实的线索；
5. 第 10 集：减少纸面文件，强化直播前公开自毁的不可逆动作。

这些修法都不需要改 PRD，也不需要再加 workflow 步骤。
