# PRD V18 v8workflow 主控判定

## 1. manifest 检查

本轮符合执行协议的前提：

- `workflow_run_manifest_V18_v8workflow.md` 是本轮第一个正式产物；
- Step 0-8 均有文件证据；
- reviewer 是干净两轮审稿；
- 本文件补齐 Step 9 主控判定与版本对比。

本轮可以判为：

```text
execution_status: complete
delivery_status: revise
```

注意：complete 只代表 workflow 跑完，不代表正文合格。

## 2. reviewer 结论

Reviewer 结论：`revise`。

Reviewer 的主要判断：

- V18 能看，不是 block；
- 前 3 集压迫有效；
- E4-E6 的回归与反杀基本站住；
- E10 公开认罪卡点方向对；
- 主要问题在 E7-E8：公开揭谎段仍偏程序化，人物对抗不够强；
- E6 到 E7 的 Adrian 状态有断层；
- 英文对白有局部硬伤和功能化。

我认同 reviewer 的 `revise`，但认为它漏掉了一个更上游的问题：

```text
E1/E2 的同构风险比 reviewer 说得更严重。
```

## 3. 主控新增判断

V18 的关键问题不是“关键场面迁移卡没有执行”。

恰恰相反，正文执行了迁移卡。

问题是迁移卡在 E1/E2 选错了事件载体：

- E1 写回厨房；
- E1 写回孕检单；
- E1 写回吞孕检单；
- E2 写回车祸；
- E2 写回两边同时遇险；
- E2 写回火光弃救。

这些都是源本高辨识动作形状。

所以 V18 暴露的是：

```text
v8 补回“关键场面迁移卡”是对的；
但迁移卡本身必须有更强的同构 gate。
```

否则 workflow 会把错误场面稳定写进正文。

## 4. 对 V18 正文的判断

### 能保留的优点

- E4 胸针被摘、保镖换边、债权控制进入现场，短剧场面有效；
- E5 黑卡被剪、门禁失效、媒体可见，动作和后果比单纯文件强；
- E9 遗物火葬场基本成立；
- E10 直播认罪卡点比枪响更贴豪门商战壳。

### 必须判 revise 的问题

1. E1/E2 同构过高，不能给导演。
2. E7-E8 仍像合规听证，人物撕扯不足。
3. Adrian 从 E6 知道双胎到 E7 主持会议，中间状态没有足够可见的自欺、恐惧或硬压。
4. 英文对白有明显问题，例如 `Who is she doing in my house?`。
5. V18 没有整体超过 V5。

## 5. 对 workflow 的判断

v8 方向部分成立：

```text
v7 太薄；
补回关键场面迁移卡是必要的；
正文确实更容易得到动作链。
```

但 v8 还不能正式化：

```text
关键场面迁移卡缺少“高辨识动作形状”过滤。
```

下一步如果修 workflow，不能继续扩主流程，也不能把所有症状都写成新规则。

只修一处：

```text
关键场面迁移卡的通过标准。
```

把它从：

```text
避开源本事件名、道具名、场景名
```

升级为：

```text
避开源本高辨识动作形状。
```

## 6. 是否继续修 V18

不建议现在继续修 V18 正文。

原因：

- E1/E2 是迁移卡层选错，不是正文局部润色；
- 直接修正文会再次进入“打补丁”；
- 应先修 workflow 中的迁移卡 gate，再决定是否重跑或局部重建 E1/E2、E7/E8。

## 7. 主控结论

```text
V18：execution complete / delivery revise
v8：候选 workflow 有效但不能正式化
下一步：只修关键场面迁移卡的同构 gate，不扩 PRD，不新增主流程
```
