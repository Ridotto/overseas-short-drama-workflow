### 2.2 正文效果过第一轮

第一轮 reviewer 只读需求和正文，给 `creative_pass`。

关键理由：

- 1-10 集付费链清楚；
- Episode 2、6、7、8、10 卡点强；
- 高压场不是流程证明；
- 人物互相逼迫成立；
- 情绪通过 ultrasound、brooch、glass door、wristbands、live button 等动作和物件被看见。

### 2.3 源本迁移过第二轮

第二轮 reviewer 读 Source Bible / Anchor / Story State / Episode Plan / Brief / 正文后，给 `source_audit_pass`。

关键判断：

- 没洗飞：仍是替代、误判、放弃、失子、回归、揭谎、追悔、不赦免；
- 没改名复刻：没有复用黑手党庄园、车祸、夜枭军火、亲子报告、仓库小毛衣、雨中跪地枪口整套外形；
- 没短剧降级：核心高压场由人物选择和身体动作推动；
- 没提前消费：live button 卡住，没有给原谅。

## 3. 仍然存在的问题

这些不是 block，但要记录。

### 3.1 E5 的“动利益”还可以更显性

E5 当前“动人”很强：Chloe 闯 penthouse，被 Serena 压到窗前，说漏 Lydia。

但 Source Retention Anchor 里要求 E5 同时证明女主能动利益和动人。V15 主要把 E5 写成“逼证人线”，商业利益被 E4 承接较多。

如果继续 polish，可在 E5 开头加一个很短的可见后果，比如 Chloe 是因为 Blackwell donor / lender 被 North Capital 冻结才冲上来。

### 3.2 E8 的公开围观感比 V14 稍弱

V15 的 E8 在 private hallway，虽然有记者快门和安保线，但公开处刑感不如 V14 的 ribbon ceremony 全场围观。

如果继续 polish，不需要改结构，只要让记者/董事/医院人员隔着安保线听见 Chloe 那句反咬，增强 Adrian 颜面被挂起。

### 3.3 V15 不是纯文本层面碾压 V14

V15 的流程链路更完整，E6/E9 的主动追查更清楚。

但 V14 的 E8 和 E10 有些地方更干净：

- V14 E8 当众感更强；
- V14 E10 的空 press room + 两部 phone + signet ring 更简洁；
- V15 E10 更公开、更适合接全局后果，但也更拥挤。

所以 V15 应作为“v4 草案完整跑通过的候选 checkpoint”，不是说 V14 的文本价值作废。

## 4. 是否需要回滚

不需要回滚到 Source Bible、Episode Plan 或 Pressure-chain Brief。

如果继续修，只回正文 polish：

1. E5 加一笔 Blackwell 利益被卡的可见后果；
2. E8 抬公开围观感；
3. 不新增规则；
4. 不重跑完整 workflow；
5. 不改 PRD。

## 5. 对 v4 草案的判断

本轮 v4 草案有作用。

真正起作用的不是“更多步骤”，而是两条接线：

```text
Source Retention Anchor
-> Story Operating State
-> Pressure-chain Writer Brief
```

它让正文没有丢源本留存链，也没有回到 V12 的机制审判机器。

但 v4 仍不能直接正式化。原因：

- 只验证了一个样本；
- V15 文本没有明显碾压 V14；
- v4 流程比 v3 更厚，后续需要看它是不是稳定提高质量，而不是只是更完整。

## 6. 主控建议

当前可以把 V15 封为一个 candidate checkpoint。

下一步不要再为了这版继续无限修。

更合理的后续是：

1. 把 V15 和 V14 一起给导演或外部 reviewer 看；
2. 问重点不是“哪句更好”，而是：
   - 哪版更像短剧；
   - 哪版更想看下一集；
   - 哪版不像改名；
   - 哪版更不程序化；
3. 如果导演选 V14 的 E8/E10 优点，可以把它作为 V15 的局部 polish 参考，而不是回滚整个 workflow。
