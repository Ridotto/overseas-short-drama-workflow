# V20 v10 workflow 独立 reviewer round 1

审稿范围：只读产品定义、v10 workflow 审稿要求、V20 正文、manifest。  
未读：源本、写作前工作稿、作者自检、rewrite report、dialogue polish、story memory、版本对比、旧版正文。  
本轮任务：先判断 V20 正文本身像不像短剧、能不能追、有没有明显硬失败。

## 结论

`revise`

V20 正文不是 block。它已经具备短剧首批的基本追看能力：1-3 集压迫和事故强，4-6 集复仇回归有可见权力反转，9-10 集追悔和交权也有画面承载。它不像说明书，也不是纯站桩聊天。

但它还不能直接 pass。主要问题集中在中后段：7-8 集把继承人真假和 Chloe 崩盘压在认证台、屏幕结果、当众口供上，场面更像“系统验证 + 结果播报”，人物被迫行动不够；8 集又把 Serena 事故、流产、Adrian 选择等真相一次性由 Chloe 说穿，吃掉了后续火葬场调查的悬念。5、7、8、10 集也反复使用权限冻结、屏幕通知、文件提交来解决冲突，商业壳的机制感仍偏重。

## 正文效果判断

### 能追，但不是全段稳定

1-3 集追看力成立：

- Ep1 有明确的怀孕秘密、假孕入侵、离婚逼签、当众耳光，以及 Chloe 结尾下令制造事故，最后停在“她要杀 Serena”的动作前后。
- Ep2 的双电梯取舍很直观：Serena 被困货梯、Chloe 在 VIP 玻璃梯，Adrian 当众授权把电力给 Chloe。这里有身体危机、公开选择、不可逆后果。
- Ep3 从失子到 Nova Frost 回归，完成了第一轮“被弃救 -> 死而复生 -> 复仇登场”的闭环。

4-6 集继续能看：

- Ep4 用 debt trigger、供应商离席、安保失效、继承墙熄灯，把 Serena 的新权力落成现场后果。
- Ep5 的 penthouse 反制有动作，Chloe 带人冲入、保安权限被撤、手被高跟鞋踩住，爽点明确。
- Ep6 的烧伤疤和事故档案让 Adrian 第一次动摇，关系从对抗推进到追悔入口。

7-8 集开始变弱：

- Ep7 主要动作是“认证台扫描 -> 屏幕要求亲子验证 -> nameplate 悬停”，压力有，但更多来自机器结果，不是来自角色互相逼到行动。
- Ep8 直接给出 NOT VERIFIED、亲子不匹配、Chloe 口头坦白。信息量大，但场面推进像证据公布会，人物动作相对少。

9-10 集情绪强度回升：

- Ep9 的旧对讲器、双胞胎 ultrasound、半织 baby blanket、雨中下跪自扇，有画面和身体动作。
- Ep10 的公开下跪、交出信托/辞职/口供、血按 biometric scanner，结尾停在 SUBMITTING，短剧卡点成立。

### 不淡，但机制味偏重

V20 不淡，压迫、羞辱、火灾、失子、回归、当众揭露、跪求都有。但豪门商战壳经常通过“权限系统 / 屏幕结果 / 文件授权”制造爽点：

- Ep4：credit facility suspended、安保因 creditor clause 不动、heir wall 熄灯。
- Ep5：access revoked、card declined、foundation privileges suspended、供应商终止通知。
- Ep7-8：trust event locked、paternity required、not verified、foundation seat frozen。
- Ep10：federal attorney submit、biometric accepted、submit confession。

这些元素可以用，但现在中后段密度偏高，容易让观众感觉 Serena 的力量来自系统开关，而不是一场场人物选择和权力肉搏。

## 分层问题

### 1. screenplay / scene packet 层：7-8 集场面主要靠验证机制推进

Ep7-8 的核心戏是 Leo 是否 Bexley heir。正文把冲突设计成继承墙认证仪、屏幕结果、样本盒、直播镜头。它有公开羞辱，但现场行动链偏短：

```text
Serena 带认证台进场
-> 系统要求亲子验证
-> 系统判定不是继承人
-> Serena 放样本盒
-> 屏幕给出生物学结果
-> Chloe 被 Adrian 逼问后口头爆料
```

问题不是不能有 DNA/信托认证，而是人物没有被持续逼到更强动作。Chloe 除了抱紧 Leo、否认、被拽手腕，缺少试图毁证、抢孩子、切断直播、反咬 Serena、暴露更大筹码等可拍动作。Adrian 也主要是批准 proceeding、质问、沉默。结果是硬信息很强，现场搏斗感弱。

对应断链：

- 场面动作链：偏靠屏幕结果。
- 权力变化 / 打脸链：有结果，但夺权过程不够肉身化。
- 情绪 / 爽点链：爽点兑现偏快，蓄力和反扑不足。

建议回退：Step 8 Scene packet，重建 Ep7-8 的高压场动作链。

### 2. 留存 / 信息差链：Ep8 吃掉了 Ep9 的调查悬念

Ep8 里 Chloe 当众说出：

```text
She screamed in that lift. You heard her. You cut the power. You chose me. You left her there.
Her miscarriage has your signature on it.
```

这能制造当场爆点，但它把事故真相、Adrian 的责任、Serena 的流产几乎一次说完。Ep9 再去 sealed floor 找旧对讲器、ultrasound、blanket，变成情绪补证，而不是“观众跟 Adrian 一起发现更可怕真相”。首批可以让 Adrian 开始知道，但不宜让 Chloe 一口气替后续调查把核心答案说干。

对应断链：

- 主线期待 / 信息差链：真相释放过满。
- 留存 / 钩子 / 卡点链：Ep8 之后的追问从“真相是什么”降成“Adrian 如何崩溃”。
- 连续性 / 伏笔链：Ep9 的旧物仍有效，但惊讶度被削弱。

建议回退：Step 7 Episode map 或 Step 8 Scene packet，重新切分 Ep8/Ep9 的信息释放边界。

### 3. adaptation / scene 层：第 2 集强，但同构风险需要第二轮查源本

只看正文，Ep2 是强戏：两部电梯、二选一、Adrian 授权、Serena 烟中护腹，短剧感成立。

但它的事件形态非常高概念、强桥段化，且是整批最关键的悲剧触发点。是否属于“新壳有效承载”还是“源本强桥段改名复刻”，第一轮不能判断，必须第二轮读源本和迁移证据。

本轮暂不把它判硬失败，但标为第二轮最高优先级审计点。

### 4. dialogue 层：英文台词大多短，但个别句子解释感重

整体台词比“电视剧聊天”短，很多句子有功能：

- `Sign it.`
- `My heir.`
- `You chose.`

问题主要在少数高压处直接说主题：

- Ep8 Chloe 连续解释事故责任；
- Ep9 Adrian 连说 `I killed them. / I killed our children.`
- Ep10 Adrian 解释三份文件和赎罪诉求。

这些句子不是都要删，但应优先让动作、屏幕、旧物、他人反应承担一部分信息，避免人物替观众总结。

建议回退：若 Ep7-8 场面重建后，再进 Step 11 dialogue polish；不要只在现有结构上修句子。

## 硬失败检查

| 检查项 | 判断 |
|---|---|
| 洗飞源本有效性 | 第一轮不能判，需第二轮查源本 |
| 改名复刻 | 第一轮不能判定；Ep2 标记高风险 |
| 短剧降级 | 局部存在，集中在 Ep7-8 机制验证戏 |
| 正文长剧化 / 站桩聊天 | 未构成硬失败 |
| 高压节点只有台词 | 未构成硬失败，但 Ep8 口供吃信息过多 |
| 证据链代替场面 | 局部存在，集中在 Ep7-8，次级在 Ep4/5/10 |
| 每集没有变化 | 未发现；每集至少有关系、权力或信息变化 |
| 卡点把结果说完 | Ep8 偏过度消费，需修 |
| 状态漂移 | 第一轮未见明显漂移；需第二轮查 story memory |

## 修订优先级

1. 重建 Ep7-8：让 Chloe 和 Adrian 在现场被逼到更具体的行动，不只被认证系统判死。
2. 拆开 Ep8/Ep9 真相释放：Ep8 可以打穿假继承人，但不要把事故、流产、Adrian 授权责任全部说完。
3. 降低屏幕/权限通知密度：每次屏幕结果后必须有即时人物动作、权力剥夺或关系变化，不能只显示红字。
4. 二轮重点审 Ep2：确认双电梯/弃救是否属于可接受的新壳承载，而不是源本强桥段换名复刻。

## Round 1 结论

V20 正文可以进入第二轮源本审计，但不能直接 pass。

当前结论：`revise`。  
不是 block 的原因：首批主线清楚、可看、能追，强节点大多有可视化动作和当众后果。  
不能 pass 的原因：Ep7-8 的机制验证戏和信息释放边界会削弱中后段追看；源本迁移真伪尚未审计。
