# V30 vs V5 vs 商业平台稿质量判断

## 对比对象

- V30：`PRD_v30_clean_v16_runtime_constraints/07_screenplay正文初稿_首批1-10集.md`
- V5：`PRD_v5_豪门商战版_首批1-10集.md`
- 商业平台稿：`/Users/jiakun/Downloads/《暗涌》.docx`

说明：商业平台稿不是完整同层正文。它包含：

- 信息流：目标受众、题材类型、核心设定、风格元素、世界观、核心亮点、核心梗概；
- 人物设定；
- 粗纲；
- 37 集集纲；
- 正文内容只展开到第 1-5 集。

因此，商业平台稿只能分两层比较：

```text
前 5 集正文写法
+ 1-37 集集纲的短剧刺激设计
```

不能把它当完整 1-10 集正文与 V30 等量比较。

## 总判断

```text
商业平台稿：短剧本能刺激最强，但同构和混稿最严重。
V5：场面动作链最粗暴直接，是旧强度基准，但仍是残次品。
V30：洗稿更干净、结构更稳、正文更像 screenplay，但本能冲击还没打过 V5 / 商业稿。
```

如果只问“观众会不会马上想看下一集”：

```text
商业平台稿 > V5 > V30
```

如果问“洗稿是否干净、是否能作为长期系统方向”：

```text
V30 > V5 > 商业平台稿
```

如果问“当前项目有没有明显进步”：

```text
有。V30 比 V28/V24 更像戏。
但还没到能说 workflow 已经解决短剧感的程度。
```

## 为什么 V30 仍没打过 V5 / 商业稿

### 1. V30 的强场面更“高级”，但没有更“本能”

V30 的 wrong door / old lab / safety chamber 是一个更干净的新壳。

但短剧观众第一反应不是欣赏结构干净，而是要立刻感到：

```text
她要死了；
她被羞辱了；
他错得太狠了；
这个反派该被打；
下一集我必须看她怎么反击。
```

商业稿和 V5 在这点上更直接。

V30 经常需要观众理解：

```text
权限、旧命令、救援钥匙、originator、foundation chair、safety chamber
```

这些能成立，但刺激来得慢。

### 2. V30 解决了“证据链”，但有点滑向“安全系统意象链”

V28 / V24 的问题是证据、终端、文件太多。

V30 改成了：

```text
门、钥匙、旧实验室、空椅子、安全舱
```

这比证据链好，且更统一。

但风险是全剧前 10 集过度围绕同一套安全系统隐喻，导致观众记住的是“这套系统”，不是每集不同的情绪爆点。

### 3. V30 的女主更强，但痛感和恨意没有商业稿那么外放

商业稿第 2 集直接让林薇在防爆玻璃里拍门、窒息、爆炸；第 3 集醒来发现永久失去生育能力；第 4 集回归直接耳光和枪口。

V30 的 Ava 也有伤手、旧门、旧实验室，但她的表达更克制。

克制不是错，但女频短剧在高压节点不能一直端着。V30 比过去好，但仍偏冷。

### 4. V30 的 EP7/EP8 还是没完全摆脱“系统证明”

V30 已经让 Celeste 当众打不开救援门、让董事离席、让 Adrian 摘名牌。

这比报告强很多。

但本质仍有：

```text
钥匙认证失败 / access denied / proxy record / terminal
```

商业稿和 V5 更粗暴：

```text
掐脖子 / 踹开 / 枪抵头 / 当场验血 / 疯狂自扇
```

V30 更干净，但没有同等原始冲击。

## V30 的真实位置

V30 不是废稿。

它证明：

```text
S3/S4/S5 的生产动作小修确实有效。
干净 agent 也能把证据链转成更像戏的动作链。
```

但它还没有证明：

```text
我们的 workflow 已经能稳定产出“比商业平台更能留人”的短剧。
```

所以当前判断不能是 pass。

更准确：

```text
V30 = 候选方向有效，但短剧强度仍不够。
```

## 根因更新

现在最准确的根因不是“writer 完全不会写戏”。

V30 已经证明 writer 能写出戏。

但现在剩下的问题是：

```text
writer 能把功能写成动作链，
但还没有稳定把动作链写成最能刺激观众的短剧强场面。
```

也就是：

- 从“任务单”到“戏”这一步，V30 已经明显进步；
- 从“戏”到“爆款短剧强戏”这一步，还没打穿。

这不是继续加 reviewer 能解决的。

下一步如果继续修，重点应是 S5/S6：

```text
writer 在生成每集施工稿时，不只问“这场怎么演”；
还要比较“这场有没有选择最能产生观看冲动的演法”。
```

但这不能变成固定模板。

正确方向不是写死“每集必须耳光、跪、枪、爆炸”，而是让 writer 在每个高价值节点做创作选择：

```text
这个节点的观众冲动是什么？
最本能、最可见、最有付费欲望的承载是什么？
有没有因为防复刻或怕出错，把它写得太冷？
```

## 最终拍板

```text
V30 比 V28/V24 明显进步；
V30 没有超过 V5；
V30 更没有超过商业平台稿的短剧刺激度；
但 V30 比两者都更干净、更适合作为系统方向继续打磨。
```

所以：

- 不回滚到 V5；
- 不照抄商业平台；
- 不正式化当前 v16 小修；
- 保留 V30 作为当前有效方向样本；
- 下一步只修“强戏选择 / 观看冲动选择”这一层，不再扩 reviewer，不再堆规则。
