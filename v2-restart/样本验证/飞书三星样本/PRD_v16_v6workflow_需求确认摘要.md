# PRD v16-v6workflow 需求确认摘要

版本：V16_v6workflow  
使用 workflow：`workflow_spec_v6_能力回收修正版草案.md`  
状态：受控验证，不代表 v6 已正式升版。

## 1. 本轮任务

按 v6 草案重新跑飞书三星样本首批 1-10 集，验证这条链路是否真的把源本能力传到 writer 和正文：

```text
Source Bible
-> Source Retention Anchor
-> Story Operating State
-> Episode function map
-> Writer brief with Scene Sequence
-> Screenplay
-> Beat survival check
```

本轮不是 V15 局部润色；允许吸收 V15 / V14 已证明有效的短剧场面经验，但必须重新产出分析、brief、正文和自检证据。

## 2. 输入与范围

| 项 | 本轮确认 |
|---|---|
| 源本 | `[修改版] 黑手党少主痛哭求原谅_前妻她杀疯了.txt` |
| 阅读范围 | 源本 1-10 集精读，同时理解全剧人物设定和后续追妻火葬场方向 |
| 新壳 | 海外豪门商战 / 家族企业权力斗争 |
| 目标市场 | 海外短剧 |
| 输出范围 | 首批 1-10 集 |
| 集数 | 不变 |
| 改写力度 | 中度偏重：保留源本追看体验，替换高辨识外形 |
| 正文语言 | 英文对白为主 |
| 审稿语言 | 中文 |
| 正文用途 | AI 出海短剧可视化剧本，画面和声音应基本等于最终能看见/听见的东西 |

## 3. 本轮必须守住

- 女主被替代、被误判、被男主亲手放弃，必须是身体和孩子级不可逆损失，不能降成职位/股权损失。
- 男主必须是做错选择的人，不是被系统、董事会或流程完全代替责任的人。
- 白月光必须当众失态，不能只被文件、报告或屏幕击败。
- 女主五年后必须亲手改变房间秩序，而不是只递交审计材料。
- 男主追悔必须来自主动追回旧伤和旧物，不能靠别人把证据端到他手里。
- 首批结尾必须卡在女主是否公开处刑 / 是否给原谅之前。

## 4. 本轮特别修正目标

对照 V15：

- 保留 V15 已验证的强方向：服务通道二选一、双胎腕带、儿童公益场爆雷、直播处刑权。
- 修 V15 局部弱点：E5 要同时看见“动利益”和“动人”；E8 必须更公开，不能缩成私人走廊；E10 要比 V15 更简洁，减少拥挤解释。
- 按 v6 增加 writer brief 的 Scene Sequence，让每集不是只有压力链结论，而是有可写的场景顺序、进入状态、动作推进和退出状态。

## 5. 不做

- 不改 PRD、入口文档或 workflow 草案。
- 不把 v6 草案升正式版。
- 不做网页 App / 工程 spec。
- 不把审计大表塞进正文写作面。
- 不启动独立 reviewer；本轮主创完成后只判断是否可进入干净 reviewer。
