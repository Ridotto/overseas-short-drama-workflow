# PRD v7 vs V3 / V6 版本对比

日期：2026-06-30  
本版：`PRD_v7_豪门商战版_首批1-10集.md`  
参照：`PRD_v3_豪门商战版_首批1-10集.md`、`PRD_v6_豪门商战版_首批1-10集.md`、`导演反馈合并判断_豪门商战1-10.md`

## V7 vs V3

| 维度 | V3 | V7 | 判断 |
|---|---|---|---|
| 源本有效性 | 保留较多，压迫、失子、回归、火葬场都在 | 同样保留，且首批闭环更清楚 | 不低于 V3 |
| 同构风险 | 仍有“分居协议、CO2 档案库、公开揭证据、男主火葬场”贴源本功能和形状 | 换成 Atlas 署名、Dock 9 封港、保险认证、root-key、停盘 | V7 明显优于 V3 |
| 短剧场面 | 有可拍桥段，但偏豪门室内和档案库危机 | 港口吊臂、红色封条、品牌墙拆除、董事会吊臂、停港停盘更视觉化 | V7 优于 V3 |
| 动作链 | 有动作，但证据/对话解释仍较多 | 每个证据触发现场行动或权力变化 | V7 优于 V3 |
| 台词 | 较完整，部分说明性 | 更短，更偏选择和压迫 | V7 优于 V3 |
| 钩子/卡点 | 成立 | 每集 end button 更明确，且第 10 集停牌黑屏有新壳强卡点 | V7 优于 V3 |
| 拉扯/关系 | 男主误判和火葬场成立 | 男主的企业指令直接成罪责，关系更硬 | V7 优于 V3 |
| continuity | 基本成立 | root-key、Dock 9、腕带、徽章、CEO 权限线更统一 | V7 优于 V3 |
| 读感 | V3 更直给，V7 名词略多 | V7 需要拍摄时把术语转成动作 | V7 有轻微复杂度风险 |

结论：V7 不低于 V3，且在同构扣除、场面动作链和新壳承载上明显更强。

## V7 vs V6

| 维度 | V6 | V7 | 判断 |
|---|---|---|---|
| 源本有效性 | 保留强，1-10 闭环完整 | 保留强，闭环完整 | 基本持平 |
| 同构风险 | 电梯事故、继承信托 DNA、物证室、餐厅枪响仍接近源本与旧版桥段 | 改成港口封锁、项目冒领、root-key、停港停盘 | V7 优于 V6 |
| 短剧场面 | 已比 V5 正文更 screenplay 化，但第 7-8 集仍有证据认证感 | 第 7-8 集用吊臂、投票权冻结、旧权限台推动 | V7 优于 V6 |
| 动作链 | 有动作，但电梯/信托/物证室链条较顺，容易看成豪门版源本 | 动作链更换了载体，和源本距离更远 | V7 优于 V6 |
| 台词 | 已压缩，但仍有“你切过我的电”等强解释回收 | 台词更短，回收点变“红色封条 / 总停机” | V7 略优 |
| 钩子/卡点 | Ep10 枪响黑屏强，但仍像源本枪响 | Ep10 停港停盘黑屏，新壳化更强 | V7 优于 V6 |
| 拉扯/关系 | Adrian 误判、后悔成立 | Adrian 的企业权力亲自造成伤害，并亲手毁权力求赎罪 | V7 优于 V6 |
| continuity | 电梯事故线完整 | Dock 9 / root-key / CEO 权限线完整 | 持平或略优 |
| 读感 | 更容易懂 | V7 商战术语更密，需要注意可读性 | V6 在直观度上略占优 |

结论：V7 对 V6 的核心进步是“去同构”和“新壳动作链”。主要退步风险是项目术语比 V6 多，后续可通过台词和美术动作降噪。

---

## V7 进步

1. 事件链从源本式婚恋事故转成企业权力事故：
   - 不是车祸 / 电梯；
   - 是 Dock 9 红色封锁、港口自动化、CEO 应急指令。
2. 女配谎言从假孕 / 非亲生，改成项目功臣和投票权资格冒领。
3. 男主火葬场从“找到遗物然后下跪”，改成“签责任书才有资格打开真相，再亲手停港停盘”。
4. 第 10 集卡点从物理自杀测试，变成商战版的家族帝国自毁选择。
5. 每个证据都有动作后果，降低文件/听证会感。

---

## V7 退步或风险

1. Atlas / root-key / 保险认证 / 停保等词有一定理解成本。
2. 港口自动化场面拍摄上比普通豪门室内戏更重，需要导演判断预算承载。
3. Adrian 按总停机键的赎罪动作很大，后续必须控制 Serena 不快速软化。

---

## 是否允许进入导演 review

主创侧判断：可以进入独立 reviewer。  
不建议跳过 reviewer 直接给导演。

原因：

- V7 的同构问题比 V6 明显改善；
- 但 V7 新壳术语和港口场面承载度需要第三方判断；
- reviewer 应先看正文和写作前工作稿，不先看作者 quality gate。

## 下一步
