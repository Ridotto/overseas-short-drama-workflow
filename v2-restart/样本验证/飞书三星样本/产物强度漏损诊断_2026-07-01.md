# 产物强度漏损诊断

日期：2026-07-01

## 0. 这份诊断回答什么

问题不是：

```text
V5 是不是很好？
V23 / V24 哪个更稳？
workflow 有没有完整跑完？
```

真正问题是：

```text
为什么流程越来越完整，产物还是不够好看、不够强、不够能打？
```

本诊断基于 V5 / V22 / V23 / V24 的中间产物、正文、reviewer 结果和 workflow spec。

## 1. 总结论

根因不是某一个字段缺失，也不是单纯“短剧感规则不够”。

更准确的根因是：

```text
当前 workflow 的主目标在实际执行中变成了“降低失败风险”，
而不是“制造更强观看冲动”。
```

它确实越来越能做到：

- 不洗飞；
- 不改名复刻；
- 信息边界不乱；
- 事实自洽；
- 有钩子；
- 有可见动作；
- 有 reviewer 能指出回滚层。

但这些只证明：

```text
这稿不太容易死。
```

它没有证明：

```text
这稿足够好看，足够爽，足够让用户付费追。
```

所以 V23 / V24 的问题不是“没活”，而是“活得太低保”。

## 2. 最核心的机制问题

### 2.1 Gate 变成存活检查，不是强度发动机

V23 / V24 都大量使用这类判断：

```text
追问还在吗？
压抑释放还在吗？
现场戏还在吗？
对抗燃料还在吗？
```

这些问题有用，但它们只能判断“有没有死”。

它们不能判断：

```text
这个追问够不够想看？
这个释放够不够爽？
这个现场戏是不是最狠的写法？
这个对抗燃料是不是能继续烧？
```

证据：

- V24 的 `Playable Writer Brief` 四个存活问全部 pass；
- V24 reviewer 仍指出 4 个 should-fix：7-8 证据链可信度、Serena 主体性、9-10 转向太快、Chloe 反扑尾巴没落地；
- 也就是说，存活问全部通过，产物仍然不够强。

结论：

```text
当前 gate 防死有效，提强无效。
```

### 2.2 Writer brief 被压薄时，创作热量一起被压没

“writer brief 要薄”这个方向是对的，避免上下文污染。

但现在的问题是：

```text
薄 brief 不只去掉了噪音，也去掉了场面里的创作热量。
```

V5 的写作前材料虽然粗，但它给 writer 的不是抽象存活问，而是：

- Scene purpose；
- Starting power state；
- Action chain；
- Evidence triggers what action；
- Relationship shift；
- Power shift；
- Performance / body / space；
- Dialogue functions；
- Exit state。

这些东西能直接把场面写成戏。

V23 / V24 的 brief 更干净，但经常变成：

```text
本集追问是什么；
本集兑现什么；
选中的场面是什么；
最后卡什么；
四个存活问 pass。
```

它让 writer 知道“要写什么”，但不够让 writer 知道“怎么把这一场写狠”。

结论：

```text
brief 薄是对的；
但现在薄错了方向。
它删掉的应该是审计噪音，不应该删掉场面发动机。
```

### 2.3 上游材料知道短剧原则，但没有变成正文优先级

V24 的 `Short Drama Creative Plan` 其实写得不差。

它明确说：

```text
观众不等泛泛商业谈判；
商战动作必须转成门被锁、账户冻结、董事站队、LED 屏改写、直播红灯。
```

但正文里，很多地方还是先完成“信息流程”，再补一个可见动作。

典型例子：

- Ep7-8 上游知道要公开审判、信托、继承骗局；
- Scene Carrier 也写了证据触发动作；
- 正文仍然滑成 `Trust Officer / DNA kit / result / Lucas match` 的顺序。

这说明：

```text
上游审美判断没有进入 writer 的第一优先级。
```

writer 仍在优先完成：

```text
信息说清楚；
事实不矛盾；
流程能过；
场面可拍。
```

而不是优先完成：

```text
这一场哪一下最让观众爽 / 怒 / 想追。
```

### 2.4 “避免规则怪谈”的反作用：强度标准被降成二元最低线

为了防止无限 reviewer 和规则怪谈，我们把很多判断收窄成二元：

```text
在 / 不在；
活 / 不活；
pass / risk / fatal。
```

这确实减少了无限挑刺。

但副作用是：

```text
只要某能力“在”，它就过了。
```

例如：

- 有现场戏，不代表现场戏强；
- 有可见后果，不代表后果有记忆点；
- 有反派燃料，不代表反派真的还会让观众期待；
- 有 End Button，不代表观众非看下一集不可。

所以现在不是“加更多规则”的问题。

现在的问题是：

```text
二元问题问得太低。
```

它们不该问：

```text
有没有？
```

而应该问：

```text
这一集最让人想继续看的那一下是什么？
如果拿掉它，这集还剩什么？
这一版是不是选择了最有戏的承载，而不是最安全的承载？
```

### 2.5 流程完整，但没有真正的“创作选择责任人”

现在链路里每一层都有自己的责任：

- Source Bible 负责读源本；
- Boundary 负责防复刻；
- Creative Pack 负责故事成立；
- Stage / Episode 负责追问；
- Scene Options 负责候选；
- Writer Brief 负责可写；
- Reviewer 负责查问题。

但没有一层真正承担：

```text
在多个都能过检的写法里，选择最有观众冲动的一版。
```

结果就是：

```text
每层都 pass，
但最终选出来的是安全解，不是强解。
```

V24 的 Ep10 就很典型：

- SEC 自白直播很贴商战壳；
- 它合规、清楚、可拍；
- 但它没有天然等于“最强火葬场”。

它需要被写成：

```text
公开失去帝国信用 + 被所有投资人直播审判 + Serena 是否阻止他自毁
```

如果只写成 “Go Live / I, Adrian Blackwell...” ，它就只是一个合理卡点，不是强卡点。

### 2.6 洗稿约束被过度理解成“功能继承 + 外形替换”

洗稿确实要保留源本追看机制。

但现在很多环节实际执行成：

```text
源本功能是什么；
新壳换什么外形；
不要复刻；
不要洗飞。
```

这会得到一个合格的改编骨架。

但爆款洗稿还需要多一步：

```text
新壳里有没有比原壳更有力、或至少同等有力的强场面。
```

如果只是“车祸 -> 电梯 / 档案 / 信托 / 直播”，那只是载体替换。

真正要问的是：

```text
这个新载体有没有同等强的：
身体危险；
当众羞辱；
不可逆失去；
关系断裂；
权力反转；
下一集追问。
```

V24 Ep2 的双玻璃电梯比 V23 Level 42 更强，就是因为它更接近“身体危险 + 公开选择 + 视觉冲击”。

但 V24 Ep7-8 又弱，是因为它从“人被当众击穿”滑回“程序证明事实”。

## 3. 为什么流程越完整，反而更平

因为完整流程带来了三种副作用。

### 3.1 产物越来越像“合规答案”

中间产物越多，agent 越容易把任务理解成：

```text
逐项满足上游字段。
```

于是正文会变得：

- 结构完整；
- 信息清楚；
- 风险较少；
- 但缺少野心和狠劲。

这就是用户反复感受到的：

```text
像程序，不像剧本。
```

### 3.2 reviewer 越清楚，writer 越保守

reviewer 分层越明确，writer 越知道哪些地方会被打回。

这会让 writer 倾向于选择：

```text
更容易解释；
更容易自洽；
更不容易同构；
更不容易越界。
```

而短剧强场面经常需要：

```text
更冒险；
更尖锐；
更情绪化；
更不体面。
```

所以如果没有一个同等强的“创作冲动目标”，writer 会自然滑向安全解。

### 3.3 “薄 brief”如果只传任务，不传戏，就会写成流水线

writer 不是不能只看 brief。

但 brief 不能只告诉它：

```text
本集要兑现什么。
```

brief 必须告诉它：

```text
这一场的戏剧发动机是什么：
谁想要什么；
谁挡住谁；
谁被迫当众选择；
谁失去什么；
观众应该在哪一下产生爽 / 怒 / 痛；
最后停在哪个没答案的动作。
```

现在 brief 有这些内容的影子，但不够强，且经常被压成一句话。

## 4. 这不是 V5 多好，而是 V5 暴露了一个事实

V5 不是成品标准，也不是多高级的 workflow。

V5 甚至有明显问题：

- 内部标签多；
- 第 10 集枪响类型边界危险；
- 部分设定粗；
- 也没有真正解决长期稳定生产。

但 V5 暴露了一个事实：

```text
只要场面动作链够硬，观众会先被抓住。
```

V5 的有效部分不是“某个桥段”，而是：

```text
证据一出现，马上触发人物动作；
动作马上触发权力变化；
权力变化马上变成可见羞辱 / 失去 / 卡点。
```

这条链在后续版本里被拆散了。

后续版本更会证明自己没错，但不如 V5 会制造观众冲动。

## 5. 当前应修的不是“再加规则”，而是换目标函数

现在不能继续问：

```text
还缺哪条规则？
还缺哪个字段？
还要不要加一个 agent？
```

应该问：

```text
每个高价值节点，当前候选是不是强度最高的可用承载？
如果不是，为什么选了它？
writer brief 有没有把这个节点的戏剧发动机传下去？
reviewer 有没有权力因为“太平庸”打回，但又不能无限挑刺？
```

这不是规则怪谈。

这是把 workflow 的目标从：

```text
不失败
```

改回：

```text
写出更想看的短剧。
```

## 6. 下一步建议

### 6.1 不要继续跑新版本

继续跑只会得到更多“结构没大错但强度不够”的稿。

### 6.2 先做强度漏损对照

选 5 个节点：

- Ep1 开场羞辱；
- Ep2 生死选择；
- Ep4 回归压场；
- Ep7-8 公开揭谎；
- Ep9-10 火葬场 / 付费卡点。

每个节点对照：

```text
源本强在哪里；
V5 为什么相对更抓人；
V23 / V24 为什么变平；
是载体选弱了，还是 brief 传弱了，还是正文执行弱了，还是 reviewer 放过了。
```

### 6.3 再改 workflow，只改一处核心接口

最可能要改的是：

```text
Scene Options -> Writer Brief
```

不是加新表，而是把 writer brief 里的每个高压节点从“任务摘要”改成“戏剧发动机卡”。

这张卡只回答：

```text
这场观众最想看的那一下是什么；
谁被逼到没有退路；
谁当众失去什么；
爽 / 痛 / 怒 落在哪个动作；
结尾停在哪个未完成动作；
为什么这个载体比其他候选更狠。
```

### 6.4 reviewer 也要改，但不能回到无限挑刺

reviewer 不应该因为“还能更好”无限打回。

但 reviewer 应该能判：

```text
高价值节点太平，没有形成观看冲动。
```

前提是它必须指出：

```text
是哪一个高价值节点；
原本期待的观众冲动是什么；
当前稿哪一下没打出来；
应该回 Scene Option 还是 Writer Brief，而不是全稿重写。
```

## 7. 最终判断

当前项目不是没有流程。

当前项目的问题是：

```text
流程太擅长证明“这稿没错”，
不擅长逼出“这稿够狠”。
```

V23 / V24 的失败不是执行失败，而是目标函数失败。

下一步必须围绕“高价值节点强度如何传递到正文”修，而不是继续扩 PRD、扩 reviewer、扩 reference。
