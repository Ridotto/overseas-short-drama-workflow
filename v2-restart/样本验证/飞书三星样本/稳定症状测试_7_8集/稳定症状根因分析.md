# 第 7-8 集稳定症状 · 根因分析

> 日期：2026-06-30
> 基于：5 个独立 writer 样本 + reviewer A / reviewer B 诊断

---

## 1. 最底层根因

当前第 7-8 集正文失败的根本原因不是“少写了视听语言”，也不是“writer 没听话”。

根本原因是：

**写作前工作稿把第 7-8 集组织成了“证据揭露链”，而不是“短剧行动场面链”。**

换句话说，它给 writer 的主引擎是：

```text
CFO 证词
-> 付款记录
-> 维保商
-> 信托核验
-> 非亲生结果
-> Chloe 反咬
-> 物证室
```

writer 为了完成任务，会自然按这个顺序执行。

所以它稳定写成：

```text
放证据
-> 人物惊讶
-> 再放证据
-> 人物否认
-> 再放证据
-> 反派解释
-> 男主去下一处证据
```

这不是短剧场面，而是证据播报。

---

## 2. 根因拆解

### 2.1 输入形态错了

当前写作包告诉 writer “要揭露哪些信息”，但没有把这些信息改造成“谁被迫做什么动作”。

信息链：

```text
Chloe 假孕被揭穿
孩子非亲生
男主当年也有罪
男主去物证室
```

短剧行动链应该是：

```text
证据出来后，谁立刻失去身份？
谁必须当众选择护谁？
谁被谁抛弃？
谁被拖离权力中心？
谁的身体反应把情绪打出来？
这一集必须卡在哪个动作前？
```

当前稿子主要给了前者，没有足够硬地给后者。

### 2.2 “视听化”被当成装饰，不是生成骨架

writer 知道要写屏幕、胸针、门禁、血、红灯。

但这些只是可见物件。

它没有把视听元素组织成动作因果：

```text
胸针被摘下
-> Chloe 失去继承人母亲身份
-> Adrian 公开切割她
-> 董事会当场倒戈
-> Chloe 反咬男主
-> 男主被迫从审判者变成罪人
```

所以结果是“东西可见了”，但戏没有真正动起来。

### 2.3 正文模型默认策略是“覆盖要求”，不是“创造场面”

当输入里有很多必须发生的信息点时，writer 会优先保证都写上。

这会导致它把短剧写成 checklist：

```text
诊所 CFO 写了
维保商写了
付款记录写了
信托核验写了
胸针写了
Chloe 反咬写了
物证室写了
```

每个点都在，但整体不好看。

### 2.4 第 8 / 第 9 集边界设计太近

第 8 集要求：

```text
Adrian 冲去事故物证室
```

第 9 集要求：

```text
打开物证室，看见物证，听见录音，彻底崩溃
```

这两个动作距离太近。

writer 为了让第 8 集结尾更重，会自然多写两步：

```text
到门口
开门
看见物证
听到录音
```

于是第 9 集被提前消费。

## 4. 真正要解决的不是再补规则

如果继续补规则，很容易变成：

```text
不要听证会
不要证据播报
不要反派自白
不要提前物证室
Serena 要更主动
男主要更挣扎
第 7 / 第 8 集要区分
```

这会再次变成规则怪谈。

更准确的产品问题是：

**写作前中间产物需要从“信息 / 证据清单”改成“短剧行动场面链”。**

也就是，每个高压节点不能只给：

```text
要揭露什么证据
要完成什么剧情功能
```

还必须转成：

```text
这场戏谁先占上风？
第一份证据出来后，谁被迫做什么动作？
第二次反转让谁当众失去什么身份？
谁从保护者变成抛弃者？
谁的身体反应证明他被打穿？
这一集卡在哪个不可逆动作前？
下一集核心爽点绝不能提前露出什么？
```

---

## 5. 当前确定答案

当前第 7-8 集失败的根本原因是：

**我们让 writer 按“证据揭露顺序”写正文，而不是按“短剧行动升级顺序”写正文。**

所以 5 个独立 writer 都能完成剧情，却稳定写不出够短剧的正文。
