## V31 比 V30 好在哪里

### 1. S5 真做了强戏演法选择

V30 的强场面已经比 V28/V24 好，但很多场面仍像“选了一个干净装置”：

```text
wrong door / old lab / rescue key / safety chamber
```

V31 的 S5 每集都多了一步：

```text
弱演法 -> 可用演法 -> 选定演法
```

这一步让 writer 不再默认拿最安全的实现。

例如 E8：

```text
弱演法：DNA report 甩地。
可用演法：现场抽血验证。
选定演法：继承人 biometric lock 拒绝 + Adrian 亲手摘 pin。
```

这比 V30 的 permission / terminal 叙事更接近短剧动作。

### 2. V31 的物件和动作更稳定

V31 的首批记忆点：

- donor badge 被摘；
- old maternity wing locked door；
- baby bands；
- foundation pin 掉进 champagne；
- heir pin 被 Adrian 亲手摘下；
- CEO badge 被砸弯；
- live mic 公开认罪。

这些比 V30 的系统意象更容易被观众记住。

### 3. V31 的男主罪责更直观

V30 的男主罪责仍偏“系统旧命令 / 安全舱 / 旧房间复盘”。

V31 里罪责链更直接：

```text
E2 Adrian 亲口下令 keep Serena contained
E9 录音回放这句命令
E10 他公开承认 locked door
```

这条线比 V30 更清楚。

## V31 为什么仍没打过 V5

### 1. V5 的强度更原始

V5 用的是非常粗暴但有效的短剧刺激：

- 夫人胸针被扯下，锁骨划破；
- 服务电梯坠落，血从门缝渗出；
- 黑卡被剪，Serena 高跟鞋踩碎；
- 继承墙红封条；
- 物证室里血卡、银铃、烧黑胸针；
- Adrian 自扇耳光到嘴角出血；
- EP10 枪口抵眉心，一命换两命。

这些不精致，但非常直接。

V31 没有这么强的身体冲击。它更干净，但也更冷。

### 2. V31 仍偏“机构装置戏”

V31 虽然比 V30 更动作化，但核心承载仍是：

```text
badge / lock / trust / system / record / foundation / CrossCare
```

这些比文件报告好，但它们仍然是制度和装置。

V5 的强点更多是：

```text
人当众下手；
人亲手剥夺；
人身体崩溃；
人被逼到极限。
```

所以 V31 读起来更像干净的豪门商战短剧，V5 更像会让观众立刻上头的狗血短剧。

### 3. V31 的 EP10 没有 V5 的强卡点

V31 EP10：

```text
Adrian 公开交权、摘 nameplate、拿 live mic 认罪；
灯灭，Cross emergency siren 响。
```

成立，但这个卡点的刺激来自“下一批会发生什么”。

V5 EP10：

```text
枪抵太阳穴；
一命换两命；
枪响黑屏。
```

虽然有壳偏离和过火风险，但短剧付费冲动明显更强。

## V31 跟商业平台稿比

商业平台稿最强的地方不是文本精细，而是它非常懂“本能刺激”：

- 第 1 集：当场逼离婚、假孕、耳光、孕检单；
- 第 2 集：车祸、困车、弃救、爆炸；
- 第 4 集：回归、掌掴、枪口、身份承认；
- 第 7-10 集：掐脖子、亲子鉴定、崩溃、跪地、枪口。

它的问题也很明显：

- 同构风险极高；
- 黑帮 / 科技两套设定混稿；
- 文本粗糙，OS 和解释多；
- 不适合直接作为本项目 workflow。

但它证明一件事：

```text
商业短剧会优先把“观众本能反应”打出来，再考虑精细。
```

V31 正好相反：

```text
先把洗稿干净、结构自洽、场面可拍做好；
但本能刺激不够狠。
```

## 对 v17 接入的判断

v17 接入不是无效。

它有效解决了：

```text
writer 不再只渲染任务单；
S5 会比较演法；
S6 会执行 S5 选出来的场面。
```

V31 能明显看出这个变化。

但它还没有解决：

```text
S3/S4 给 writer 的候选演法空间仍偏冷、偏机构、偏制度。
```

也就是说，`/episode` 接到了 writer 手上，但 writer 能选的材料仍然被上游限制在比较安全、干净、文明的商战系统里。

所以当前根因要更新为：

```text
不是 writer 完全不会写；
而是强戏选择发生得太晚。

到 S5 时，writer 已经只能在 S4 给出的冷载体里选最强版本。
真正的“观众本能刺激”需要更早进入 S3/S4 的承载选择。
```

## 当前拍板

```text
V31 > V30；
V31 未超过 V5；
V31 未超过商业平台稿；
v17 /episode 接入方向有效，但不足以单独让产物达标。
```

不能做的结论：

- 不能说 v17 已经可以正式化；
- 不能说 0xsline 接入失败；
- 不能回滚到 V5；
- 不能照抄商业平台稿；
- 不能继续加 reviewer 规则。

应该做的结论：

```text
保留 v17 的 /episode 接入；
下一步把“观看冲动 / 爽感类型 / 本能刺激承载”前移到 S3/S4，
让场面承载原料本身就有更强的候选空间。
```

更具体地说：

```text
S5 负责选演法；
但 S4 必须先给出足够强的可选演法。

如果 S4 给的都是 badge / lock / system / record，
S5 再会写，也只能写出一版更好看的机构戏。
```

V31 是有价值的一版，因为它第一次证明：

```text
writer 动作能变强；
但上游强戏候选空间还不够强。
```
