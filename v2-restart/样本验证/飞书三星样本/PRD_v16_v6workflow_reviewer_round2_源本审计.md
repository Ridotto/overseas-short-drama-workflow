# PRD v16-v6workflow reviewer round2：源本审计

> Reviewer 第二轮输入：源本 txt 1-10 集、`PRD_v16_v6workflow_Source_Bible.md`、`Adaptation_Boundary.md`、`Source_Retention_Anchor.md`、`Story_Operating_State.md`、`Episode_Function_Map.md`、`HighPressure_Scene_Choice.md`、`Writer_Brief_Scene_Packet.md`、`Beat_Survival_Check.md`、正文 `PRD_v16_v6workflow_豪门商战版_首批1-10集.md`。

## 总结论

`revise`

第二轮审计后，V16 不是源本洗飞，也不是简单改名复刻。Source Bible 抓住了源本 1-10 集最有效的观看发动机：妻位被夺、孕情不被信、生死二选一、失子断裂、五年后强势回归、白月光公开崩、男主从受骗者坠成共犯、主动追旧物、公开跪求但女主不赦免。

这条有效性链路基本进入了 V16 正文。问题不在 Source Bible、Anchor、Episode map 或 writer brief 大方向，而在正文局部执行：第 3 集失子到复仇的场面承接略薄，第 8 集 Chloe 反咬 Adrian 的触发偏便利，第 9-10 集 Adrian 从追证到主动直播忏悔略顺。这些属于可修的 screenplay / pressure-chain execution 问题，不是需要回到源本重拆的 block。

## 1. 源本有效性是否洗飞

没有洗飞。

源本 1-10 集的强度不只是“前妻复仇”，而是：

```text
她被旧爱和白月光公开替代
-> 她喊出怀孕仍被男主丢进死局
-> 她失子后重生
-> 她回来先改变男主熟悉权力房间
-> 白月光的人设和孩子叙事公开崩
-> 男主不能只说自己被骗，必须面对自己亲手做过选择
-> 男主找到旧物后公开跪求
-> 女主保留最终处刑权
```

V16 对应为：

```text
brooch / rescue case / ultrasound
-> red folder / glass corridor / fire door / cutter
-> twins lost / Aria North
-> children wing naming rights / North audit
-> Noah calls Daddy Ben / guardian form / Bennett admits
-> Chloe says Adrian signed and turned away
-> Baby A / Baby B wristbands / sealed corridor
-> GO LIVE button
```

这不是只保留剧情梗概，源本的追看问题和情绪走势都还在。

## 2. 是否改名复刻

不构成改名复刻。

V16 的事件外形已经从黑手党、车祸、军火拍卖、亲子鉴定、仓库旧物、雨中跪地枪口，迁移成豪门商战可用的新事件载体：公司发布会、救场 case、合规甩锅、玻璃服务通道、公益儿童翼、基金会审计、guardian form、旧服务通道、press room live。

需要注意的是：情绪顺序和功能骨架仍高度贴源本，这是当前产品目标允许甚至需要的。它不是问题；问题只会出现在“新事件载体只是旧事件换名词”时。V16 多数高压节点不是换词，而是让新载体改变权力、空间和公开后果。

## 3. Source Bible -> Source Retention Anchor

基本未丢。

Source Bible 标出的 HV1-HV9，在 Anchor 中都有压缩保留：

- HV1/HV2 的“被替代 + 怀孕不被信 + 二选一”进入 E1-E2 留存动作。
- HV3 的“失子断裂”进入 E3 高痛转冷。
- HV4/HV5 的“回归压场 + 动利益”进入 E4-E5。
- HV7 的“白月光骗局崩 + 男主被拖成共犯”进入 E7-E8。
- HV8/HV9 的“主动追旧物 + 公开跪求”进入 E9-E10。

轻微损失点：Source Bible 里 HV3 的“刚失子就看到罪魁仍被公开保护”在 Anchor 和正文里保留了电视新闻，但它后续没有被当成高压节点重点打磨。因此第 3 集在成稿里比 E2、E7、E10 薄。这个不是链路断裂，而是优先级没有覆盖到 E3。

## 4. Anchor -> Episode Function Map

基本未丢，且每集功能清楚。

Episode map 没把 Anchor 变成泛泛剧情摘要，而是给了每集 Opening Hook、Core Friction、Max Spike、Change、End Button 和 Do Not Consume。尤其：

- E2 明确要求唯一 cutter、玻璃服务通道、血手印。
- E5 明确“动利益 + 动人”，避免只打脸。
- E7 明确先由孩子本能反应撕开，再用表格加压。
- E8 明确保持公开 ribbon ceremony，不缩成私人审问。
- E9 明确 Adrian 必须主动撕开旧空间，不拿证据礼包。
- E10 明确空 press room、GO LIVE、不给原谅结果。

风险点：E6、E9 的“证据便利性 / 不能像证据礼包”在 map 中已识别，但正文执行仍偏顺。说明 map 发现了风险，但成稿还可以再加阻力。

## 5. Episode map -> Writer brief

没有退化成结果任务单。

Writer brief 的有效点是它给了 Scene Sequence：

- 每场有功能、进入状态、动作推进、退出状态；
- 高压集附加了压力链；
- 文件/屏幕被限定为逼人物行动，不替高潮收尾；
- 明确保留了“场不能离开 ribbon hall”“场内人物少，外部压力用门外声”等写作约束。

这比单纯写“本集要揭露某真相”强，因为它能指导正文写动作、空间、物件和卡点。

不足：E3 没有被纳入高压场压力链，只靠普通 Scene Sequence 承接，因此从“失去孩子”到“选择复仇”还略像剧情桥。若下一版修，应补一条 E3 小压力链，而不是回源本重拆。

## 6. Writer brief -> Screenplay

大部分进入正文。

### 执行较好的节点

- E1：rescue case、anniversary card、ultrasound、spouse brooch、red folder 全部进正文，并产生可见后果。
- E2：glass corridor、cuff chain、one override cutter、fire door、blood handprint 都进正文，源本“车祸爆炸二选一”的强度被新空间承载住。
- E4：children wing naming rights、donor wall 改名、brooch 投进 champagne bucket，把“回归压场”做成了新壳动作。
- E7：Noah 先喊 Daddy Ben，guardian form 后到，符合“动作先爆，文件加压”。
- E9：Adrian 徒手拆 plywood、摸 blood streak、看 cuff notch，再被 Lydia 逼着说 my wife / my children，基本避免了纯报告火葬场。
- E10：paper 被踢开，signet ring 落地，罪责短句化，GO LIVE 停在一拍前，执行了 brief。

### 执行不足的节点

1. E3 失子断裂偏省。正文有摸小腹、医生宣判、电视新闻、折 ultrasound，但缺少一个能像 E2 血手印那样留住创伤的动作。它能成立，但不够硬。
2. E8 Chloe 反咬触发偏便利。Brief 要求公开压力、Noah 哭声、Bennett 承认把她逼到保命反咬；正文有公开场和 Bennett 承认，但 Chloe 从“he forced me”到“you closed it too”的转折还像作者安排她把关键信息说出。
3. E9-E10 的男主坍塌略顺。E9 已有旧空间代价，但 Lydia 给 Baby B 的阻力仍偏轻；E10 Adrian 主动要 Serena press live 很有效，但最好再让他承受一个不可撤回的公开动作，增强“不是求原谅，是自毁赎罪”。

## 7. Scene Sequence 是否真的进入正文

进入了。

逐集对照，Scene Sequence 没停留在 brief 里：

- E1 三场：救场/替代/尾钩都进入正文。
- E2 三场：公开甩锅/身体危机/不可逆卡点都进入正文。
- E3 三场：失子/复仇入口/五年后入场都进入正文，但场面强度偏薄。
- E4 三场：回归压场/当众打脸/旧门暗刺都进入正文。
- E5 三场：动利益/上门作死/证人线尾钩都进入正文。
- E6 三场：身体旧伤/追查受阻/Baby A 尾钩都进入正文。
- E7 三场：继承人母亲位置/孩子本能撕谎/表格加压都进入正文。
- E8 三场：Bennett 公开承认/Chloe 反咬/Baby A 推向 Baby B 都进入正文。
- E9 三场：主动追旧空间/追回 Baby B/跪到 press room 外都进入正文。
- E10 三场：处刑场/公开跪求/终钩都进入正文。

所以 V16 的问题不是“brief 没被 writer 执行”，而是少数场面执行得还可以更狠。

## 8. Beat survival check 是否有效

有效，但不能当质量总证明。

它抽查了 E2、E5、E7、E8、E10 五个最高风险节点，检查了 Anchor -> Episode map -> Writer brief -> Screenplay 的存活情况。抽查方向是对的，也能证明核心高压节点没有变成文件戏或口供戏。

但它有两个盲区：

1. 没查 E3。E3 正是本轮“失子断裂承接略薄”的地方。
2. 没查 E9。E9 虽然整体有效，但 “Baby B 取得阻力是否足够” 值得纳入抽查。

因此 Beat survival check 不是摆设，但它只能证明被抽查节点活到了正文里，不能替代 reviewer 的全局质量判断。

## 9. 分层失败定位

没有发现以下硬失败：

- `source_bible_failure`：未失败。源本有效性读出来了。
- `anchor_failure`：未失败。留存锚点没有丢核心追看问题。
- `episode_map_failure`：未失败。每集功能和卡点明确。
- `writer_brief_failure`：未失败。brief 有 scene sequence，不是单纯任务单。
- `screenplay_structure_failure`：未失败。正文能追，首批闭环成立。

存在的失败层：

```text
screenplay_pressure_execution / local_scene_trigger
```

它的含义是：正文大结构和迁移链路成立，但局部场面触发、阻力和身体动作还不够硬。

## 必须修改项

1. E3 补一个更强的失子断裂动作：围绕被踩 ultrasound、旧手铐伤、病床/救援残留物或电视新闻，让 Serena 的复仇选择不是一句话成立。
2. E8 重做 Chloe 反咬触发：保留公开场，但让 Bennett、Noah、记者/直播或 Adrian 的逼问把她逼到失控，避免她像主动替剧情补口供。
3. E9-E10 加一点不可撤回代价：Baby B 不要像 Lydia 轻易交出；Adrian 的公开认罪最好先牺牲一个具体 Blackwell 身份/权力动作，再请求 GO LIVE。

## 可选优化

- E5 “Next, I buy witnesses” 可以改成更准确的权力表达，避免 Serena 像买证词。
- E6 旧医院记录被自己签名 sealed 很好，可以在正文里稍微强化 Adrian 发现“他过去的签字还在继续害她”的身体反应。
- Beat survival check 后续可补 E3 / E9 两个抽查项，形成更完整的风险覆盖。
