# 能力归层与交接矩阵

> 日期：2026-07-01  
> 当前 stage：工作流归层校准  
> 上游：`全版本能力回收总账_2026-07-01.md`  
> 用途：把回收能力放回正确层级，明确每层输入、输出、禁区、下游可读内容和回滚位置。  
> 注意：这不是 writer 输入，不是新版 workflow spec。

---

## 0. 总原则

这张矩阵只解决一个问题：

```text
A 环节产出 X；
B 环节只吃 X；
B 环节产出 Y；
C 环节只吃 Y。
```

下游不能回头翻上游厚材料。  
如果下游必须回头翻，说明上一层产物不合格，应回上一层重做。

---

## 1. 层级总表

| 层级 | 角色 | 只负责 | 输入 | 输出 | 禁止 | 下游只能读 |
|---|---|---|---|---|---|---|
| S0 | 需求确认 | 拿到可开工边界 | 用户需求、源本入口 | `最小需求确认` | 分析源本、定剧情、写方案 | S1/S2 读需求边界 |
| S1 | 源本拆文 | 拆源本为什么能追、能付费 | `最小需求确认 + 源本` | `源本有效性包 / Source Bible` | 设计新壳、写新剧情 | S2 读体验功能和高辨识外形 |
| S2 | 洗稿边界 | 区分继承体验和替换外形 | `源本有效性包` | `洗稿边界包 / Boundary` | 写新剧情、为了反同构洗飞源本 | S3 读继承项、自由项、禁用外形 |
| S3 | 新本故事架构 | 搭新壳故事、真相、阶段和分集骨架 | `洗稿边界包` | `新本故事开发包` | 写具体高压场面、锁死源本外形 | S4 读故事状态和分集功能 |
| S4 | 场面承载原料 | 把故事功能变成可写戏的原料 | `新本故事开发包` | `场面承载原料包` | 排完整戏、写台词、写最后一帧 | S5 只读场面原料 |
| S5 | writer 施工稿 | 把原料排成单集短剧戏 | `场面承载原料包` | `单集短剧施工稿 / 写作卡` | 证明流程、读 reviewer、读 run log、照抄上游字段 | S6 只读施工稿 |
| S6 | screenplay 正文 | 把施工稿写成可看可听剧本 | `单集短剧施工稿` | `screenplay 正文初稿` | 写说明书、内部标签、站桩聊天、回头翻 S1-S4 | S7 只读正文 |
| S7 | 台词 / 视听 polish | 修表达、删水词、强化动作声音身体空间 | `screenplay 正文初稿` | `台词视听 polish 稿` | 重设主线、替 S4/S5 重新造场面 | S8/S9 读最终稿 |
| S8 | 连续性 / 信息边界 | 查事实、错信、DNC、伏笔、状态 | `台词视听 polish 稿` | `story memory checkpoint` | 审美评价、短剧感打分 | S9/S11 读状态和风险 |
| S9 | 作者自检 | 主创侧先判明显失败和回滚层 | `polish 稿 + checkpoint` | `作者自检与返修路由` | 把硬风险写成偏好、把问题丢给 reviewer | S10/S11 读结论 |
| S10 | 干净 reviewer | 独立兜底，查红线和明显漏损 | 需求、边界摘要、最终稿、必要源本片段 | `reviewer report` | 生产短剧感、逐句代写、无限挑刺 | S11 读 issue 和回滚建议 |
| S11 | 主控拍板 | 对比、归因、反馈回流、交付判断 | 全部最终产物、run log、版本对比 | `主控版本对比与拍板` | 把 run log 当质量证明，让 reviewer 对用户拍板 | 用户只看结论和必要依据 |

## 2. 能力归层矩阵

| 能力 | 主归属层 | 必须传到 writer 的形态 | 不该出现在哪里 | 失败回哪层 |
|---|---|---|---|---|
| 源本有效性 | S1/S2 | 本集观众正在等什么、源本体验功能是什么 | 不能在 S6 变成 source_ref 证明 | S1/S2 |
| 核心期待 / 主追问 | S1/S3 | 当前集追问、阶段追问、下一集追问 | 不能只在 reviewer 里说“主线散” | S3/S5 |
| 留存 / 钩子 / 卡点 | S1/S3/S5 | 最后一拍停在哪个未完成动作/声音/画面 | 不能变成“有钩子”标签 | S5/S6 |
| 付费 / 留存节点 | S1/S3/S5 | 付费追问、兑现义务、不可提前消费 | 不能照搬固定比例 | S3/S5 |
| 爽点 / 压抑释放 | S1/S4/S5 | 爽之前怎么压低，释放落在哪个动作 | 不能只标爽点类型 | S4/S5 |
| 情绪密度 | S1/S3/S5/S7 | 本集情绪怎么升温、谁被压到什么状态 | 不能只在自检写“情绪不够” | S5/S7 |
| 信息密度 | S1/S3/S8 | 本集哪个信息推进局势，哪个不能说 | 不能由对白空转冒充推进 | S3/S8 |
| 情节密度 / 权力变化 | S1/S3/S4/S5 | 本集结束谁的位置、脸面、资源或主动权变了 | 不能被流程动作冒充戏剧动作 | S3/S4/S5 |
| 人物关系 / 拉扯 | S1/S3/S5 | 谁靠近、谁后退、谁误判、谁怕失去 | 不能只在人物小传里静态存在 | S3/S5 |
| 角色状态 / 错信 | S1/S3/S8 | 当前谁忍、谁崩、谁伪装、谁握主动、谁误会 | 不能只在写后 memory 里保存 | S3/S8 |
| 反派压力 / 反扑燃料 | S1/S3/S5 | 反派本阶段资源、误判、下一步能如何反扑 | 不能只加坏人标签 | S3/S5 |
| 信息差 / 真相揭露 | S1/S3/S8 | 本集能揭什么、不能揭什么、谁知道什么 | 不能混成证据链任务单 | S3/S8 |
| Do Not Consume | S1/S3/S4/S5/S8 | 本集禁说、禁演、禁兑现内容 | 不能只在末端 reviewer 提醒 | S3/S8 |
| 改写力度 / 变量策略 | S0/S2 | 本轮哪些体验贴近、哪些等效、哪些自由、哪些删除 | 不能默认深度或默认换皮 | S0/S2 |
| 防改名复刻 | S2/S3/S4 | 禁止复用的源本高辨识外形 | 不能压过源本有效性 | S2/S3/S4 |
| 新壳等效承载 | S2/S3/S4 | 新壳里自然长出的压力来源、行动方式、可见后果 | 不能靠机构/屏幕/文件偷懒 | S2/S4 |
| 故事核 / 世界观 / 题材 | S3 | 必要背景、题材承诺、市场语感 | 不能变设定百科 | S3 |
| 人物小传 / 行动方式 | S3 | 角色欲望、恐惧、行动方式、本集选择依据 | 不能变静态标签 | S3/S5 |
| 阶段骨架 / 粗纲 | S3 | 阶段追问、兑现、留白、压放、反派压力 | 不能变三幕模板 | S3 |
| 分集功能 / 单集提纲 | S3/S5 | 开场、摩擦、峰值、变化、结尾按钮、DNC | 不能变剧情摘要 | S3/S5 |
| 场面承载原料 | S4 | 谁压谁、谁旁观、谁被迫行动、压力和释放后果 | 不能写成半成品正文 | S4 |
| 高压节点场面优选 | S4/S5 | 候选承载、强度比较、最终选择理由 | 不能成为全剧唯一主轴 | S4/S5 |
| 短剧开场法 | S5 | 本集第一个可见动作/声音/冲突 | 不能写成“开场要强” | S5/S6 |
| 单集节奏曲线 | S5/S8 | 开场抓人、中段升压、末尾卡点 | 不能变评分表 | S5/S8 |
| 钩子设计手艺 | S5 | 未回答问题、卡住动作、下一集兑现义务 | 不能只在 reviewer 里判断 | S5 |
| 台词功能 / 角色声线 | S6/S7 | 每句台词的攻击、防守、试探、威胁、揭露、选择等功能 | 不能让 S5 写满具体台词 | S7 |
| 视听表达 / page-to-screen | S6/S7 | 可见动作、身体、空间、声音、物件、当众后果 | 不能只在 reviewer 里事后喊 | S6/S7 |
| 海外市场语感 | S0/S3/S7 | 市场、称谓、角色说话方式、文化常识 | 不能只做到英文对白 | S0/S3/S7 |
| 合规 / 禁区 | S0/S10/S11 | 红线、灰区、用户禁区 | 不能成为创作发动机 | S0/S10/S11 |
| Story memory | S8/S11 | 已播/未播、角色状态、信息差、伏笔 | 不能让 S6 直接读厚 memory | S8/S3 |
| 作者自检 | S9 | 失败层级、返修范围、保留体验、禁止外形 | 不能给 writer 当输入 | 对应层 |
| 返修诊断 / rewrite report | S9/S11 | 失败类型、失败层、必须保留项、禁止破坏项、建议回滚动作 | 不能变成逐句正文补丁；不能跳过归层直接让 writer 重写 | 对应失败层 |
| 干净 reviewer | S10 | 致命阻断、应改风险、可选优化、回滚层、suggestion | 不能生产短剧感 | S11 转译 |
| 版本对比 | S11 | 与 V5、V22、V24、V25、上一版的全维度对比 | 不能给 writer 当输入 | S11 或对应层 |
| 用户 / 导演反馈 | S11 | 需求、边界、故事、场面、施工稿、正文、台词、连续性等归层 | 不能逐条变规则 | 对应层 |
| run log | 主控治理层 | 运行证据、产物路径、交接情况 | 不能进 writer；不能当质量证明 | 主控 |

## 3. 每层产物完整性

| 产物 | 必须回答 | 不能混入 | 不合格时 |
|---|---|---|---|
| `最小需求确认` | 源本、新壳、市场、语言、集数、力度、禁区、正文用途 | 源本分析、新剧情 | 回 S0 |
| `源本有效性包` | 为什么能追、能付费；钩子、压放、拉扯、DNC、高价值节点 | 新壳创作、正文桥段 | 回 S1 |
| `洗稿边界包` | 强继承体验、等效迁移、自由变量、禁用外形、必须删除 | 具体新剧情 | 回 S2 |
| `新本故事开发包` | 故事核、真相账本、错信、阶段骨架、分集功能、人物行动方式 | 高压场面完整外形、正文动作串 | 回 S3 |
| `场面承载原料包` | 谁压谁、谁旁观、谁被迫行动、压力来源、释放后果、候选动作、DNC | 完整节拍、台词、成片最后一帧 | 回 S4 |
| `单集短剧施工稿` | 一拍一拍行动链、开场、升压、压放、卡点、信息禁区、短剧手艺使用 | 流程证明、上游字段复读、reviewer 解释 | 回 S5 |
| `screenplay 正文` | 可看可听、人物行动、空间、身体、声音、台词功能、卡点 | 内部标签、分析说明、心理判断替代画面 | 回 S6/S7 |
| `polish 稿` | 删水词、保声线、强化动作/声音/身体/空间 | 改主结构、重设场面 | 回 S7 或上游 |
| `story memory checkpoint` | 真相、错信、已播/未播、伏笔、关系/权力状态 | 审美评价、短剧感打分 | 回 S8/S3 |
| `作者自检与返修路由` | 问题、严重度、回滚层、保留体验、禁止外形 | 把硬风险写成偏好 | 回 S9 |
| `返修诊断 / rewrite report` | 失败属于哪一层、为什么必须返修、返修时保什么、不能破坏什么 | 单句挑刺、泛泛“加强短剧感”、替 writer 直接写正文 | 回对应失败层 |
| `reviewer report` | issue、严重度、回滚层、suggestion | 逐句代写、生产短剧感、无限挑刺 | S11 归层 |
| `主控拍板` | 是否交付、为什么、横向对比、返修层、是否显著提升 | 只看 run log、只看 reviewer | 回对应层 |

## 4. 最关键的边界

### 4.1 分析层不能写戏

S1-S4 可以给：

- 体验功能；
- 故事状态；
- 压力来源；
- 可见后果方向；
- 禁止外形；
- 不可提前消费信息。

S1-S4 不能给：

- 完整高压戏；
- 具体台词；
- 最后一帧；
- 源本强桥段整串替换版。

### 4.2 S5 是创作施工层，不是审计层

S5 不是把上游字段复制成任务单。  
S5 必须把上游材料消化成“一集戏怎么发生”。

判断标准：

```text
读 S5 时应该像读一集短剧的粗稿，
不是读检查表、任务单或审计摘要。
```

### 4.3 S6 只能从施工稿写正文

S6 不能直接读：

- 源本拆文包；
- 洗稿边界包；
- 新本故事开发包；
- 场面承载原料包；
- story memory 全文；
- 外部标准库；
- reviewer report；
- 作者自检；
- run log；
- 旧版对比。

如果 S6 需要读这些，说明 S5 不合格。

### 4.4 检查层只回滚，不生产短剧感

S8/S9/S10/S11 可以判：

- 哪一层失败；
- 为什么失败；
- 应该回哪层；
- 修复方向是什么。

不能负责：

- 重新写戏；
- 替 writer 造短剧感；
- 每次都给“还能更好”的返修。

## 5. 下一步使用方式

新版 workflow spec 必须以这张矩阵为交接合同。

任何步骤如果写不清：

```text
吃什么；
产什么；
禁止什么；
下一层只能读什么；
失败回哪层。
```

就不能进入 workflow spec。
