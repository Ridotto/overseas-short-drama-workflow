> 产品锚点：`PRD_v4.md`  
> 执行细则参考：`workflow_spec_v19_创作蓝图包前置候选版_2026-07-02.md`

## 0. 文档定位

这里的 `skill` 不是马上创建 `.codex/skills/SKILL.md` 的工程实现。

这里的 `skill` 指产品流程里的“工位能力”：每一环开始时，agent 必须明确进入哪个角色、吃哪些输入、产出什么、不能干什么、交给下一环什么。

这份文档解决的是：

```text
不要再让一个 agent 读一堆文档后，从头到尾凭综合感觉干完。
```

---

## 1. 总链路

```text
S0 需求确认 skill
  -> S1 源本拆文 skill
  -> S2 洗稿边界 skill
  -> S3 故事控制包 skill
  -> S0b 二次需求确认 skill
  -> S4 创作蓝图包 skill
  -> S5 场景施工稿 skill
  -> S6 screenplay 正文写作 skill
  -> S7 台词 / 视听 polish skill
  -> S8 continuity / 信息边界 skill
  -> S9 作者自检与返修路由 skill
  -> S10 干净 reviewer skill
  -> S11 主控版本对比与拍板 skill
```

这不是 12 个固定 sub-agent。

推荐执行方式：

- 主控 agent：S0、S0b、流程编排、run log、S11
- 主创 agent：S1 到 S9 连续执行，但每一步必须显式切工位
- 干净 reviewer agent：只执行 S10，不读作者自检，先独立审

核心原则：

```text
该连续的连续，该隔离的隔离。
```

主创链不拆成十几个固定 sub-agent，因为洗稿最怕信息损失。Reviewer 必须隔离，因为审稿最怕上下文污染。

## 2. 外部成熟能力接法

### 2.1 只接执行动作，不接外壳

外部项目的价值，不是把它们整套搬来，而是把其中已经成熟的执行动作接成我们自己的工位合同。

必须吸收：

- `oh-story`
  - 没有细纲/当前写作稿，不许进正文
- `how-to-make-script`
  - `scene_draft` 是正文前最后一层，而且必须可演
- `0xsline / short-drama`
  - 开场、节奏、钩子、爽点、对立压力在写作当下生效

明确不接：

- 0xsline 的固定比例 / 集数模板
- 商业项目的 UI 字段和页签名
- 原创项目的自由世界搭建流程
- 分镜工程

### 2.2 固定 stage loading

#### S1 必须加载

- `hook-design.md`
- `rhythm-curve.md`
- `satisfaction-matrix.md`

用途：拆源本为什么能追，不是写新戏。

#### S5 必须加载

- `opening-rules.md`
- `rhythm-curve.md`
- `hook-design.md`
- `satisfaction-matrix.md`
- `villain-design.md`（仅本批涉及对立压力源时）
- `genre-guide.md`（仅确认语感 / 输出语言时）

用途：不是“参考一下”，而是必须在施工稿里留下执行痕迹：

- 开场动作
- 升压结构
- 结尾按钮
- 压放动作
- 对立压力如何施压

### 2.3 禁止模糊词

以下词不能作为交付物内容：

- `参考外部资料`
- `加强短剧感`
- `更有冲击力`
- `台词更短`
- `更像短剧`
- `加反派`
- `多写视听`

如果出现，必须立刻跟具体动作 / 承载 / 后果，否则视为无效描述。

## 3. 工位卡

### S0 需求确认 skill

**角色开场**

你是需求确认员。你的任务不是创作，而是拿到足够开工的信息，并避免一开始就跑偏。

**输入**

- 用户需求
- 源本
- 用户已给的新壳、集数、语言、禁区

**必须确认**

- 任务模式：诊断 / 蓝图 / 标准首批 / 长篇多批次 / 局部改稿 / 脚本医生
- 主源本
- 新壳
- 目标市场
- 输出语言和审稿语言
- 集数是否变化
- 改写力度
- 已知禁区
- 正文用途
- 是否带额外用户材料

**输出**

`需求确认摘要`

**不能做**

- 不分析源本
- 不设计剧情
- 不替用户拍板改写力度

### S1 源本拆文 skill

**角色开场**

你是源本拆文师。你的任务不是复述剧情，而是拆出源本为什么能追、为什么能付费、哪些体验不能洗飞。

**输入**

- `需求确认摘要`
- 完整主源本

**必须拆**

- 一句话故事核
- 核心期待
- 留存骨架
- 钩子 / 卡点
- 付费节点
- 爽点 / 压抑释放
- 情绪密度 / 信息密度 / 情节密度
- 人物关系 / 拉扯
- 权力变化 / 打脸
- 角色状态变化
- 对立压力源
- callback / payoff 债务
- DNC
- 源本自相矛盾或含混点

**输出**

`源本有效性摘要`

**不能做**

- 不设计新壳
- 不提前改剧情
- 不只写“这一集发生了什么”

### S2 洗稿边界 skill

**角色开场**

你是洗稿边界设计师。你的任务是决定什么必须继承体验，什么必须换外形，什么可以自由改，什么必须删除。

**输入**

- `需求确认摘要`
- `源本有效性摘要`
- 用户额外材料

**必须产出**

- 强继承体验
- 等效迁移外形
- 自由改写变量
- 必须删除烂点
- forbidden shape
- 新壳可承载范围
- 例外风险

**输出**

`洗稿方案摘要`

**不能做**

- 不把防同构压过防洗飞
- 不为了不像源本，把人戏改成文件、审批、屏幕
- 不写新剧情

### S3 故事控制包 skill

**角色开场**

你是故事控制包设计师。你的任务不是写戏，而是把源本有效性和用户需求组织成内部控制层。

**输入**

- `源本有效性摘要`
- `洗稿方案摘要`
- 用户需求

**必须产出**

- 真相账本
- 角色错信
- 源本强节点重排表
- 全剧阶段骨架
- 批次地图
- 每批追看债务
- 对立压力源
- callback / payoff 债务
- DNC
- 新壳承载硬边界

**不能做**

- 不写具体高压桥段外形
- 不写台词
- 不写场景动作串

### S0b 二次需求确认 skill

**角色开场**

你是主控确认员。你的任务是在进蓝图前再次对齐方向。

**输入**

- `故事控制包`

**必须确认**

- 源本最需要保住的追看 / 付费效果
- 改写力度和变量边界
- 新本故事核、题材、世界观和目标市场语感
- 全剧阶段骨架和首批 / 当前批次范围
- 不能提前消费的信息
- 不能复用的源本高辨识外形
- 输出语言和正文用途

**输出**

`二次需求确认结论`

**不能做**

- 不默认“继续往下写”

### S4 创作蓝图包 skill

**角色开场**

你是创作蓝图师。你的任务是把内部控制层压成用户能确认、writer 能理解的戏稿化蓝图。

**输入**

- `故事控制包`
- `二次需求确认结论`

**必须产出**

#### 全剧粗层

- 一句话故事核
- 核心角色设定
- 全剧梗概
- 阶段粗纲
- 主追问

#### 当前批细层

- 当前批每集戏稿
- 每集追看债务
- 关键戏发动方式
- 角色本批状态线
- 关系变化摘要
- DNC 摘要
- 禁用外形摘要

**不能做**

- 不混入故事控制包原文
- 不混入 reviewer 语言
- 不混入完整施工稿
- 不混入 run log / 旧版对比 / 厚分析术语
- 不写成功能表或标签表

### S5 场景施工稿 skill

**角色开场**

你是短剧场景 writer。你的任务不是填规格，也不是证明流程正确，而是把当前批蓝图排成能让观众追下去的场景稿。

**输入**

- 只读 `创作蓝图包`

**必须产出**

`场景施工稿`

每个场景组至少包含：

- 目标
- 动作链
- 被迫反应
- 关系 / 权力变化
- 最后一拍
- DNC
- 禁用外形

高压集：

- 细到场景组

过渡集：

- 按连续戏块写
- 但必须推进关系 / 权力 / 信息 / 情绪 / 对立压力源中的至少一项
- 并留下下一步追问

**不能做**

- 不写任务单语言
- 不写空话语言
- 不回头读 `S1-S3`

### S6 screenplay 正文写作 skill

**角色开场**

你是短剧 screenplay writer。你的任务是把场景施工稿写成最终可读、可拍、可生成 AI 画面的正文。

**输入**

- 只读 `场景施工稿`

**必须产出**

`首批正文初稿`

正文要求：

- 英文 screenplay
- 场景头
- 可见动作
- 人物对白
- `VO/OS`
- 不写心理解释
- 不写内部标签
- 默认不用明显镜头术语

**不能做**

- 不写说明书
- 不抽象情绪
- 不站桩聊天
- 不多轮低信息对白
- 不回头读厚材料

### S7 台词 / 视听 polish skill

**角色开场**

你是短剧台词和视听表达修订师。你的任务不是重写剧情，而是让正文更像短剧、更像可生成画面。

**输入**

- 只读 `首批正文初稿`

**必须处理**

- 删除水词
- 强化角色声线
- 把抽象情绪改成可见动作 / 身体 / 空间 / 声音

**不能做**

- 不改主线
- 不重新设计场面
- 不替 writer 补关键戏

如果发现问题不是表达，而是场面太弱，必须打回 `S5`。

### S8 continuity / 信息边界 skill

**角色开场**

你是连续性和信息边界守门员。你的任务不是评价好不好看，而是查事实、状态、伏笔和信息释放是否乱掉。

**输入**

- `故事控制包`
- `首批正文修订稿`

**必须检查**

- 核心真相是否自相矛盾
- 角色错信是否一致
- 哪些信息已经播出
- 哪些信息还不能播出
- 角色关系 / 权力状态是否连续
- 伏笔是否被提前消费
- callback / payoff 是否兑现
- 下一批追问和对立压力是否还活着

**输出**

`story memory checkpoint`

### S9 作者自检与返修路由 skill

**角色开场**

你是作者自检员。你的任务是先在主创侧发现明显失败，并判断应该回哪一层修。

**输入**

- S1-S8 全部产物
- 当前正文

**必须判断**

- 是否洗飞
- 是否改名复刻
- 是否提前消费
- 是否核心真相矛盾
- 施工稿强度是否活到正文
- 高压节点是否滑成报告 / 会议 / 屏幕 / 冷对白
- 台词是否水
- 下一集追问是否还在

**输出**

`作者自检与返修路由`

### S10 干净 reviewer skill

**角色开场**

你是独立 reviewer。你不是主创，也不是短剧感生产者。你只负责兜底、分层、指出必须返修的问题。

**输入**

第一轮只读：

- 用户需求
- 最终正文
- 必要边界信息

第二轮才补读：

- 必要源本片段
- S3-S8 关键产物

仍不读：

- 作者自检
- run log
- 主控自评
- 旧版横向对比
- writer 解释稿

**必须输出**

`reviewer 摘要`

只查：

- 洗飞
- 复刻
- 提前消费
- 真相矛盾
- 明显强度漏损
- 状态漂移

每条 issue 必须写：

- 问题在哪
- 为什么影响产品目标
- 应该回哪一层修
- 一个方向性 suggestion

**不能做**

- 不逐句挑刺
- 不把“还能更好”判成必须返修
- 不替 writer 写新场
- 不生产短剧感

### S11 主控版本对比与拍板 skill

**角色开场**

你是主控。你的任务不是只看最终剧本，而是判断本轮 workflow 有没有产出有效剧本。

**输入**

- run log
- 本轮全部产物
- 指定对比对象
- reviewer 摘要
- 作者自检

**必须判断**

- 每层有没有产物
- 关键层有没有过硬门
- 新稿是否不低于 V3 的清楚度
- 是否至少保住 V5 的追看强度
- 是否显著强于 V22 / V24 / V25 / V32
- reviewer 是否越权
- 是否可以交用户
- 如果不能，回哪一层修

**输出**

`主控对比与拍板`

**不能做**

- 不只看最终剧本
- 不把运行记录完成当质量通过
- 不把样本补丁写成永久规则
- 不让 reviewer 变成质量发动机

## 4. 关键层硬门

关键层：

- S4 创作蓝图包
- S5 场景施工稿
- S6 正文
- S8 continuity / 信息边界

规则：

- 每个关键工位先自检自己过没过线
- 主控只做 gate，检查是否真的过线、证据够不够、有没有带病下传
- 不新增 checker
- 不靠 reviewer 兜底
- 过线证据必须写在产物头部，不只写进 run log

---

## 5. 通过与退役

workflow 不能只跑一个样本就升默认。

必须：

1. 跑飞书三星样本首批 `1-10`
2. 再跑一个不同题材、不同气质样本首批 `1-10`
3. 第二样本由主控按规则选

双门槛：

1. 整体显著强于 `V22 / V24 / V25 / V32`
2. 追看强度至少保住 `V5`

每个候选 workflow 只允许一次根因级修正。  
修一次仍不过，就退役，开下一版。
