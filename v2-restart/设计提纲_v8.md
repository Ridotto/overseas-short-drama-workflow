## 一、核心定义

这个产品不是评分器，也不是分析报告工具。它最终要产出一个新剧本。

它做的事情是：

**拿一个已经验证过的爆款源本，读懂它为什么能抓住观众，再把这套有效性迁移到新壳里。**

这里保留的不是人物名字、具体场景、原句对白，也不是机械照抄第几集几个钩子。

真正要保留的是：

**源本有效性骨架**。

也可以口语叫“命脉”，但主文档里不再把“命脉”写成一个容易误解的元素清单。后续统一用：

- **源本有效性骨架**：源本如何让观众继续看、愿意付费。
- **迁移基准**：改写时默认要对照和继承的部分。
- **变量迁移策略**：每类内容到底是强继承、等效迁移、自由改写，还是必须删除。它不是锁死不动，而是标明这类内容该怎么迁移。

一句话原则：

**分析可以很细，但分析结果不是机械命令。源本的留存骨架默认继承；只有新壳、集数、烂点、目标市场或用户要求造成冲突时，才做等效调整。**

## 二、主流程：7 个迁移动作

7 个动作保留为主干，但 v8 调整了表达：它不是填表流程，而是判断与创作流程。

```
源本 + 用户需求
   ↓
动作1：分析源本有效性骨架
   ↓
动作2：把源本内容分层
   ↓
动作3：形成迁移基准与变量迁移策略
   ↓
动作4：判断新壳/新集数能否承载
   ↓
动作5：迁移、等效替换或删除
   ↓
动作6：生成洗稿方案
   ↓
动作7：首批验证有没有洗飞
   ↓
新本首批 + 改动说明
```

### 动作1：分析源本有效性骨架

要回答：这个源本为什么能让观众继续看、愿意付费？

分析不是为了写报告，而是为了找改写时不能乱动的迁移基准。

分析按五层看：

1. 留存骨架
2. 情绪爽点
3. 主线推进
4. 人物压力
5. 干净度

### 动作2：把源本内容分层

源本里的内容分四类：

| 类别 | 含义 | 改写态度 |
|---|---|---|
| 有效性骨架 | 让源本能抓人的运行方式 | 默认继承 |
| 外壳 | 题材、身份、场景、世界观、具体桥段包装 | 可改写 |
| 水分 | 注水、拖沓、无效过渡、重复桥段 | 删除或压缩 |
| 烂点 | 假动机、逻辑洞、虚假悬念、不适合目标市场的设定 | 修掉，不继承 |

关键不是问“这个东西要不要保留”，而是问：

**它到底是在制造留存，还是只是源本恰好这么写？**

### 动作3：形成迁移基准与变量迁移策略

v7 里叫“减到 minimum”，v8 不再主推这个说法。

更准确的说法是：

**把源本拆出的内容，标成不同迁移策略。**

每个变量都要判断：

- 强继承
- 等效迁移
- 自由改写
- 必须删除

这样既不会脚本化，也不会让模型瞎飞。

### 动作4：判断新壳/新集数能否承载

要回答：用户想换的新壳、新题材、新市场、新集数，能不能承载源本的迁移基准？

承载不了，不代表放弃源本，而是进入等效迁移：

- 原形式不适合，就换形式。
- 原功能不能丢，要找新壳里成立的替代方式。

### 动作5：迁移、等效替换或删除

改写不是只有“保留”和“替换”两种。

实际有四种动作：

| 动作 | 用在什么情况 |
|---|---|
| 强继承 | 源本追看功能链、情绪波形、主线期待等核心骨架 |
| 等效迁移 | 原形式不适合新壳，但观众体验功能必须保 |
| 自由改写 | 外壳、人物身份、场景、职业、组织、对白、桥段包装 |
| 删除/修复 | 水分、烂点、假动机、虚假悬念、AI 味、不适合目标市场的设定 |

### 动作6：生成洗稿方案

洗稿方案不只是“换成什么题材”。

它至少要说清：

- 新壳是什么；
- 哪些变量强继承；
- 哪些变量等效迁移；
- 哪些自由改写；
- 哪些水分烂点删除；
- 集数如果改变，留存节奏如何重排；
- 首批交付覆盖哪个留存小闭环。

方案给用户确认后，才开始写首批。

### 动作7：首批验证有没有洗飞

首批不是为了证明全剧已经成了，而是证明方向没洗飞。

验证分两层：

1. **洗稿验证**：源本有效性有没有迁过去。
2. **剧本质量验证**：新本自己是否自洽、好看、不假、不 AI。

没过就回到对应动作修，不是继续往后写。

## 三、分析环节：五层分析表

分析要细，但调用要分层。模型不应该一次性盯几十条规则，而是围绕五层逐层判断。

| 层 | 分析对象 | 看什么 | 迁移时怎么用 |
|---|---|---|---|
| 留存骨架 | 开头、每集钩子、关键卡点、付费/留存节点 | 源本怎样让观众不划走、不弃剧、愿意继续看 | 默认强继承追看功能；节奏位置按改写力度处理 |
| 情绪爽点 | 情绪峰谷、压抑释放、爽点兑现、高潮递进 | 源本怎样压住观众情绪，再释放或反转 | 继承情绪波形和兑现周期，不机械照数值 |
| 主线推进 | 核心期待、信息差、秘密揭露、每集/每场新进展、信息增量、推进效率 | 观众一直在等什么，什么时候给答案，什么时候升级问题；每集/每场到底推进了什么 | 保住期待链和信息密度，防止新本散掉、空转或注水 |
| 人物压力 | 主角驱动力、关系功能、反派层级、压迫升级 | 谁推动故事，谁制造压力，谁放大爽点 | 人物外壳可换，叙事功能要有人承接 |
| 干净度 | 水分、烂点、AI 味、非视听语言、不可拍描述、目标市场不适配 | 哪些内容拖、假、重复、不适合、拍不出来 | 标为删除或修复，不进入迁移基准 |

这五层不是五份报告，而是一套读源本的顺序。

## 四、改写变量与迁移策略表

这是 v8 新增的核心表。它回答“什么默认不动，什么可以动，什么必须动”。

迁移策略不是锁死。轻度换壳、集数不变、用户明确要贴近源本时，可以高度复刻源本的节奏和卡点；中重度改写、集数变化、新壳承载方式不同，或用户要求变化时，就保留追看功能，位置和形式做等效迁移。

| 变量 | 默认等级 | 为什么 | 什么时候允许调整 |
|---|---|---|---|
| 留存/追看功能链 | 强继承 | 这是源本让观众继续看的核心方式 | 轻度换壳可高度贴近源本；中重度改写时功能不能丢，形式可换 |
| 关键卡点位置 / 钩子形式 / 节奏落点 | 条件强继承 / 等效迁移 | 位置和形式服务追看功能，不是硬数字 | 集数不变、改动很轻、用户要求复刻时可贴近；新壳、集数、市场、烂点或用户要求导致冲突时等效迁移 |
| 付费/关键留存节点 | 强继承 | 决定观众是否继续投入 | 集数改变时按阶段比例重排 |
| 情绪峰谷走势 | 强继承 | 观众追剧靠情绪波形，不靠平铺剧情 | 数值不照抄，但压、爽、虐、甜、反转的走势要对应 |
| 爽点兑现周期 | 强继承 | 拖太久泄气，太早掏空也泄气 | 可按新集数重排，但不能无故拖延或提前清空 |
| 主线期待链 | 强继承 | 观众一直追的是核心问题 | 只有深度重写时可重构，但必须有新的同等期待链 |
| 主角处境变化节奏 | 强继承 | 主角被压、反击、升级的节奏影响代入和爽感 | 可换事件，不应丢阶段功能 |
| 反派压力升级 | 强继承 / 等效迁移 | 反派强度决定爽点强度 | 反派身份可换，压力阶梯要在 |
| 信息差结构 | 等效迁移 | 谁知道什么，决定悬念和主动性 | 表现形式可换，信息控制不能乱 |
| 钩子类型 | 等效迁移 | 钩子类型服务留存功能 | 新题材不适合原类型时换同功能钩子 |
| 爽点呈现形式 | 等效迁移 | 形式可以不同，但压抑-释放要成立 | 根据题材、市场、人物关系替换 |
| 关键道具/秘密/证据 | 等效迁移 | 常用于证明、揭露、反杀、引爆 | 道具可换，剧情功能要保 |
| 人物关系功能 | 等效迁移 | 压迫源、裁决者、见证者、保护者、背叛者等功能不能缺 | 人物身份可换，功能不能无故消失 |
| 题材 / 世界观 / 背景 | 自由改写 | 这是换壳主要动的地方 | 不能破坏承载逻辑 |
| 身份 / 职业 / 组织 / 地点 | 自由改写 | 用来适配新壳和目标市场 | 必须服务新壳自洽 |
| 具体桥段包装 | 自由改写 | 避免看起来像照抄 | 关键情绪和信息功能要对上 |
| 台词和表达 | 自由改写 | 新本必须像新剧本 | 不能保留 AI 味、翻译腔、解释腔 |
| 集数与篇幅 | 条件改写 | 用户可能要求扩写或缩写 | 重排留存节奏，不是重写一切 |
| 水分 / 重复 / 无效过渡 | 必须删除 | 留着会拖垮新本 | 无需等效迁移 |
| 假动机 / 逻辑漏洞 / 虚假悬念 | 必须修复或删除 | 这是烂点，不是有效性 | 不能因为源本有就继承 |
| 目标市场不适配设定 | 必须替换 | 出海时可能不成立或反噬 | 换成目标市场能理解的等效压力 |

注意：强继承不等于照抄。强继承的是追看功能和观众体验骨架，不是原文句子、原桥段或硬性位置数字。

## 五、力度与维度：不是三档模式，是变量控制

用户的需求不是简单“轻度/中度/重度”三档，而是很多变量同时变化。

示例：

- 世界观大改，但留存节奏尽量不动。
- 人物身份全换，但人物关系功能不动。
- 集数从 40 改 80，卡点位置要按新长度重排。
- 爽点从身份碾压换成情感爆发，但兑现周期仍贴源本。
- 目标市场改成欧美，部分中式压力要替换成当地能理解的压力。

因此，“力度”只决定每个动作走多深，不决定所有变量一起松开。

| 修改力度 | 默认处理 |
|---|---|
| 轻度换壳 | 留存骨架强继承；外壳大改；钩子/爽点形式尽量等效 |
| 中度改写 | 留存骨架保大惯性；部分主线顺序可调整；人物关系功能必须承接 |
| 深度重写 | 可重构剧情顺序和部分机制，但必须保住源本最核心的商业发动机 |
| 集数改编 | 不是独立模式，可叠加在任何力度上；重点是重排节奏，不是简单加戏删戏 |

## 六、用户参与点

用户不是来填表的，但必须在关键节点拍板。

1. **需求对齐**：新壳、目标市场、集数、改写力度、不能动的偏好。
2. **迁移基准确认**：工具给出变量迁移策略，用户确认或调整。
3. **洗稿方案确认**：方向对了才写首批。
4. **首批交付确认**：首批没洗飞，才继续。
5. **续写确认**：分批推进，避免整本方向错了再返工。

工具负责读源本、做判断、写新本、解释改动。

用户负责方向、偏好、取舍、最终拍板。

---

## 七、首批交付原则

首批不固定 3 集、5 集或 10 集。

默认原则：

**首批要覆盖第一个完整留存小闭环。**

一个完整留存小闭环至少包括：

1. 开头抓人；
2. 核心压力或期待建立；
3. 至少一次阶段性兑现，或一次足够强的悬置；
4. 新一轮追剧问题成立。

如果源本前 3 集就完成这个闭环，首批可以是 3 集。

如果源本节奏更慢，或目标集数改变，首批可以更长。

判断标准：观众看完首批，既觉得已经看到一点兑现，又想知道接下来会怎样。只有兑现没有新问题，会清空追剧动力；只有问题没有兑现，会损失信任。

## 八、自检环节：洗飞是硬底线

自检不是评分器，不合成总分。

自检有两层标准：硬底线是源本有效性是否迁移，洗飞就失败；质量标准是新本自身是否自洽、好看、不假。两者同时检查，但前者优先级更高。

| 检查项 | 级别 | 失败后回哪 |
|---|---|---|
| 源本有效性骨架是否迁移 | 硬失败 | 回动作1-5 |
| 强继承项是否被无故改掉 | 硬失败 | 回动作3/5 |
| 新壳是否承载得住 | 硬失败 | 回动作4 |
| 钩子/卡点是否仍有同等留存功能 | 硬失败或重警告 | 回动作5 |
| 情绪波形是否塌掉 | 重警告 | 回节奏编排 |
| 爽点是否有蓄力和兑现 | 重警告 | 回重写 |
| 主线期待是否断掉或空转 | 硬失败或重警告 | 回动作1/5 |
| 人物动机和行为是否站得住 | 硬失败 | 回动作4/5 |
| 水分烂点是否被继承 | 硬失败或重警告 | 回动作2/5 |
| 是否硬堆钩子爽点 | 硬失败 | 回重写 |
| 伏笔、秘密、信息差是否断线 | 重警告 | 回连续性追踪 |
| 角色状态、关系、时间线是否漂移 | 重警告 | 回连续性追踪 |
| 镜头感/可拍性是否过关 | 硬失败或重警告 | 回场景重写 |
| 台词是否 AI 味、解释味、翻译腔 | 警告到硬失败 | 回表达重写 |
| 用户口味和取舍 | 用户判断 | 停在人审点 |

镜头感/可拍性的意思是：剧本不能用抽象概念替代场面。关键情绪、大事件、人物转变，都必须能被摄影机拍出来。比如不能只写“角色突然黑化”“发生一场惨绝人寰的战争”“两人关系彻底破裂”。必须写出观众能看到的动作、选择、冲突、后果或场面变化。

短剧标准库的数量、比例、频率只能做参照和异常提醒，不能压过源本。

## 九、多 agent 原则

这个产品可以用多 agent 分析和自检，因为源本长、维度多，单模型一次性盯所有东西容易散。

但多 agent 不是产品主干，只是执行方式。

推荐分工：

- 留存骨架 agent：看开头、钩子、卡点、留存节点。
- 情绪爽点 agent：看情绪波形、爽点兑现、高潮递进。
- 主线人物 agent：看期待链、信息差、主角驱动力、反派压力。
- 质量自检 agent：看新壳自洽、水分烂点、AI 味、连续性。

必须有一个总编角色合并判断。

**可以并行分析，不能并行拍板。**

否则多个 agent 会各自放大自己的规则，最后又变成规则怪谈。

## 十、参考资产如何使用

参考资产不单独建主文档。它们在 v8 里的归位如下：

| 参考资产 | 用途 | 注意 |
|---|---|---|
| short-drama 标准库 | 钩子、爽点、节奏、卡点、反派、题材、出海参考 | 做参照，不压过源本 |
| oh-story | 拆文报告结构、情节节点、写作手法、伏笔/角色/时间线追踪 | 借结构，不照搬成报告产品 |
| how-to-make-script | 分层诊断、自适应质检、硬失败/弱项分离 | 借方法，不照搬完整质检格式 |
| shortdrama-pipeline | 人工审核门、部分量化质检、suggestion 思路 | 偏生产质检，只做辅助 |
| Dramatron | 先骨架后正文的分层思路 | 只借分层思路，不当逆向拆解工具 |
