## 2. 当前人话判断

现在已经确认：

```text
不是缺短剧概念。
不是 reviewer 不够努力。
不是 run log 本身直接污染 writer。
不是继续修某一个剧本就能解决。
```

当前总根因是：

```text
源本为什么好看、外部短剧手艺、V5 的场面发动机，
没有稳定变成 writer 当下可执行的生产动作。
```

但注意：

```text
当前不是把项目收窄成“只管短剧感”。
短剧手艺只是必须生效的一条生产线，
不能挤掉源本有效性、钩子卡点、爽点压放、人物关系、拉扯、信息差、反派压力、连续性和新壳承载。
```

具体断口：

1. S3/S4 仍可能把源本高辨识外形塞进新本骨架或场面原料；
2. S4 仍可能替 writer 排完整戏；
3. S5 仍可能吃到半成品场面、旧外形或流程证明；
4. 0xsline 的开场/节奏/钩子/爽点仍可能只停在“参考加载”，没有变成每集写作动作；
5. S8 记录信息重复消费后可能软放行；
6. S9 作者自检可能把硬风险写成“主控偏好”；
7. run log / complete 语义容易被误读成质量通过。

## 3. 当前候选 workflow

当前最新候选是：

```text
workflow_spec_v16_全能力交接候选版.md
```

它继承：

- `workflow_spec_v2.md` 的完整创作链；
- `skill_chain_spec_v1.md` 的主控 + 主创链 + 干净 reviewer；
- V5 证明有效的“场面发动机”；
- 0xsline 的阶段化短剧手艺加载；
- V25 暴露出的 S3/S4/S5/S8/S9 交接问题。
- 三份重收口文件里的全版本能力回收、能力归层和交接链草案。

它不做：

- 不改 PRD 主定义；
- 不把 v14 正式化；
- 不继续修 V25 正文；
- 不把 reviewer 变成质量发动机；
- 不把 0xsline 当原创短剧流程照搬；
- 不把每条导演反馈写成新规则。

## 6. 执行时必须记住

如果 agent 声称“按 workflow 跑完”，必须同时给出：

- 本轮使用哪个 workflow；
- 每个工位产物在哪；
- 哪些步骤只是 execution complete；
- 最终创作判断是什么；
- 是否比 V5 / 上一版更强；

- writer 没有读取 run log / reviewer / 作者自检 / 旧版对比 / 临时诊断；
- reviewer 没有变成质量发动机；
