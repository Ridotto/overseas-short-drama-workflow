## 3. 当前不要再误判的东西

现在已经确认：

```text
不是缺短剧概念。
不是 reviewer 不够努力。
不是 writer 还在拿任务补戏。
不是中间产物又散了。
不是继续磨 V40 正文就能解决。
```

当前最新根因不是单个短剧技巧缺失，而是：

```text
PRD 写的是产品形态和创作能力，
但从 PRD 编译到 spec / workflow 时，
容易滑成内部字段、gate、reviewer、打回和 S 编号执行合同。
```

更具体一点：

- 用户可见主确认文档必须是商业项目形态的 `创作蓝图包`
- 内部产物必须稳定支撑好剧本，而不是靠 reviewer 兜底
- 质量责任必须写进产物定义，而不是写成无限打回机制
- 外部项目和商业项目只能校准写法与形态，不能覆盖 PRD 和洗稿边界
