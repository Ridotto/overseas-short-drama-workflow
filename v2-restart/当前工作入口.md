## 2. 当前用户层怎么跑

自然语言示例：

```text
源本在 input/my-source-script.txt。
请按当前主链，把它改写成海外女频短剧。
新壳方向：豪门医疗复仇。
正文语言：English。
先做标准首批 1-10 集。
先做源本导入、洗稿方向和创作蓝图确认，不要直接写正文。
```

高级用户层命令：

```text
/rewrite-start
/rewrite-blueprint
/rewrite-write
/rewrite-polish
/rewrite-review
/rewrite-continue
/rewrite-export
/rewrite-status
```

这些是用户层别名，不是底层工位重命名。

## 3. 当前内部主链

```text
shortdrama-main-controller
-> source-import
-> short-drama-write(/write-from-source /plan /characters /outline /episode)
-> dialogue-polish
-> review
-> clean reviewer
-> batch-state 或 export
```

实际文件含义：

1. `skills/shortdrama-main-controller/SKILL.md`
   - 接用户自然语言和 `/rewrite-*` 命令；
   - 选择蓝图模式、标准首批、多批次续写或局部返修；
   - 管理确认点；
   - 调度生产链；
   - 归因返修责任层。

2. `skills/source-import/SKILL.md`
   把短剧源本导入成源本库，产出源本有效性、事件账本、人物功能、爽点钩子、付费压力、可迁移结构、禁抄边界、新壳迁移建议和写稿交接包。

3. `skills/short-drama-write/SKILL.md`
   读取源本交接包，执行商业项目包、角色、分集执行包、正文、台词精修、review、续批状态和导出。

4. `contracts/short_drama_form_lock_v1.md`
   锁最终形态：必须是短剧，不得滑成电视剧、网剧、小说、泛型 screenplay 或审稿导向文本。

5. `vendor/`
   保留外部原件和追溯证据。当前产品不直接跑 vendor 原入口。

## 7. run log 规则

任何执行都必须在新剧项目目录下留下 `run_log.md`。

run log 必须写：

- 使用的入口 skill；
- 源本库；
- 新剧目录；
- 已执行命令；
- 已产出文件；
- 未执行步骤；
- 当前质量判断。

没有 run log，不能声称按当前主链完整执行。
