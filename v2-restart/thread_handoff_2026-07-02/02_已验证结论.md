# 已验证结论

以下结论已经被真实样本或真实执行反复证明，不要再从头争：

## 1. 中间层必须成戏

已经证明：

- 只有“分析结果 + 功能表”，writer 会补成平戏
- `S4` 必须是戏稿蓝图
- `S5` 必须是场景施工稿

## 2. writer 不能吃厚材料

已经证明：

- writer 回头读 `S1-S3` 厚材料，只会更乱
- 正确关系是：
  - `S4` 给蓝图
  - `S5` 给施工
  - `S6` 只吃 `S5`

## 3. reviewer 不能当质量发动机

已经证明：

- reviewer 适合查硬伤
- 不适合负责生产短剧感
- 不适合无限打回去把剧本越改越程序化

## 4. v19 方向成立

`V33` 证明：

- 在飞书三星样本上
- `故事控制包 -> 创作蓝图包 -> 场景施工稿 -> 正文`
- 第一次真正落地了

`V40` 证明：

- 在第二个不同题材样本上
- 这条链依然能把中间层落成戏
- 不会自动滑回功能表和说明书

## 5. 当前不是“链断了”

当前已经不是：

- 中间层散掉
- writer 拿任务补戏
- reviewer 兜底一切

当前更像：

> 这条链已经会“写对”，但还不会稳定“写爆”。
