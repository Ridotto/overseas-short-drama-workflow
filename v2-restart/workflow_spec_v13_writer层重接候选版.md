# workflow_spec_v13_writer 层重接候选版

状态：候选版，未正式化。

日期：2026-07-01

## 0. 本版解决什么

本版不改 `PRD_v4.md`，不推翻 `workflow_spec_v2.md` 的完整创作链。

它只修一个执行层问题：writer 不能直接拿上游规格写正文。writer 必须先把本集排成“短剧施工稿”，再从施工稿写正文。

过去几版的核心失效不是“上游没分析”，而是很多短剧能力停在分析层、检查层或参考资料里，没有在 writer 写正文的那一刻变成写作动作。

## 1. 本版不改什么

继续保留：

- 源本分析
- 洗稿边界
- 核心真相 / 追问兑现
- 分集骨架
- 场面承载
- 连续性追踪
- 作者自检
- 独立 reviewer

但 reviewer 降权：reviewer 是兜底和调优，不是短剧感发动机。

## 2. Scene Carrier 收窄为原料层

`Scene Carrier / 场面承载` 只负责告诉 writer 这一场有哪些原料，不能替 writer 把戏排完。

它必须给：

- 源本这个节点为什么有效；
- 本集要保住的追看功能；
- 新壳下可用的事件载体；
- 哪些源本外形不能照抄；
- 哪些事实、关系、情绪和追问不能提前消费；
- 已知降级风险。

它不应该给：

- 完整正文节拍；
- 台词口味；
- 逐句动作安排；
- reviewer 式检查清单。

一句话：分析给事实和边界，writer 负责把它排成戏。

## 3. Writer 必须先写“单集短剧施工稿”

每一集写正文前，writer 先写一份短剧施工稿。

施工稿不是表，不是任务单，不是“开场钩子=xxx”的填空。

施工稿要像一集戏的粗稿，至少说清：

- 开场如何立刻抓人；
- 中段压力如何一步步升高；
- 爽点如何先压再放；
- 关系、权力、信息如何发生变化；
- 结尾停在哪个画面、动作或声音；
- 哪些信息绝对不能提前消费。

高压节点必须写成一拍一拍的演出：谁做了什么、谁被迫反应、谁失去什么、谁看见了、下一拍为什么更糟。

## 4. Writer 写施工稿时加载短剧手艺

短剧手艺不能只放在资料池或 reviewer 里。它要在 writer 写施工稿的时候被使用。

本阶段至少加载：

- 开场法：非首集先回应上集悬念，再立刻制造本集张力；
- 单集节奏：开头抓人，中段升压，结尾释放或硬切；
- 爽点压放：先让压力和看衰成立，再释放；
- 钩子设计：让观众必须知道答案，但停在答案前或答案刚翻面的一拍；
- 台词功能：攻击、防守、威胁、揭露、选择、反击，不能水聊；
- 视听表达：能用动作、空间、身体、声音、物件、当众后果写出来的，不用解释性对白替代。

这些不是新增规则表，而是 writer 写施工稿时的工作方式。

## 5. 强度轴只给 writer 自检

writer 写完施工稿后，自己用强度轴过一遍：

- 更强：即时、肉身、当众、主动、可见后果；
- 更弱：隔屏、推迟、转述、机制、报告播报。

这把尺只用于防止 writer 滑回流程播报。

它不能变成 reviewer 的逐句打分表。

## 6. 正文从施工稿写，不逐条复刻施工稿

正文必须从施工稿写出来，但不要求逐条照抄施工稿。

允许 writer 在正文中把戏写得更顺、更狠、更自然。

不允许：

- 把施工稿变成正文里的内部标签；
- 把施工稿逐条机械翻译成说明书；
- 把施工稿里的强度写丢；
- 为了对齐施工稿，牺牲正文自然度。

正文验收重点是：施工稿里的强度、追问、信息边界和卡点是否活在正文里。

## 7. 作者自检改看“施工稿是否活下来”

作者自检不再主要证明自己写了多少步骤。

作者只回答：

1. 本集施工稿是不是像戏，不像任务单？
2. 正文有没有保住施工稿里的核心强度？
3. 高压节点有没有滑成报告、会议、屏幕或冷对白？
4. 结尾有没有停在一个可追的画面、动作或声音？
5. 是否提前消费了后续事实、关系、情绪或追问答案？

如果第 2、3、4 条失败，作者先返修，不交给 reviewer。

## 8. Reviewer 降权

Reviewer 只查：

- 是否洗飞源本有效性；
- 是否改名复刻；
- 是否突破用户硬边界；
- 是否提前消费；
- 是否有明显强度漏损；
- 是否存在核心真相自相矛盾。

Reviewer 不做：

- 逐句台词口味审；
- 逐条施工稿 fidelity 检查；
- 替 writer 生产短剧感；
- 把“还能更好”升级成阻断。

Reviewer 的 issue 必须说清楚回哪层修：边界、真相、分集骨架、场面承载、施工稿、正文。

## 9. 运行要求

如果用本候选版跑样本，manifest 必须新增记录：

- 本次使用的 workflow spec 是 `workflow_spec_v13_writer层重接候选版.md`；
- 每集是否有 writer 施工稿；
- 作者是否完成施工稿存活自检；
- reviewer 是否只按降权口径审查；
- 是否出现“施工稿像任务单”或“正文写丢施工稿强度”的失败。
