## 0. 这版到底合什么

v9 不重新发明流程。

它把三类已经摆在手上的东西按正确权重合起来：

| 来源 | 在 v9 里的位置 | 不再误用成什么 |
|---|---|---|
| V3 | 方向底线：主线清楚、读起来不乱、豪门商战壳基本站得住 | 不把 V3 当成最终质量上限 |
| V5 | 主工作流骨架：源本拆文包 -> beat sheet -> 场面迁移卡 -> episode map -> scene packet -> 正文 | 不只抽走“关键场面卡”这个外壳 |
| 当前版 / V8 | 执行纪律：manifest、作者问题清单、干净 reviewer、主控版本对比、manifest 不等于质量通过 | 不把 V8 的薄链路当主创作链 |
| PRD v4 | 产品定义：洗稿，不是从零原创；不洗飞、不改名复刻、新本自己站住 | 不把 PRD 扩成写作规则大全 |

一句话：

```text
V3 保方向，V5 管创作链，当前版管执行纪律。
```

v9 的重点不是“多一条规则”，而是恢复 V5 被验证有效的写作前产物链，避免 writer 只拿到规则和检查项，却拿不到能直接写成戏的材料。

---

## 1. 产品目标不变

产品仍然是：

```text
把一个已被市场验证或值得参考的短剧源本，
洗成一个新壳下仍然好看、能追、能付费的新剧本。
```

第一阶段交付的是首批剧本文字，不是分析报告。

通过标准仍然是三条同时成立：

```text
1. 没洗飞源本有效性；
2. 不是源本改名复刻；
3. 作为一个新短剧自己站得住。
```

---

## 2. 为什么 V5 能 pass，而 V18 不行

V5 的有效点不是“更会写细节”，而是写作前链路更对。

V5 给 writer 的不是一堆规则，而是一组能直接写戏的输入：

```text
源本拆文包
-> 洗稿边界包
-> 新本 beat sheet / outline
-> 新壳场面迁移卡
-> Episode function map
-> Scene packet
-> Screenplay draft
```

这条链会不断追问：

```text
这一集观众等什么？
这一场谁压谁？
谁被迫行动？
信息出现后造成什么现场后果？
关系和权力怎么变？
卡点停在哪个动作前一拍？
```

V18 的问题是：

```text
它恢复了“关键场面迁移卡”的形式，
但没有完整恢复 V5 的 beat / episode map / scene packet 承接链。
```

所以迁移卡一旦选错场面，正文反而会稳定执行错误。

v9 修的是这个结构断点。

## 3. v9 主流程

```text
0. 最小需求卡
1. 源本拆文包
2. 二次需求确认
3. 洗稿边界包
4. 新本 beat sheet / outline
5. 高压节点场面选型
6. 新壳场面迁移卡
7. Episode function map
8. Scene packet
9. Screenplay draft
10. Rewrite report / 短剧化返修
11. Dialogue polish
12. 作者 quality gate + story memory checkpoint
13. 干净 reviewer
14. 主控版本对比与判定
```

注意：

```text
不是每一步都写厚。
高压节点厚写，普通过渡薄写。
```

这版恢复 V5 的主链，但吸收当前版的减法原则：

- 不为完整而完整；
- 不把所有字段都每集硬填；
- 不把 reviewer 当创作替代；
- 不把 manifest complete 当质量 pass。

## 4. 0. 最小需求卡

目标：先拿到足够开工的信息，不一开始把用户问爆。

必须确认：

```text
任务类型：洗稿 / 改编 / 重写 / 诊断 / 审稿
源本：
新壳：
目标市场：
集数是否变化：
改写力度 / 变量边界：
本轮输出范围：
输出语言 / 审稿语言：
正文用途：人审稿 / AI 视频生成用剧本 / 中英双语 / 字幕台词版
已知禁区：
```

默认：

```text
台词英文；
动作和场景描述中文；
内部分析中文；
目标是 AI 出海短剧剧本文字。
```

如果用户没有明确说改写力度，默认是：

```text
中度重构：
保留源本追看发动机、阶段钩子、情绪走势、爽点周期、关系拉扯；
重建高辨识事件外形、场景、道具、证据形式、角色职业身份和具体对白。
```

---

## 5. 1. 源本拆文包

目标：不是复述剧情，而是把源本为什么好看、为什么能追、为什么能付费拆成下游能用的创作材料。

输出结构沿用 V5 / workflow v2 的有效形态：

```text
1. 一句话故事核
2. 核心期待
3. 首批 1-10 集源本功能
4. 高价值节点表
5. 钩子与卡点
6. 爽点 / 压抑-释放
7. 关系 / 权力变化
8. 信息差与真相路径
9. 写作手法 / 表达手法
10. 源本水分 / 烂点 / 禁止复用外形
```

高价值节点必须写清：

```text
具体发生什么；
观众为什么痛、爽、急、气、期待；
强度靠什么成立；
谁压谁、谁退、谁误判、谁失去主动权；
卡点停在哪一拍；
哪些外形不能复用。
```

通过标准：

```text
能让下游知道“保什么体验、避什么外形、怎么写成新戏”。
```

失败信号：

```text
只剩剧情摘要；
只剩“有钩子、有反转、有爽点”；
只说源本功能，不说强度来源；
没有把钩子、拉扯、情绪走势、信息差拆出来。
```

---

## 6. 2. 二次需求确认

目标：读完源本后，主创先判断新壳能不能承载源本，再让用户确认关键取舍。

主创必须先给建议，不把问题全甩给用户。

必须确认：

```text
新壳是否能承载源本最强追看机制；
本轮改写力度；
哪些变量贴近源本；
哪些变量等效迁移；
哪些变量自由改；
哪些源本外形必须避开；
哪些源本烂点必须删；
集数变化时，钩子和留存阶段如何重排；
用户特别想保留或特别不想要什么。
```

如果用户需求会明显削弱源本有效性，必须直接提示风险。

---

## 7. 3. 洗稿边界包

目标：把“这次怎么洗”先固定，避免后面写着写着漂。

输出结构：

```text
必须保留的体验功能：
必须换掉的源本外形：
可以自由换的外壳变量：
必须删除 / 修复的烂点：
需要用户拍板的问题：
```

禁止：

```text
不要扩成万能变量矩阵；
不要把“强继承”理解成照抄事件；
不要为了去同构丢掉源本好看的节奏、钩子、关系和情绪。
```

---

## 8. 4. 新本 beat sheet / outline

目标：先搭新本骨架，再进入单集和场面。

输出结构：

```text
Story Engine：
- 主角想要什么
- 阻碍是什么
- 利害关系是什么
- 新壳里冲突为什么会自然发生

Beat List：
- Beat 编号
- 对应阶段
- 新本事件
- 源本体验功能引用
- 局势变化
- 情绪弧变化
- 不可提前消费内容

Emotional Arc：
- 首批情绪走势
- 拉扯 / 误会 / 追悔 / 反击如何递进
```

通过标准：

```text
不能只是剧情梗概；
每个 beat 至少改变局势、关系、信息或情绪中的一项；
必须标出不可提前消费的后续内容；
必须能看出源本追看发动机被迁移到了新壳。
```

---

## 9. 5. 高压节点场面选型

这是 v9 相比 V5 和 V8 都要显性补回的一步。

它不是新规则表，而是防止“源本外形直接回流”的创作选择动作。

适用范围：

```text
只做高压节点、同构高风险节点、首批开头、强危机、强羞辱、强打脸、身份揭露、火葬场、首批结尾。
```

每个高压节点先做轻量选型：

```text
源本节点：
必须迁移的体验 / 强度：
源本高辨识动作形状：

候选 A：
- 新壳场面是什么：
- 为什么能保强度：
- 同构风险：
- 短剧可见性：

候选 B：
- 新壳场面是什么：
- 为什么能保强度：
- 同构风险：
- 短剧可见性：

候选 C：可空

选择：
淘汰理由：
```

通过标准：

```text
选中的场面必须来自新壳故事发动机；
不能只把源本场景、道具、动作换名；
不能只因为“好解释、好留证、好过检”就选；
必须保住源本强度，不把强危机洗成文件、会议、冷对白；
如果所有候选都像源本外形，回到 beat sheet / 洗稿边界包重想。
```

这一步要解决的是：

```text
先选对戏，再写卡。
```

不是：

```text
先写卡，再靠 reviewer 抓错。
```

## 10. 6. 新壳场面迁移卡

目标：把选中的新壳场面写成正文能直接使用的动作链。

输出结构沿用 V5 有效版本：

```text
源本节点：
源本体验功能：
源本强度来源：
源本禁止外形：
选中的新壳场面：
新场景目的：
新壳故事发动机如何支撑这一场：

新冲突动作链：
  1. 起手动作：
  2. 压迫升级：
  3. 反击 / 反转 / 选择：
  4. 不可逆后果：

权力变化：
关系变化：
信息差变化：
情绪变化：

观众直接看到 / 听到什么：
证据、文件、录音、屏幕信息如果出现，会触发什么人物行动：
台词功能上限：
卡点停在什么动作、反应、选择或后果前：
禁止提前消费：

结论：可进入 episode map / 需重建 / 放弃
```

不通过：

```text
只是“放证据 -> 众人震惊 -> 反派否认 -> 再放证据”；
人物没有被迫行动；
权力、关系、信息或情绪没有变化；
只能靠对白讲出情绪；
卡点已经把下一集核心爽点说完；
新场面只是在源本事件上换名词。
```

通过：

```text
这一场有具体动作链；
每个重要信息都会造成现场后果；
观众能看见谁失去什么、谁压过谁、谁被迫选择；
writer 能直接拿它写正文。
```

## 11. 7. Episode function map

目标：把首批每集的追看功能、最大刺激点和结尾钩子固定下来。

字段：

```text
Episode：
Title：
Summary：
Opening Hook：
Core Friction：
Max Spike：
Change：关系 / 局势 / 信息 / 情绪至少一项变化
End Button：
Source Function Ref：
Forbidden Shape：
Do Not Consume：
Pressure Progression：起压 / 升压 / 爆点 / 收住
Carry-through From Source：源本节点 -> 本集压力推进
```

通过标准：

```text
每集必须有 Opening Hook、Max Spike、End Button；
高压集必须写清 Pressure Progression；
Source Function Ref 不能回指源本外形，只能回指源本追看功能；
连续两集不能只是同一种场面形态重复。
```

---

## 12. 8. Scene packet

这是 V5 能 pass 的关键，不允许再被抽掉。

目标：正文前给 writer 的真正输入。

Scene packet 不是分析报告，不是证明材料，也不是审计底稿。

它只服务一件事：

```text
让 writer 能直接写出一场可看、可演、能推进的短剧场面。
```

每集结构：

```text
Episode N Scene Packet

上一集结束状态：
本集 Opening Hook：
本集目标：
本集 Source Function Ref：
本集 Max Spike：
本集 End Button：
本集 Do Not Consume：
本集 Pressure Progression：
本集 Carry-through From Source：

Scene Sequence：
```

每个高压 scene 至少写：

```text
Purpose：
Characters：
Location：
Starting Power State：
Action Chain：
Evidence / Object：
Evidence triggers what action：
Relationship Shift：
Power Shift：
Information Shift：
Dialogue Functions：
Performance / Body / Space：
Exit State：
End Button：
```

注意：

```text
这些标签只存在于 scene packet。
最终正文不能出现 Purpose / Function / Source Ref 这类内部标签。
```

通过标准：

```text
writer 不需要回头翻厚分析，也能知道这一场怎么写；
每个高压场都有行动、身体/空间/物件/声音、关系变化和现场后果；
对白有功能，不是聊天；
卡点能停在画面或动作上，不是解释上。
```

失败信号：

```text
只写“揭露 X / 证明 Y / 让 A 知道 B”；
没有动作链；
没有角色状态；
没有空间压力；
没有卡点画面；
像会议纪要，不像剧本前准备。
```

## 13. 9. Screenplay draft

目标：交付真正的剧本文字。

默认格式：

```text
EPISODE N

INT./EXT. 场景 - 时间

动作行：只写观众能看见、听见、感受到的外部行为、空间变化、物件、声音、身体反应。

角色名
English dialogue.
```

正文要求：

```text
1. 正文不能出现 Purpose / Function / Source Ref / Gate / 这一场的作用是。
2. 每场必须有当前行动，不写空泛心理说明。
3. 高压场不能靠多人低信息对白拖时间。
4. 台词要有功能：推进、压迫、遮掩、误判、试探、反击、卡点。
5. 能用动作、空间、物件、身体反应、当众后果表达的，不优先用解释性对白。
6. 结尾钩子停在动作、反应、选择或后果前一拍。
7. 不写孤立占位动作，例如只写“她抬眼 / 他僵住 / 全场安静”，除非前后有具体动作和状态支撑。
8. 高压场必须消化 scene packet，不能只借一句口号。
```

低压过渡可以克制，但不能空转，不能用无效聊天冒充推进。

## 14. 10. Rewrite report / 短剧化返修

目标：作者先修正文里明显不像短剧、不像剧本、像说明书的地方。

只看正文，不重开大方案。

检查：

```text
1. 哪些场面像会议纪要 / 证据播报；
2. 哪些场面台词多但信息少；
3. 哪些高压节点没有动作、空间、身体反应或当众后果；
4. 哪些卡点说完了；
5. 哪些场面同构风险被正文放大；
6. 哪些角色状态接不上。
```

返修原则：

```text
能局部修就局部修；
如果问题来自场面选型或 scene packet，不能硬在正文上贴动作，必须回上游重做对应场面。
```

---

## 15. 11. Dialogue polish

目标：处理台词功能、潜台词、节奏和角色声音。

不是把台词改口语。

每句台词至少服务一类功能：

```text
压迫；
遮掩；
误判；
试探；
攻击；
反击；
求救；
拒绝；
威胁；
揭露；
卡点。
```

失败信号：

```text
多人轮流说短句，但没有新信息、新动作、新情绪；
角色像冷静成年人聊天，高压节点没有失控边缘；
台词解释已经能用画面表达的东西；
英文语法硬伤影响阅读。
```

---

## 16. 12. 作者 quality gate + story memory checkpoint

作者必须先自检，但作者自检不能替代 reviewer。

作者 quality gate 看：

```text
洗飞；
改名复刻；
短剧降级；
正文长剧化；
状态漂移；
钩子和卡点是否说完；
关系拉扯是否变平；
scene packet 是否真正进入正文。
```

Story memory checkpoint 记录：

```text
每个主角当前知道什么；
误会还剩什么；
秘密揭到哪一层；
关系状态；
权力状态；
身体/创伤状态；
后续不能忘的伏笔。
```

---

## 17. 13. 干净 reviewer

Reviewer 每次单开临时干净上下文。

### 第一轮

第一轮只读：

```text
最小需求卡；
首批正文。
```

第一轮判断：

```text
这是不是一个能看的短剧首批；
有没有明显长剧化、程序化、说明书化；
有没有台词水、人物冷、视听弱、卡点说完；
是否不看源本也站得住。
```

### 第二轮

第二轮再读：

```text
源本拆文包；
洗稿边界包；
beat sheet / outline；
场面选型；
场面迁移卡；
episode map；
scene packet；
必要源本片段或源本节点。
```

第二轮判断：

```text
有没有洗飞源本有效性；
有没有改名复刻；
有没有把源本强戏洗成弱场面；
有没有把钩子、卡点、爽点周期、关系拉扯、节奏惯性洗丢；
场面选型是否正确；
scene packet 有没有真正进入正文；
失败如果存在，应该回到哪一步。
```

Reviewer 输出只允许：

```text
pass / revise / block
```

如果是 revise，必须说明失败层：

```text
需求没对齐；
源本没读透；
洗稿边界错；
beat / outline 错；
场面选型错；
场面迁移卡错；
episode map 断；
scene packet 弱；
正文执行弱；
局部台词/场面可修；
状态连续性断。
```

## 18. 14. 主控版本对比与判定

主控必须看：

```text
manifest；
正文；
作者 quality gate；
story memory checkpoint；
reviewer 第一轮；
reviewer 第二轮；
和 V3 / V5 / 上一版本的对比。
```

主控只能给四种结论：

```text
可给用户/导演看；
局部 revise；
回上游重做方案；
暂停，不再修这个样本。
```

判断重点：

```text
V3：方向底线，不能比 V3 更乱；
V5：当前最重要的 pass 参考，尤其 scene packet 和高压场面链；
上一版：看本轮到底解决了什么，又引入了什么问题。
```

如果问题只在 1-2 个局部场面，才做局部 revise。

如果问题是整体不像剧本、整体程序化、整体丢源本追看冲动、整体又像源本剧情链，不允许继续在正文上打补丁，必须回到场面选型、scene packet 或 beat sheet。

## 20. 当前样本验证要求

下一次用本 workflow 跑飞书三星样本时：

```text
源本：[修改版] 黑手党少主痛哭求原谅_前妻她杀疯了
新壳：海外豪门商战 / 家族企业权力斗争
目标市场：海外短剧
集数：不变
输出范围：首批 1-10 集
改写力度：中度重构
正文默认：中文动作/场景描述 + 英文对白
```

跑完后必须整体对比：

```text
V3：方向底线；
V5：当前 pass 参考；
V18：恢复关键场面卡但失败的证据；
新版本：本轮产物。
```

对比不能只看旧问题。必须整体看：

- 源本有效性；
- 同构风险；
- 新壳是否自然长剧情；
- 高压场面选型；
- Episode map 和 Scene packet 是否真的支撑正文；
- 关系拉扯；
- 钩子和卡点；
- 情绪和爽点递进；
- 台词功能；
- page-to-screen 剧本感；
- 是否过度程序化；
- 是否比 V5 更值得继续。
