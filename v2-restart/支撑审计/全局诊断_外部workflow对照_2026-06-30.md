# 全局诊断：外部 workflow 对 v3 问题的解释

日期：2026-06-30

本文不是 PRD，不是规则清单，也不是新的执行模板。

本文只回答一件事：当前样本为什么连续暴露问题，外部成熟项目到底能解释什么，我们下一步应该把注意力放在哪一层。

---

## 1. 总判断

当前项目不是缺几个“视听语言 / 落差 / 拉扯感”的规则。

这些点都是真的，但它们只是导演反馈里暴露出来的症状。更大的问题是：

**源本分析到新本正文之间，中间工序还不够硬。**

具体说，PRD v3 已经写了源本有效性、变量迁移、同构红线、阶段骨架、分集功能表、关键场景草案，但执行时仍然容易发生两类失败：

1. **事件载体同构**：源本的事件链、强桥段、证据形态、冲突方式被整串带进新本，只换人名、身份和场景名。
2. **短剧场面降级**：为了避开同构，事件载体换了，但新场面变成文件、会议、冷对白、解释性信息交付，源本的视觉冲击、情绪爆发和短剧感被洗淡。

这两类失败看起来相反，其实都来自同一个断点：

```text
源本有效性分析
  -> 没有足够强地转成“新事件载体 / 新场面承载方案”
  -> 正文写作只能靠模型临场发挥
  -> 最后要么沿用源本外形，要么换成安全但弱的文件对白场
```

---

## 2. 导演反馈不是碎点，是分层问题

导演反馈可以归成几层，而不是一百条规则。

| 层 | 典型反馈 | 本质问题 |
|---|---|---|
| 事件载体层 | 第 2 集“车祸”被换成“安全档案室”，冲击力比不了 | 新壳事件变了，但承载强度降级 |
| 场面承载层 | 第 6 集录音方向对，但最好叠加监控视频 | 信息不能只靠文件/录音，要转成观众能直接看到的画面证据 |
| 台词层 | 第 7 集台词太多，像电视剧，来回反复 | 正文缺少台词压缩和功能筛选，人物在聊天而不是行动 |
| 情绪外化层 | 第 8 集发现背叛、求饶、惩罚都不够 dramatic | 高压节点缺身体反应、动作、空间压迫、当众后果 |
| 关系拉扯层 | 男女主像冷静成年人谈事，缺反差和戏剧性 | 人物关系没有通过靠近/退开、误会/试探、伤害/保护发生变化 |
| 落差层 | 第 4 集开头就高姿态，缺众人不信再被打脸 | 爽点前没有足够压低、看衰、误判，翻转不够爽 |
| 卡点层 | 第 3 集女主回归如果说话，可能把气口说掉 | 集尾没有停在最想看的前一秒，而是把期待说完 |
| 集内经济层 | 第 2 集 2-1 可用男主一句话 cover，单开一场拖节奏 | 单集内有重复功能场，短剧时间被水掉 |

所以问题不是“有没有视听语言”这么窄。

问题是：**写正文前没有把每个高压节点设计成一个新的、可拍的、短剧化的场面。**

---

## 3. 真正根因

### 3.1 PRD 有流程骨架，但工序承压不够

PRD v3 的方向是对的，它已经避免了最早那种“直接改名写正文”的风险。

但现在的问题是：很多产物还停留在“要想清楚什么”，没有完全变成“进入正文前必须过的关口”。

例如：

- `源本节点拆解稿` 会问节点功能，但还不一定强制扣出“源本外形不能复用的部分”。
- `迁移义务与同构红线稿` 会问同构风险，但还不一定要求给出一个足够强的新事件载体。
- `关键场景草案` 会看高压节点，但还不一定把“短剧场面承载”作为写正文前的硬门。
- `作者自检 / reviewer` 会看不可拍和台词问题，但更像写完后发现问题，不一定能在正文前拦住。

### 3.2 写作中间层断裂

当前链路容易变成：

```text
知道源本有效
知道要换壳
知道不能太同构
然后直接写正文
```

缺的是：

```text
源本这个节点靠什么事件/画面/关系/证据成立
哪些源本外形必须扣掉
新壳用什么完全不同的事件载体承载同一体验
这个新事件有没有短剧场面冲击
这个场面有没有动作、身体、空间、当众后果
台词是不是只剩必要功能
集尾是不是卡在气口
```

### 3.3 Reviewer 没有分层判错

过去 reviewer 容易把“源本有效性迁移了”和“新本不是改名复刻”混在一起。

实际上它们是两个独立硬项：

```text
有效性没迁移 -> 洗飞
事件载体同构 -> 改名复刻
事件载体不同但场面弱 -> 短剧降级
```

这三个失败不能互相抵消。

---

## 4. 外部 workflow 共同证明了什么

这轮子代理和主控复核的结论一致：外部成熟项目里没有一个可以直接搬来的“短剧洗稿器”，但成熟项目共同证明了一件事：

**它们不会从分析直接跳正文。**

它们都有正文前的中间产物和审核门。

### 4.1 0xsline/short-drama

它是从零短剧创作，不是洗稿。

完整链路是：

```text
/start
-> /plan
-> /characters
-> /outline
-> /episode
-> /review
-> /compliance
-> /export
```

它证明：成熟短剧流程至少需要先有创作方案、角色体系、分集目录，再写单集正文，再做 review。

可借的是“先方案、再人物/分集、再单集、再审稿”的工序感。

不能搬的是固定比例、题材模板、付费卡点数字，因为我们不是从零原创，也不是标准库套模板。

### 4.2 Toonflow-app

它最接近“源材料改编链”。

它的链路是：

```text
导入原著/源材料
-> 章节事件提取
-> 项目参数确认
-> 故事骨架
-> 监督审核骨架
-> 改编策略
-> 监督审核策略
-> 逐集剧本
-> 分镜/资产/视频生产
```

它证明：源材料不能直接变正文，中间要经过“事件表、故事骨架、改编策略、监督审核”。

这能直接解释我们的失败：

- 没有先扣掉源本事件外形，就会改名复刻。
- 没有先设计新事件载体，就会靠模型临场换成文件/会议。
- 没有策略审核，就会等正文写完才发现方向不对。

### 4.3 how-to-make-script

它不是短剧洗稿器，但它的分层诊断和输出契约有价值。

它的链路是：

```text
请求分类
-> 锁定意图/媒介/阶段/输出/约束
-> 加载对应 protocol + rubric
-> 生成指定产物
-> 按产物自检
-> 质量门审查
```

它证明：诊断、重写、对白打磨、质量审查不是一锅炖。

对我们来说，Reviewer 应该能判断失败发生在哪一层：

- 源本分析层；
- 事件载体迁移层；
- 场面承载层；
- 正文对白层；
- 连续性追踪层。

### 4.4 oh-story

它是网文写作和拆文工程，不是短剧。

但它的拆文和追踪机制很有价值：

```text
扫榜
-> 拆文
-> 拆文库
-> 大纲/细纲
-> 正文
-> 状态追踪
-> 去 AI 味/审查
```

它证明：拆文不是剧情摘要，而是要拆出情节节点、情绪强度、写作手法、关系状态、伏笔、时间线。

对我们来说，源本节点拆解不能只写“这个节点功能是什么”，还要写：

- 它靠什么场面/证据/动作成立；
- 关系状态在这里怎么变化；
- 信息差在这里怎么变化；
- 情绪从哪里压到哪里、从哪里爆出来；
- 哪些外形不能复用。

### 4.5 huobao-drama / Jellyfish / ReelForge-YAML

这些更偏剧本到分镜、资产、视频生产或 YAML 质检。

它们不能替代编剧流程，但它们证明：

```text
剧本文字最终必须能变成镜头、动作、人物状态、道具状态和可见后果。
```

它们能解释 v3 为什么像长剧：

- 文件很多，但场面动作少；
- 信息很多，但人物身体反应少；
- 证据很多，但权力变化不够可见；
- 对白很多，但可分镜的动作少。

这说明“视听语言”不是后期分镜再补的东西。剧本阶段就必须写出可被分镜拆开的场面。

## 5. 当前项目缺的不是规则，而是三个关口

### 关口一：事件载体重建

位置：源本节点拆解之后，正文写作之前。

要解决的问题：

```text
源本这个节点为什么有效？
源本用了什么具体事件外形承载？
这些外形哪些不能复用？
新壳下用什么新事件承载同一体验？
```

没有这个关口，就会改名复刻。

### 关口二：短剧场面等效

位置：关键场景草案阶段。

要解决的问题：

```text
新事件虽然不同，但它有没有短剧场面冲击？
观众直接看到的是什么？
人物身体/动作/空间/当众后果在哪里？
权力、关系、情绪有没有可见变化？
```

没有这个关口，就会把车祸洗成档案室，把情绪爆发洗成文件会议。

### 关口三：正文后压缩返修

位置：正文初稿之后、作者自检之前。

要解决的问题：

```text
哪些台词是水词？
哪些信息可以变成动作/道具/画面？
哪些高压节点人物太冷静？
哪些卡点说完了？
哪些场景是重复功能？
```

没有这个关口，就会出现第 7 集那种冷静聊天、低信息多轮对白。

---

## 6. 不应该怎么做

不要把上面这些变成每集必填大表。

不应该写成：

```text
每集都检查视听语言、落差、拉扯、台词、卡点、情绪密度、信息密度……
```

这会回到规则怪谈。

更合理的方式是：

```text
全剧层：阶段骨架控制大方向
分集层：分集功能表控制留存和推进
高压节点层：事件载体重建 + 短剧场面等效
正文层：写完后压台词、去水词、校卡点
Reviewer 层：分层判错，指出失败发生在哪一层
```

也就是说，不是增加更多规则，而是把已经存在的流程接实。

---

## 7. 现在最该解决的产品问题

当前最核心的问题不是 PRD 厚不厚，而是：

**PRD v3 的主流程已经有骨架，但“源本分析 -> 新本正文”之间的工序还需要更像成熟生产流程。**

要补的不是知识点，而是产物之间的关系：

```text
源本节点拆解稿
  -> 事件载体重建
  -> 关键场面等效
  -> 正文写作
  -> 台词/场面/卡点返修
  -> 作者自检
  -> 独立 reviewer 分层审稿
```

这里每一步不是为了多写文档，而是为了防止三种失败：

1. **改名复刻**：源本事件外形没扣掉。
2. **短剧降级**：新事件载体弱，场面变文件对白。
3. **正文长剧化**：台词水、人物冷、动作少、卡点说完。

---

## 8. 后续进入 PRD 的候选点

这里先只列候选，不直接改 PRD。

### 候选 1：强化“源本节点拆解稿”

新增或明确它必须拆：

- 源本功能；
- 源本事件载体；
- 源本场面冲击；
- 信息差状态；
- 情绪压低/释放；
- 关系变化；
- 不能复用的高辨识外形。

### 候选 2：把“迁移义务与同构红线稿”改得更硬

它不能只说“哪些不能照搬”，还要给出：

- 新事件载体；
- 新证据形态；
- 新压力来源；
- 新行动方式；
- 新场面承载。

如果没有新承载方案，不能进入正文。

### 候选 3：关键场景草案增加“短剧场面等效”

不新增大环节，但在关键场景草案里明确：

- 观众看到的冲击是什么；
- 人物身体反应是什么；
- 空间/道具/声音怎么参与；
- 当众后果是什么；
- 卡点停在哪个动作或信息前。

### 候选 4：正文后增加轻量返修 pass

不是新增一堆检查表，只是正文初稿后做一次：

- 删水词；
- 把解释性台词换成动作/画面；
- 强化高压节点外化；
- 修正卡点；
- 删除重复功能场。

### 候选 5：Reviewer 改成分层判错

Reviewer 不只说好不好，而要指出失败层：

- 源本分析失败；
- 事件载体迁移失败；
- 场面承载失败；
- 正文对白失败；
- 节奏/卡点失败；
- 连续性追踪失败。

---

## 9. 当前结论

Cloud 那句“找到根了”基本是对的。

真正的根不是“我们不知道视听语言”，也不是“还没测试够”，而是：

**我们没有完整吸收成熟项目的生产工序，只从外部资料里摘了很多维度词，导致分析、自检越来越厚，但写作中间层仍然不够稳。**

现在应该做的是：

1. 先停止扩规则；
2. 把外部项目拆出来的工序吸收成一条简洁生产链；
3. 再判断 PRD v3 哪些地方只需要改措辞，哪些地方需要补硬门；
4. 最后再落 PRD，而不是现在边想边堆。
