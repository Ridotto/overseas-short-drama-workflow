# 生产链能力执行对账

> 日期：2026-07-02  
> 状态：生产链审计，不是新 workflow，不是返修单。  
> 范围：只查 S0-S8 生产链，暂不管 reviewer。  
> 目的：逐项确认“重要短剧能力 / 外部知识 / 历史建议”是否真正变成 agent 执行动作。  

---

## 0. 先说结论

当前主要问题不是“文档没写”，而是：

```text
很多重要能力已经写进 workflow / 对账表 / skill 工位链，
但在实际运行里只停留在“出现过、提到过、声明过”，
没有变成下游必须依赖的生产产物。
```

这类能力可以分三档：

| 状态 | 含义 |
|---|---|
| 真执行 | 有明确产物，正文能看到它产生的效果 |
| 半执行 | 文档有、产物也写了，但没有稳定转成正文强度 |
| 假执行 | 文档有名字，实际运行里没有变成动作或阻断 |

V28 是很好的对照：

- 深度重构让“距离轴”动了：旧外形拉开。
- EP1-EP3 证明生产链可以写出更强画面。
- EP6-EP10 证明很多短剧能力仍然没有稳定执行，尤其是反派压力、爽点压放、证据转人戏、付费追问和场面发动。

所以当前不是继续问“还有什么规则要加”。

当前应该问：

```text
这项能力对短剧重要吗？
重要的话，在哪一层必须产出？
下游是否只能依赖这个产物？
失败时是否会卡住？
如果不会，它就是软建议，不是运行约束。
```

## 1. 对账口径

每条能力按五个问题查：

1. 是否对短剧洗稿产物重要；
2. 文档里是否有；
3. V28 实跑是否执行；
4. 如果没执行，断在哪一层；
5. 结论：保留、补硬门、改产物形态，还是降级为参考。

本次不把 reviewer 纳入生产能力判断。

目标是：

```text
没有 reviewer，S0-S8 跑完也应该产出 70-80 分的 author-ready 稿。
```

## 3. 目前最重要的断点

### 3.1 S4 没有真正卡住“证据承载”

文档说：

```text
S4 禁止用文件、屏幕、会议、系统替代人和人的现场冲突。
```

但 V28 里：

- EP6 用旧音频；
- EP7 用定位图；
- EP8 用 DNA 结果页；
- EP10 用文件和终止按钮。

不是说这些都不能出现，而是它们连续成为后半段主推进。

结论：

```text
S4 的禁令是软建议，不是运行硬门。
```

### 3.2 S5 没有真正变成“逐拍演出稿”

V28 S5 的确比普通提纲强，但它仍然是：

```text
开场第一拍 / 中段 / 压低 / 释放 / 结尾
```

不是：

```text
谁做了什么；
谁被迫反应；
谁失去什么；
谁看见了；
下一拍为什么更糟。
```

所以 S6 可以顺着它写出正确剧情，但不一定写出强戏。

结论：

```text
S5 的产物形态还是半提纲，不是强制逐拍施工。
```

### 3.3 外部项目能力在 S5 里没有逐项落地

v16 写了：

```text
opening-rules / rhythm-curve / hook-design / satisfaction-matrix / paywall-design / villain-design
```

但 V28 S5 没有明确展示：

- 本集使用哪种开场法；
- 本集节奏曲线如何压放；
- 本集钩子类型和兑现义务是什么；
- 本集爽点属于哪类、压低动作是什么、释放动作是什么；
- 本集付费追问是什么；
- 本集反派下一步反扑资源是什么。

结论：

```text
外部能力被“写进了要求”，没有变成 S5 的必填生产动作。
```

### 3.4 反派燃料没有从 S3 传到 S5

V28 S3 写了 Owen：

```text
港口承包商，Serena 真正情人。
```

但正文 1-10 集 Owen 没有出现，也没有成为 Serena 反扑的实际资源。

这说明：

```text
“反派资源 / 误判 / 反扑燃料”只是角色表字段，没有被强制进入分集动作。
```

### 3.5 强弱分配没有成为 S3 产物

v16 要求 S3 写：

```text
强弱分配：每个阶段 / 批次的波峰、波谷、兑现点和追问点。
```

V28 S3 只有阶段功能，没有具体强弱分配。

结果是：

- EP1-3 很强；
- EP4-5 还可以；
- EP6-8 变成真相递交；
- EP10 有卡点但机构按钮残影。

结论：

```text
S3 的“强弱分配”是假执行。
```

## 4. 现在不是缺哪些东西，而是哪几类东西是假执行

### 真执行，可以保留

- 防改名复刻；
- 深度重构距离控制；
- DNC；
- 基础 story memory；
- 台词压短；
- 前段 page-to-screen；
- 结尾卡点意识；
- 源本功能链继承。

### 半执行，需要改产物形态

- 源本强度来源；
- 爽点压放；
- 情绪密度；
- 关系拉扯；
- 节奏曲线；
- 新壳等效承载；
- S5 writer 施工稿；
- 视听表达。

### 假执行，需要变成运行约束

- S4 禁证据播报；
- S5 逐拍演出；
- S5 短剧手艺加载；
- S3 强弱分配；
- S3/S5 反派燃料；
- 付费追问和兑现义务；
- 候选场面优选。

## 5. 对下一步的实际含义

如果暂时不管 reviewer，生产链要能自己产出 70-80 分稿，下一步不应该再新增知识库。

应该先把以下 6 个“假执行”改成真正的运行约束：

1. **S3 必须产出强弱分配**  
   不是只写阶段功能，要写每批 / 每集的波峰、波谷、兑现点、追问点。

2. **S3 必须把反派燃料压进分集**  
   反派资源、误判、下一步作死不能只在角色表里。

3. **S4 必须单列“谁被迫行动”**  
   如果只是“事实压人 / 规则压人 / 证据压人”，不能进 S5。

4. **S4 必须判断证据是否只是辅助**  
   证据可以存在，但不能成为高压节点本体。高压节点本体必须是人被迫行动、人当众失去、人关系撕裂。

5. **S5 必须改成逐拍施工稿**  
   每个高压节点至少要写：
   ```text
   谁做了什么；
   谁被迫反应；
   谁失去什么；
   谁看见了；
   下一拍为什么更糟。
   ```

6. **S5 必须逐项使用短剧手艺，而不是泛称“使用短剧手艺”**  
   至少要落到：
   - 开场第一拍；
   - 中段升压；
   - 爽点压低 / 释放；
   - 钩子卡点 / 下一集兑现义务；
   - 反派下一步反扑。

这 6 个都不是 reviewer 事项。

它们属于生产链自身。

## 6. 暂不建议做的事

### 不建议继续加 reviewer 规则

当前问题在生产链，不在 reviewer。

### 不建议继续跑深度重构变量

V28 已证明深度重构只能推距离，不能稳定推强度。

### 不建议继续补“视听语言要更强”这类口号

这已经写过很多遍。问题不是没写，是没强制进入 S4/S5/S6 的产物形态。

### 不建议把 V5 当目标

V5 只是证明“动作链能保住强度”，不是产品目标。

### 不建议把所有能力都硬卡

只能硬卡会直接影响剧本是否能追、能付费的生产动作。  
台词还能更狠、画面还能更漂亮、某个点更极致，这类不能硬卡。

---

## 7. 最短人话结论

现在不是缺知识，也不是缺 reviewer。

真正的问题是：

```text
很多重要能力在文档里有，但在生产链里是假执行。
```

尤其是假执行的 6 个点：

```text
S3 强弱分配；
S3/S5 反派燃料；
S4 谁被迫行动；
S4 证据只能辅助；
S5 逐拍演出；
S5 短剧手艺逐项使用。
```

如果这 6 个不变成运行约束，writer 仍然会写出“剧情正确但不够上头”的稿。
