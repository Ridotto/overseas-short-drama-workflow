# 海外短剧洗稿器 · 层级归位表 / 工位合同

> 日期：2026-07-01  
> 状态：候选执行合同，不是新规则库，不是剧本返修单。  
> 目的：解决“文档里有能力，但下游没有真正执行”的问题。

---

## 0. 这份表解决什么

过去几版一直反复出现同一个问题：

```text
源本分析、短剧手艺、外部项目能力、writer 施工稿、reviewer、自检都存在，
但最终剧本仍然不够强。
```

根因不是“缺一条短剧规则”，而是：

```text
能力没有被压成 writer 当下可执行的生产动作。
```

所以这份表不继续补知识点，只规定每一层：

- 只负责什么；
- 必须产出什么；
- 明确禁止产出什么；
- 下游只能读取什么；
- 出错必须打回哪一层。

一句话：

```text
不是让文档更完整，而是让交接物更干净。
```

## 1. 总原则

### 1.1 产品锚点

本项目是洗稿工具，不是原创短剧生成器。

所有工位都必须服务于：

```text
把已被市场验证或值得参考的短剧源本，
洗成新壳下仍然好看、能追、能付费的新剧本。
```

### 1.2 两条能力线必须同时存在

| 能力线 | 负责什么 | 不能替代什么 |
|---|---|---|
| 洗稿迁移线 | 源本为什么好看、哪些体验必须继承、哪些外形必须替换 | 不能变成原创自由发挥 |
| 短剧生产线 | 开场、节奏、爽点、钩子、反派压力、台词、视听表达怎么落成戏 | 不能压过源本有效性 |

0xsline / short-drama 的价值主要在第二条线：短剧生产动作。  
我们自己的价值主要在第一条线：源本有效性迁移。

### 1.3 不能剪了芝麻丢了西瓜

这份表不是把项目收窄成“只管短剧感”。

短剧手艺必须生效，但它不能挤掉洗稿主线。任何新 workflow 或样本运行，都必须同时保住这些能力：

| 能力 | 为什么不能丢 |
|---|---|
| 源本有效性迁移 | 这是洗稿工具的根，不是原创工具 |
| 防洗飞 | 源本为什么能追、能付费，必须活到新本 |
| 防改名复刻 | 不能只是换名字、换职业、换地点 |
| 新壳等效承载 | 新事件必须从新壳自然长出来 |
| 留存 / 钩子 / 卡点 | 原本吊住用户的位置和功能不能被洗没 |
| 爽点 / 压抑释放 | 不能只剩事件推进，丢掉情绪兑现 |
| 情绪密度 / 信息密度 / 情节密度 | 不能空转，也不能只靠解释推进 |
| 主线期待 / 信息差 / 真相揭露 | 观众追什么、什么时候知道什么，必须清楚 |
| 人物关系 / 拉扯 | 人物不能只执行剧情功能，关系张力要推进 |
| 权力变化 / 打脸 | 高压节点必须改变局势，不只是发生事情 |
| 角色状态 / 误判 / 错信 | 当前谁忍、谁崩、谁误会、谁握主动必须连续 |
| 反派压力 / 反扑燃料 | 不能女主一路碾压到剧情变平 |
| Do Not Consume | 后续火葬场、反转、关系击穿不能提前吃掉 |
| 台词功能 / 角色声线 | 台词不能水，也不能所有人一个语气 |
| 视听表达 / page-to-screen | 最终剧本必须能看到、听到、生成画面 |
| 连续性 / 伏笔 / story memory | 分批写作不能状态漂移 |
| 海外市场语感 | 默认出海短剧，不是中文内娱短剧照搬 |
| 合规 / 禁区 | 红线不能靠最后一步临时补 |
| 用户 / 导演反馈回流 | 反馈要归层，不能逐条变规则怪谈 |

如果某次修 workflow 只强化其中一项，却削弱以上任一主能力，就不是修复，是漂移。

### 1.4 writer 不吃噪音

writer 不直接读取：

- run log；
- reviewer report；
- 作者自检；
- 旧版对比；
- 厚源本分析全文；
- 外部标准库全文；
- 流程证明术语；
- 临时诊断和历史失败清单。

S5 施工稿工位只读取上一层给它的生产材料，不能回头翻厚材料：

- S4 场面承载原料包。

S4 必须已经把当前集必要故事状态、洗稿边界、不可提前消费信息、短剧手艺需求和场面原料消化进去。

S6 正文 writer 只读取：

- S5 单集短剧施工稿。

如果 S5 或 S6 需要回头读更早材料，说明上一层交接产物不合格，必须回上一层重做。

## 2. 层级归位总表

| 层级 | 只负责 | 必须产出 | 明确禁止 | 下游只能拿 | 失败打回 |
|---|---|---|---|---|---|
| S0 需求确认 | 拿到可开工边界 | 最小需求确认 | 不分析源本，不定剧情 | 源本、新壳、语言、集数、力度、禁区 | S0 |
| S1 源本拆文 | 拆源本为什么能追、能付费 | 源本拆文包 | 不设计新壳，不把源本外形当必须继承 | 观众追问、压放、钩子、关系/权力变化、DNC、源本高辨识外形 | S1 |
| S2 洗稿边界 | 区分继承体验和替换外形 | 洗稿边界包 | 不把防同构压过防洗飞，不写新剧情 | 强继承体验、 forbidden shape、新壳可承载范围 | S2 |
| S3 新本故事架构 | 搭新壳故事骨架和真相系统 | 新本故事开发包 | 不写具体高压场面外形，不锁源本高辨识动作 | 故事核、真相账本、错信、主追问、阶段骨架、分集功能、角色行动方式 | S3 |
| S4 场面承载原料 | 给 S5 可写戏的原料 | 场面承载原料包 | 不排完整节拍，不写台词，不替 writer 写戏 | 一个可被 S5 直接消化的原料包：谁压谁、谁被迫行动、压力来源、释放后果、禁止外形、DNC、候选新壳动作 | S4 |
| S5 writer 施工稿 | 把 S4 原料排成短剧戏 | 单集短剧施工稿 / 写作卡 | 不证明流程，不写工位状态，不抄上游字段，不读 reviewer，不要求 S6 回头读 S1-S4 | 开场、升压、压放、关系/权力/信息变化、最后一拍、DNC、输出语言、禁用外形 | S5 |
| S6 screenplay 正文 | 只从 S5 施工稿写成可看可听正文 | screenwriting 初稿 | 不写说明书，不出现内部标签，不站桩聊天，不回头打开 S1-S4 | S7 只承接 S6 正文 | S6 |
| S7 台词/视听 polish | 修表达，不改主结构 | polish 稿 | 不重设场面，不替 S4/S5 解决场面发动机 | 台词功能、动作/身体/空间/声音、角色声线 | S7 或回 S5/S6 |
| S8 连续性/信息边界 | 查事实、状态、伏笔、DNC | story memory checkpoint | 不评价口味，不写“还能更好” | 已播信息、未播信息、角色状态、重复消费风险 | S3/S8 |
| S9 作者自检 | 主创侧先发现明显失败并路由 | 作者自检与返修路由 | 不把问题丢给 reviewer，不把风险写成主控偏好 | 失败层级、返修范围、保留体验、禁止外形 | 对应层 |
| S10 干净 reviewer | 独立兜底和分层审稿 | reviewer report | 不生产短剧感，不逐句代写，不无限挑刺 | 只给 issue、严重度、回滚层、建议方向 | 主控转译后回对应层 |
| S11 主控拍板 | 编排、对比、反馈回流、最终判断 | 主控版本对比与拍板 | 不把运行记录完成当质量通过，不让 reviewer 对用户拍板 | 用户可读结论、返修层、是否交付 | 对应层 |

## 3. 核心工位合同

### 3.1 S3 合同：故事架构不能提前锁死高压外形

S3 可以定：

- 这一集的故事功能；
- 真相账本；
- 角色错信；
- 追问和兑现；
- 阶段节奏；
- 角色行动方式；
- 关系/权力/信息状态。

S3 不可以定：

- 枪抵眉心；
- 车祸弃救；
- 厨房签离婚；
- 当众吞纸；
- 与源本高度相似的整串动作；
- 任何已经属于“高压场面外形”的具体桥段。

如果 S3 已经写出这类具体外形，不能交给 S4/S5，必须回 S3 改成：

```text
体验功能 + 新壳可承载方向 + 禁止外形
```

### 3.2 S4 合同：场面承载只给原料，不替 writer 排戏

S4 必须回答：

- 这场承载源本哪个体验功能；
- 新壳里用什么权力/关系/空间/物件承载；
- 谁压谁；
- 谁旁观；
- 谁被迫行动；
- 释放后要造成什么可见后果；
- 哪些源本外形禁止出现；
- 哪些信息不能提前消费。

S4 不写：

- 完整节拍；
- 正文动作段落；
- 台词；
- 结尾具体镜头；
- “枪响黑屏”这类已经是成片级外形的设计。

如果 S4 写出来的不是“原料”，而是一场已经排好的戏，S5 会变成抄写，不是创作。

### 3.3 S5 合同：只吃 S4 原料包，产出给 S6 的施工稿

S5 只拿 S4 场面承载原料包。

本集写作卡必须包括：

1. 本集观众正在等什么；
2. 开场用什么画面/动作/声音抓人；
3. 中段如何升压；
4. 爽点前如何压低；
5. 谁做动作，谁被迫反应，谁失去什么；
6. 本集关系/权力/信息发生什么变化；
7. 结尾停在哪个未完成动作、声音或画面；
8. 哪些信息不能提前消费；
9. 禁止复用的源本外形；
10. 本集加载的短剧手艺。

S5 不拿：

- run log；
- reviewer report；
- 作者自检；
- 旧版对比；
- 厚分析全文；
- S1/S2/S3 原文；
- 工位状态、流程证明、旧版审计语言。

S5 的责任是把 S4 原料消化成 S6 可以直接写正文的一集施工稿。

S6 不能再回头读 S1-S4。

### 3.4 S8 合同：连续性检查不能软放行信息重复

S8 不是审美 reviewer。

但 S8 必须拦住：

- 核心真相自相矛盾；
- 同一信息刀口反复消费；
- 伏笔被提前吃掉；
- 角色错信前后不一致；
- 关系/权力状态跳变；
- 终局本该新鲜的痛点被前面说完。

如果 S8 记录了问题却给 pass，就是层级失效。

### 3.5 S9 合同：作者自检不能把硬风险写成偏好

如果作者自检发现：

- 源本高辨识外形回流；
- 新壳载体承载不住；
- 施工稿强度没有活到正文；
- 信息提前消费；
- 核心真相混乱；

不能写成：

```text
可由主控偏好决定。
```

必须直接写：

```text
应回 S3 / S4 / S5 / S8。
```

## 4. 外部项目能力如何变成生产动作

| 外部能力 | 不能怎么用 | 必须怎么接入 |
|---|---|---|
| `opening-rules.md` | 不能写成“开场要强” | S5 写作卡必须写“开场第一个可见动作/声音/冲突” |
| `rhythm-curve.md` | 不能变成节奏评分 | S3/S5 必须写本集开场、中段、结尾的压放结构；S8 查重复刀口 |
| `hook-design.md` | 不能只写“有钩子” | S5 必须写未回答问题、卡住的动作、下一集兑现义务 |
| `satisfaction-matrix.md` | 不能只标爽点类型 | S4/S5 必须写压低动作和释放动作 |
| `paywall-design.md` | 不能照搬固定集数比例 | S3/S5 必须写付费追问和兑现义务 |
| `villain-design.md` | 不能只加坏人设定 | S3/S5 必须写反派本阶段资源、误判、反扑燃料 |
| `genre-guide.md` | 不能让题材压过源本 | S0/S3/S7 用来约束市场、题材语感、角色说话方式 |
| `compliance-checklist.md` | 不能变成创作发动机 | S10/S11 只作红线和风险辅助 |

## 5. 何时不能继续往下走

### S3 不能继续

- 核心真相账本没锁；
- 主追问和阶段追问不清；
- 分集功能只是剧情梗概；
- 直接写入源本高辨识高压外形。

### S4 不能继续

- 只有“证据出现 / 文件曝光 / 系统宣判”；
- 没有谁被迫行动；
- 没有可见后果；
- 把完整场面替 writer 排好了；
- 新壳载体比源本强度明显降级。

### S5 不能继续

- 施工稿像任务单；
- 没有一拍一拍的行动链；
- 没有压低和释放；
- 结尾把答案说完；
- 仍然照抄源本外形。

### S6 不能继续

- 正文像说明书；
- 内部标签进入正文；
- 高压场只靠台词/屏幕/报告；
- 人物站着聊天；
- 施工稿强度没有落到动作、空间、身体、声音、物件。

### S8 不能继续

- 同一信息被多集重复当主刀；
- Do Not Consume 失效；
- 角色错信不一致；
- 核心真相矛盾。

## 6. 当前状态判断

这份表不是证明流程已经好了。

它只是把目前确认的根因压成可执行合同：

```text
能力必须进入正确工位；
工位必须产出干净交接物；
writer 只能吃生产卡；
检查层只能回滚，不生产短剧感。
```

下一步应基于这份表写新的候选 workflow，并用同一飞书三星样本完整验证。
